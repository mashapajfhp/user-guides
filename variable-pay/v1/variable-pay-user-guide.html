What is Variable Pay?

Overview

Variable Pay manages non-recurring compensation items like bonuses, commissions, allowances, and one-time payments. Access it through Settings → Payroll → Variable Pays or directly from the payroll table by clicking on an employee row.

Create custom pay items, assign them to employees via the payroll table, and track all variable compensation in one place. The payroll table supports multi-currency views (e.g., AED, USD, AUD) with separate tabs for each currency.

Key Benefits

  • Replace manual bonus tracking spreadsheets with system-managed records
  • Configure unlimited custom pay items for different compensation types
  • Bulk upload variable pay items across multiple employees
  • Automatic inclusion in payroll runs and WPS file generation
  • Complete audit trail of all variable compensation entries

Product Foundation Overview

How Variable Pay Fits

Variable Pay is transactional data that captures one-time or recurring compensation adjustments added to employee payroll cycles.

Mental model: Variable Pay Item (pay item + amount + associated period) → Added to Employee via Payroll Table → Calculated in payroll run → Appears on payslip

Each variable pay entry affects a single employee's compensation for the specified payroll period.

Key Decisions Before Setup

Answer these questions before creating variable pay items:

  • What pay items do you need? — Define pay items (bonus, commission, allowance) in Settings → Payroll → Variable Pays before adding entries
  • How will amounts be determined? — Determine fixed amounts for each employee and pay period
  • Which employees receive variable pay? — Individual entry vs. bulk upload for multiple employees
  • When should payments be processed? — Align variable pay entries with payroll cycle dates

Variable pay is managed through two primary entry points in Bayzat.

Pay Item Configuration

Create and manage variable pay items (bonuses, commissions, allowances).

Settings → Payroll → Variable Pays

Enter variable pay amounts for employees during each payroll cycle.

Payroll → Payroll Table → Click Employee Row → Variable Pay Section → Add/Edit Variable Pay Item

Key actions available:

Initial Setup

First-Time Configuration

Before using Variable Pay, configure pay item categories in Settings.

Create Variable Pay Items

  1. Navigate to Settings → Payroll → Variable Pays (expand the accordion section)
  2. Click Add new
  3. Enter the pay item name (e.g., Bonus, Commission, Overtime)
  4. Click Save

Categories appear in the payroll table once created and can be assigned to employees.

Required Settings

Complete these configurations before processing payroll with Variable Pay:

  • Payroll Cycle: Ensure payroll cycle dates are set in Settings → Payroll → Payroll Cycle (contact support to modify cycle dates if needed)
  • Employee Records: Add employees to the Payroll Management table before assigning Variable Pay items
  • Pay Item Configuration: Create at least one Variable Pay item in Settings → Payroll → Variable Pays (Recommended: create all pay items your organization uses before entering data)

Access Verification

Confirm you can access Variable Pay features:

  1. Navigate to Payroll → Payroll Table
  2. Click on any employee row to open the detail slide-over panel
  3. Verify the Variable Pay section appears with an "Add/Edit Variable Pay Item" button

If Variable Pay tab is missing, contact support to enable the module.

Key Tasks

Task: Manage Variable Pay Items

Configure pay items that define additional compensation types added to employee payroll.

Subtask: Create Variable Pay Item

  1. Navigate to Settings → Payroll → Variable Pays (expand the accordion section)
  2. Click Add new
  3. Enter the pay item name (e.g., "Overtime", "Commission", "Bonus")
  4. Click Save

Expected Outcome: New pay item appears in the Variable Pays list and becomes available for selection when adding variable pay entries to employees.

Subtask: View Variable Pay Items

Access Settings → Payroll → Variable Pays (expand the accordion) to view all pay items. Each row shows the item name.

Subtask: Delete Variable Pay Item

  1. Locate the pay item in the Variable Pays list
  2. Click the delete icon
  3. Confirm deletion in the popup

Important: Pay items that are currently assigned to employees may have delete restrictions. Ensure the item is not in use before attempting to delete it.

Expected Outcome: Pay item is removed from the system and no longer available for selection.

Task: Add Variable Pay to Employee Payroll

Add one-time or recurring variable compensation to individual employees.

Subtask: Add Variable Pay via Payroll Table

  1. Navigate to Payroll → Payroll Table (see Payroll Management guide)
  2. Locate the employee row (use the currency tabs — AED, USD, AUD — if the employee is on a specific currency)
  3. Click on the employee row to open the detail slide-over panel
  1. In the employee detail panel, locate the Variable Pay section
  2. Click "Add/Edit Variable Pay Item"
  3. In the entry form, fill in the following fields:
    • Pay Items (required) — Select a pay item from the dropdown (these are the items configured in Settings → Payroll → Variable Pays)
    • Associated Period (required) — Select both a Start Date and End Date for the pay period
    • Additional Remarks (optional) — Enter any notes or comments
    • Amount (required) — Enter the compensation amount
  4. Click "Save changes"

Expected Outcome: Variable pay item appears in the employee's payroll record and will be included in the next payroll run.

Note: The Adjustments tab (Payroll → Adjustments) handles Additions and Deductions only — it does not handle Variable Pay items. Variable Pay is managed exclusively through the employee detail panel as described above.

Task: Review Variable Pay History

Access historical records of variable pay items assigned to employees.

Subtask: View Pay Item History

  1. Navigate to Payroll → Payroll Table
  2. Click on the employee row to open the detail slide-over panel
  3. Scroll to the Variable Pay section
  4. Review past variable pay entries displayed in the section

Expected Outcome: Historical variable pay items are displayed with dates, amounts, and categories.

Note: Pay item history may not be accessible when payroll table cells are disabled. This is a known limitation (ticket GS-1382).

Edge Cases

  • Variable pay with deleted leave requests: If a leave request is deleted after being processed, the payroll deduction remains but loses reference to the original request. Document leave details before deletion.
  • Inactive employees with arrears: Arrears persist in the system even after employee inactivation. Plan for manual cleanup or support intervention.
  • Payroll module reset: Resetting the payroll module requires complete data deletion and reconfiguration. Ensure backups exist before requesting a reset.
  • Work expenses in closed cycles: Expenses added to closed payroll cycles cannot be bulk-deleted. Process deletions before closing cycles or contact support.

Data Integrity Checks

  • Check that all variable pay items are correctly configured in Settings → Payroll → Variable Pays
  • Ensure Bayzat Sheets mass upload rows have all required columns filled (Employee Identifier, Pay Item Type, Start Date, End Date, Amount, Reference)

FAQs & Support

Frequently Asked Questions

Can I bulk upload variable pay items for multiple employees?

Yes. Navigate to Settings → Payroll → Mass Uploads → Start to open the Bayzat Sheets spreadsheet interface. Fill in Employee Identifier, Pay Item Type, Start Date, End Date, Amount, and Reference for each row. All required fields must be completed — rows with blank required fields will cause errors.

How do I access pay item history when payroll cells are disabled?

Pay item history is not accessible when payroll table cells are disabled. You must enable the cell to view historical data.

Can I change the payroll cycle dates myself?

Payroll cycle date changes require backend intervention by the support team. Contact support to request a cycle date modification.

What happens to deleted employees in the payroll table?

Deleted employees are not automatically removed from the payroll table. Manual cleanup or support intervention is required to remove them.

Can I delete salary arrears for inactive employees?

Salary arrears cannot be deleted, even for inactive employees. Adjustments can be deleted, but arrears remain in the system.

How do I delete all work expense requests at once?

Bulk deletion of work expenses is not available in the UI. Contact support for backend deletion of historical expense records.

Can I reset the payroll module without support?

Payroll module reset requires support team intervention. There is no self-service option for resetting payroll data and configuration.

Are leave request details retained in payroll after deletion?

Leave request details are not retained in payroll deductions if the original leave request is deleted. Ensure leave requests are processed before deletion if payroll tracking is needed.
