What is variable-pay?

Overview

Variable pay enables payroll administrators to add non-recurring compensation items like bonuses, commissions, allowances, and deductions to employee payroll. These items are processed during payroll runs alongside fixed salary components in the Payroll Management module.

Note: No screenshots are currently available for this feature. This guide documents functionality based on system behavior and known limitations.

Administrators create variable pay items per employee per pay period, specifying amounts and effective dates.

Key Benefits

  • Add bonuses, commissions, and allowances without modifying base salary structures
  • Track one-time payments and deductions separately from recurring compensation
  • Process variable compensation automatically during payroll runs
  • Maintain audit trail of all variable pay entries per employee

Product Foundation Overview

How Variable Pay Fits

Variable pay items are transactional payroll components added to individual employee records for specific pay cycles. Unlike fixed salary structures, these are one-time or recurring additions processed during payroll runs.

Mental model: Variable Pay Item ‚Üí Added to Employee Record ‚Üí Processed in Payroll Cycle ‚Üí Appears in Payslip

Each variable pay item affects only the assigned employee for the specified period. Changes to variable pay do not retroactively affect past payroll cycles.

Key Decisions Before Setup

Answer these questions before adding variable pay items:

  • What type of variable compensation are you adding? ‚Äî Bonuses, commissions, allowances, or deductions
  • Is this a one-time payment or recurring? ‚Äî Determines whether the item appears in multiple pay cycles
  • Which employees receive this compensation? ‚Äî Individual assignment or bulk upload via mass entry
  • When should this appear in payroll? ‚Äî Current cycle, future cycle, or specific date range

Key Tasks

Task: Add Variable Pay to Employee Records

Assign one-time or recurring pay items to individual employees for inclusion in the next payroll run.

Subtask: Navigate to Employee Variable Pay

Payroll ‚Üí Payroll Table ‚Üí Select Employee ‚Üí Variable Pay Column

Access the employee's payroll record and locate the variable pay entry field. Click to open the variable pay assignment interface.

Subtask: Add New Pay Item

Click the add icon to create a new variable pay entry. Select the pay item type from the dropdown (e.g., Bonus, Commission, Allowance). Enter the amount and specify whether the item is one-time or recurring. Save the entry to update the employee's payroll record.

Subtask: Verify Pay Item Assignment

Confirm the pay item appears in the employee's payroll table row under the Variable Pay column. Check that the amount and recurrence settings are correct before the next payroll run.

Task: Bulk Upload Variable Pay Items

Import multiple variable pay assignments across employees using a spreadsheet template.

‚ö†Ô∏è Known Limitation: The mass upload system does not support rows with blank/empty fields for specific pay item types. All rows must have complete data for successful import.

Subtask: Download Template

Payroll ‚Üí Payroll Table ‚Üí Mass Upload ‚Üí Download Template

Download the variable pay mass upload template. The template includes columns for employee ID, pay item type, amount, and recurrence settings.

Subtask: Prepare Upload File

Fill in the template with employee IDs and corresponding pay items. Ensure all required fields are populated for each row. Do not leave blank cells in pay item type or amount columns. Save the file in the required format (CSV or Excel).

Subtask: Upload and Validate

Return to the mass upload interface and select the prepared file. Click Upload and review the validation summary. Address any errors flagged by the system before confirming the import.

Subtask: Confirm Import Results

After successful upload, verify that variable pay items appear in the affected employees' payroll table rows. Spot-check several records to ensure amounts and settings match the uploaded data.

Task: Edit Existing Variable Pay Items

Modify assigned pay items before payroll processing to correct amounts or change recurrence settings.

Subtask: Locate Pay Item

Payroll ‚Üí Payroll Table ‚Üí Select Employee ‚Üí Variable Pay Column

Navigate to the employee's payroll record and click the Variable Pay cell to view assigned items. Identify the pay item requiring modification.

Subtask: Update Pay Item Details

Click the edit icon next to the pay item. Adjust the amount, pay item type, or recurrence setting as needed. Save changes to update the employee's record.

Note: Edits made after payroll processing has started may not affect the current run. Verify the payroll status before making changes.

Task: Delete Variable Pay Items

Remove assigned pay items from employee records before they are processed in payroll.

Subtask: Access Pay Item List

Payroll ‚Üí Payroll Table ‚Üí Select Employee ‚Üí Variable Pay Column

Open the employee's variable pay assignments. Locate the pay item to be removed.

Subtask: Remove Pay Item

Click the delete icon next to the pay item. Confirm the deletion when prompted. The item is immediately removed from the employee's payroll record.

‚ö†Ô∏è Limitation: Variable pay items cannot be deleted after the payroll run has been finalized. Deletion must occur before payroll processing completes.

Task: View Variable Pay History

Review historical variable pay assignments and processed amounts for audit and reconciliation purposes.

Subtask: Access Pay Item History

Payroll ‚Üí Payroll Table ‚Üí Select Employee ‚Üí Variable Pay Column ‚Üí History Icon

Click the history icon in the Variable Pay cell to view past assignments. The history popup displays all pay items processed in previous payroll runs, including amounts and dates.

Known Issue: When payroll table cells are disabled, the history popup may not be accessible. This is a known limitation tracked under ticket GS-1382. Contact support if you need to retrieve historical data during this state.

Subtask: Export History Data

If available, use the export function to download variable pay history for reporting. Exported data includes employee ID, pay item type, amount, and payroll run date.

Task: Configure Variable Pay Items in Payroll Settings

Define the types of variable pay items available for assignment to employees.

Subtask: Access Payroll Configuration

Settings ‚Üí Payroll ‚Üí Variable Pay Items

Navigate to the payroll configuration area where variable pay item types are defined. This is typically done during initial payroll setup.

Subtask: Create New Pay Item Type

Click Add New Pay Item. Enter the pay item name (e.g., Performance Bonus, Sales Commission). Specify whether the item is taxable and how it should be calculated. Save the configuration to make the pay item type available for employee assignment.

Subtask: Edit Pay Item Type

Locate the existing pay item type in the list. Click the edit icon to modify the name, tax treatment, or calculation method. Save changes to update all future assignments of this pay item type.

Subtask: Deactivate Pay Item Type

To prevent future use of a pay item type, click the deactivate or delete icon. Confirm the action. Deactivated pay items cannot be assigned to employees but remain visible in historical records.

Ensure variable pay items are included in the payroll calculation and employee payments.

Initiate the payroll run as documented in the Payroll Management guide. The system automatically includes all assigned variable pay items in the calculation. Review the payroll summary to confirm variable pay amounts are reflected in employee totals.

FAQs & Support

Frequently Asked Questions

Can I bulk upload variable pay items for multiple employees?

Mass upload does not currently support variable pay entries when some rows have blank fields for specific pay item types. Enter variable pay individually through the payroll table or ensure all rows have complete data.

Can I view pay item history when payroll table cells are disabled?

Pay item history popups are not accessible when payroll table cells are disabled. This is a known limitation with no current workaround.

Can I delete salary arrears for inactive employees?

Salary arrears cannot be deleted, even for inactive employees. Adjustments can be deleted, but arrears remain in the system permanently.

What happens if I delete a leave request that was already processed in payroll?

Leave request details are not retained in payroll deductions after the original request is deleted. This may affect payroll tracking and audit trails.

Can I change payroll cycle dates myself?

Payroll cycle date changes require backend intervention by the support team. There is no self-service option, and changes cannot be reversed.

How do I reset the payroll module?

Payroll module resets require manual support team intervention for backend data deletion and reconfiguration. Contact support to request a reset.
