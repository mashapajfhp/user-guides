What is Work Timings?

Overview

Work Timings is the foundational configuration layer within Bayzat's Attendance module that defines when employees are expected to work. It establishes the time boundaries, flexibility parameters, and validation rules that govern daily check-ins, late arrivals, early departures, and overtime calculations across your organization.

The Work Timings configuration screen is accessed through the Attendance settings area. This interface displays all configured work timing schedules in a searchable, paginated table format, allowing administrators to view, create, edit, and delete timing configurations that will be assigned to employees.

The table displays critical configuration details at a glance: schedule name, start and end times, late arrival thresholds (with multiple levels such as "1 level"), early departure allowances, absent day triggers, extra hours calculation methods, and "Absent after" timing rules. Special indicators include a "Flexible timing" badge for schedules with variable start/end times and an overnight shift icon for schedules that span midnight. Each row provides Edit (pencil icon) and Delete (trash icon) actions, while the "Add new" button at the bottom enables creation of new timing configurations. The interface supports searching by name and includes pagination controls showing 16 pages with 78 total items, with options to display 5, 10, 25, or 50 items per page.

Key Benefits

  • Reduce attendance tracking errors by establishing clear, validated time boundaries for employee check-ins and check-outs
  • Eliminate manual schedule management by centralizing all timing configurations in a single, searchable interface
  • Ensure payroll accuracy by defining precise rules for late arrivals, early departures, and overtime calculations
  • Support diverse workforce needs with flexible timing options, overnight shift handling, and half-day configurations
  • Maintain compliance by enforcing absent day thresholds and preventing unauthorized early check-ins
  • Gain visibility into scheduling complexity with multi-level late arrival tracking and configuration overview

Product Foundation Overview

Core Concepts

Work Timing Schedule

A named configuration that defines the expected start time, end time, and associated rules for a work period. Each schedule can be assigned to multiple employees and serves as the reference point for attendance validation and overtime calculations.

Flexible Timing

A toggle that allows employees to check in and out within a variable time range rather than fixed start/end times. When enabled, the system displays a "Flexible timing" badge and adjusts validation rules to accommodate schedule variations.

Late Arrival Threshold

The number of minutes after the scheduled start time that an employee can check in before being marked as late. Supports multi-level configurations (e.g., "1 level") to distinguish between minor delays and significant tardiness.

Early Departure Rule

The number of minutes before the scheduled end time that an employee can check out without penalty. This setting balances operational needs with employee flexibility.

Absent Day Trigger

The threshold (in minutes) that determines when a late check-in or missed check-in results in marking the employee as absent for the day. Example: "12 Minutes" means checking in more than 12 minutes late triggers an absent status.

Extra Hours Calculation

The method used to calculate overtime or additional work hours. Options include "Total hours" (cumulative time worked beyond schedule) or other calculation approaches based on organizational policy.

Overnight Shift

A schedule that spans midnight, indicated by a special icon in the interface. The system automatically detects when end time is earlier than start time and applies overnight shift logic.

Half-Day Timing

A configuration option that defines reduced work hours for half-day schedules, commonly used for Fridays, religious observances, or flexible work arrangements.

Related Features

Work Timings integrates with and supports several other Attendance and HR features:

  • Employee Profiles: Work timing schedules are assigned to individual employees or employee groups within their profile settings
  • Attendance Daily Reports: Daily check-in/check-out records are validated against assigned work timing schedules to determine late arrivals, early departures, and absences
  • Overtime Policies: Extra hours calculated based on work timing schedules feed into overtime policy calculations for payroll
  • Days In Lieu Policies: Work timing configurations determine when compensatory time off is earned for working beyond scheduled hours
  • Deductions Policies for Payroll: Late arrival and early departure thresholds from work timings trigger payroll deductions when policies are configured
  • Work Centers for Shift Scheduling: Work timings can be associated with specific work centers to support location-based shift management
  • Biometric Data Integration: Biometric device check-ins are validated against work timing schedules to automate attendance tracking

Complete User Journey Guide

Complete User Journey Guide

End-to-End Journey: Setting Up and Managing Work Timings

The Work Timings journey enables HR administrators and payroll managers to define, configure, and maintain organizational work schedules. This journey spans from initial configuration through ongoing management and assignment to employees.

1

Discovery & Planning

Before creating work timings, administrators assess organizational scheduling needs. This includes identifying shift types (standard, flexible, overnight), understanding late arrival and early departure policies, and determining break allowances. Administrators review existing attendance policies and compliance requirements to ensure work timing configurations align with labor regulations and company standards.

Decision Point: Will the organization use standard fixed schedules, flexible timing options, or a combination of both?

Expected Outcome: Clear understanding of required work timing configurations, including start/end times, grace periods, and policy enforcement rules.

2

Initial Access & Navigation

Administrators navigate to the Work Timings configuration area within Attendance settings. The feature is accessed through a hierarchical settings structure, where Work Timings appears as one section among multiple attendance-related configurations.

The main Work Timings screen displays an overview table showing all existing configurations. This table provides immediate visibility into key attributes: timing names, start/end times, flexible timing status, overnight shift indicators, late arrival rules, early departure thresholds, absent day criteria, and extra hours calculation methods. Pagination controls allow navigation through multiple pages of configurations (up to 16 pages with 78 total items in production environments).

The table interface enables quick scanning of configurations through visual indicators. Flexible timing configurations display a distinctive badge, overnight shifts show a moon icon, and multi-level late arrival systems are indicated with level counts. Search functionality allows filtering by name, while info icons on column headers provide tooltip explanations for Extra Hours and Absent After calculations.

Expected Outcome: Administrator gains comprehensive view of existing work timing configurations and understands available management actions (Add new, Edit, Delete).

3

Configuration & Setup

When creating a new work timing or modifying an existing one, administrators interact with a comprehensive configuration form. This form presents all scheduling parameters in a structured modal dialog, ensuring focused attention on configuration details without navigating away from the main table view.

The configuration process involves several interconnected decisions:

  • Basic Identification: Naming the work timing configuration for easy reference and employee assignment
  • Schedule Definition: Setting start and end times, with default values of 09:00 AM to 06:00 PM for new configurations
  • Flexibility Options: Enabling flexible timing toggle for schedules that allow variable start/end times within defined boundaries
  • Half-Day Settings: Configuring half-day timing options when applicable
  • Attendance Policies: Defining late arrival thresholds, early departure rules, and absent day criteria (default: 12 minutes threshold)
  • Operational Controls: Allowing breaks, permitting out-of-office check-ins, and restricting early check-ins before work start time
  • Extra Hours Calculation: Selecting calculation method (Total hours or other available options)

The form provides immediate visual feedback on configuration choices. Toggles clearly indicate enabled/disabled states, time pickers enforce valid time ranges, and policy threshold fields accept numeric inputs with appropriate validation. The Absent day option is enabled by default, reflecting best practice for attendance tracking.

Decision Point: Should this configuration enforce strict timing (disallow check-in before work start time) or allow flexibility for early arrivals?

Expected Outcome: Complete work timing configuration saved with all policy parameters defined, ready for employee assignment through shift scheduling.

4

Review & Verification

After creating or updating work timings, administrators verify configurations by reviewing the updated table view. The system immediately reflects changes, displaying new configurations or updated values in the main table. Administrators can click the Edit button (pencil icon) to open the detail view, which serves as both a read-only review interface and an update form.

The edit form displays all previously configured values, allowing administrators to verify accuracy and make adjustments if needed. For example, a configuration named "1 hour shift" would show its specific start/end times, absent day threshold (12 Minutes), and extra hours calculation method (Total hours selected). All fields remain editable, with an Update button replacing the Save button to indicate modification mode.

Expected Outcome: Administrator confirms work timing configuration accuracy and identifies any adjustments needed before employee assignment.

5

Ongoing Management & Maintenance

Work timing configurations require periodic review and updates as organizational needs evolve. Administrators may need to adjust policy thresholds, modify time ranges for seasonal changes, or create new configurations for special projects or temporary shifts.

The management interface supports three primary ongoing actions:

  • Editing: Updating existing configurations through the edit form, with changes taking effect immediately upon clicking Update
  • Searching: Using the search textbox to quickly locate specific configurations by name in environments with numerous work timings
  • Deleting: Removing obsolete configurations with appropriate safety confirmations

When deleting a work timing, the system presents a critical warning to prevent unintended consequences. The confirmation dialog explicitly states: "Are you sure you want to delete this work timing? Employees assigned to this work timing for the future will not be able to check-in." This warning ensures administrators understand the downstream impact on employee attendance tracking.

The confirmation dialog provides Cancel and Delete buttons, allowing administrators to abort the deletion if they recognize potential disruption to employee schedules. This safety mechanism prevents accidental removal of actively used configurations.

Decision Point: Before deleting, has the administrator verified that no employees are currently assigned to this work timing for future dates?

Expected Outcome: Work timing configurations remain current, accurate, and aligned with organizational scheduling needs. Obsolete configurations are safely removed without disrupting active employee assignments.

6

Integration with Employee Scheduling

While work timing configuration is complete, the journey extends to employee assignment through shift scheduling. Administrators assign work timings to employees based on their roles, departments, or project requirements. These assignments determine which attendance rules apply to each employee's check-in/check-out activities.

The system enforces work timing rules during employee attendance tracking. When an employee checks in, the system validates against their assigned work timing: late arrival thresholds trigger notifications, early departures are flagged, and absent day criteria determine when an employee is marked absent. Extra hours calculations apply automatically based on the configured method.

Expected Outcome: Employees are assigned appropriate work timings, and attendance tracking operates according to configured rules, ensuring consistent policy enforcement across the organization.

Setup Process

Setup Process

First-Time Configuration Overview

Setting up Work Timings for the first time involves defining the organizational work schedules that will govern employee attendance tracking. This setup process establishes the foundation for accurate time tracking, policy enforcement, and payroll integration. Administrators should complete this configuration before assigning employees to shifts or enabling attendance tracking features.

The initial setup typically progresses through three phases: planning organizational schedules, creating core work timing configurations, and verifying settings before employee assignment. Each phase builds on the previous one, ensuring comprehensive coverage of organizational scheduling needs.

Phase 1: Planning Your Work Timing Structure

Before creating any configurations, administrators should document their organization's scheduling requirements. This planning phase prevents the need for extensive rework and ensures consistent policy application across the organization.

Identify Required Schedule Types

Consider the various work patterns in your organization:

  • Standard Office Hours: Fixed schedules with consistent start and end times (e.g., 9:00 AM - 6:00 PM)
  • Shift Work: Multiple schedules covering different time periods (morning, afternoon, night shifts)
  • Flexible Timing: Schedules allowing variable start/end times within defined boundaries
  • Overnight Shifts: Schedules spanning midnight, requiring special handling for date transitions
  • Part-Time Schedules: Reduced hour configurations for part-time employees
  • Half-Day Schedules: Configurations for employees working partial days

Define Policy Parameters

For each schedule type, determine:

  • Grace Periods: How many minutes late is acceptable before marking an employee as late?
  • Early Departure Threshold: When should leaving early trigger a policy violation?
  • Absent Day Criteria: After how many minutes of tardiness should an employee be marked absent? (Default: 12 minutes)
  • Break Allowances: Should employees be allowed to check out for breaks during the work day?
  • Extra Hours Calculation: How should overtime be calculated? (Total hours or alternative methods)
  • Check-In Restrictions: Should employees be prevented from checking in before their scheduled start time?
  • Out-of-Office Check-Ins: Should employees be allowed to check in from locations outside designated work centers?

Review Compliance Requirements

Ensure planned configurations comply with:

  • Local labor laws regarding work hours and overtime
  • Industry regulations for shift work and rest periods
  • Company policies documented in employee handbooks
  • Union agreements or collective bargaining arrangements (if applicable)

Phase 2: Creating Your First Work Timing

With planning complete, administrators can create their first work timing configuration. This process demonstrates the full configuration workflow and establishes a template for additional schedules.

Step 1: Access the Configuration Interface

Navigate to the Work Timings section within Attendance settings. The main table view displays any existing configurations (which may be empty for new implementations). Locate the Add new button at the bottom of the table‚Äîthis is the primary trigger for creating new work timings.

The table provides context for your new configuration by showing existing schedules. Review any pre-configured timings to understand naming conventions and policy patterns already in use. The search functionality and pagination controls help navigate existing configurations in environments with multiple schedules.

Step 2: Open the Creation Form

Click the Add new button to open the work timing creation form. The system displays a modal dialog containing all configuration options, keeping the main table visible in the background for reference.

The creation form presents a structured interface with logical grouping of related settings. This organization guides administrators through the configuration process, ensuring no critical settings are overlooked.

The form opens with sensible default values designed for standard office schedules: start time of 09:00 AM, end time of 06:00 PM, and the Absent day option enabled. These defaults provide a starting point that administrators can adjust based on their specific requirements.

Step 3: Configure Basic Settings

Begin by defining the fundamental schedule parameters:

  1. Name: Enter a descriptive name that clearly identifies the schedule's purpose and target employees. Examples: "Standard Office Hours", "Night Shift", "Flexible 8-Hour Schedule", "Part-Time Morning Shift"
  2. Start Time: Set the scheduled work start time using the time picker. Click the field to reveal hour and minute selectors, or type the time directly if supported
  3. End Time: Set the scheduled work end time. Ensure the end time is later than the start time unless configuring an overnight shift
  4. Flexible Timing Toggle: Enable this option if the schedule allows variable start/end times within defined boundaries. When enabled, a "Flexible timing" badge appears on the configuration in the main table
  5. Half-Day Toggle: Enable if this configuration represents a half-day schedule (typically 4 hours or as defined by company policy)

Step 4: Define Attendance Policies

Configure the policy enforcement rules that govern attendance tracking:

  1. Late Arrival: Specify the grace period in minutes before an employee is marked as late. Common values: 5, 10, or 15 minutes. Some organizations use multi-level systems (e.g., "1 level" indicating a single threshold)
  2. Early Departure: Define the threshold for leaving early. Set the number of minutes before the scheduled end time that triggers an early departure flag
  3. Absent After: Set the number of minutes of tardiness after which an employee is marked absent rather than late. The default value is 12 minutes. This threshold should align with company attendance policies and labor regulations
  4. Extra Hours Calculation: Select the method for calculating overtime or extra hours. The "Total hours" option is typically selected, calculating extra hours based on total time worked beyond the scheduled end time

Step 5: Configure Operational Controls

Set the operational rules that govern day-to-day attendance tracking:

  1. Allow Breaks: Enable this option if employees are permitted to check out for breaks during the work day and check back in afterward. This is essential for organizations tracking break compliance
  2. Allow Out of Office Check-Ins: Enable to permit employees to check in from locations outside designated work centers. Disable for roles requiring physical presence at specific locations
  3. Disallow Check-In Before Work Start Time: Enable this restriction to prevent employees from checking in early. This is useful for controlling overtime costs or ensuring security protocols are followed

Step 6: Save and Verify

After configuring all settings, review the form for accuracy. Verify that:

  • The name clearly identifies the schedule
  • Start and end times are correct and logical
  • Policy thresholds align with company policies
  • Operational controls match intended usage patterns

Click the Save button at the bottom of the form to create the work timing. The system validates the configuration, saves it to the database, and closes the modal dialog. The main table refreshes automatically, displaying the new configuration with all its attributes visible in the table columns.

Task: Configure Work Timing Schedules

Administrators need to define and manage work timing configurations that control employee attendance tracking, including start times, end times, flexible timing options, and attendance calculation rules. Subtask: Create a New Work Timing Configuration

Creating a work timing configuration establishes the foundational schedule parameters that will be assigned to employees. This includes defining core working hours, flexibility options, and attendance calculation rules.

To create a new work timing, navigate to the Work Timings section within Attendance settings. The interface displays a table listing all existing configurations with columns showing Name, Start Time, End Time, Extra Hours calculation method, Late Arrival tolerance, Early Departure threshold, Absent After duration, and Half-Day timing. At the bottom of this table, you'll find the primary action button to initiate configuration creation.

When you click the "Add new" button, a modal dialog opens presenting the work timing creation form. The form contains all configuration fields required to define a complete work schedule. Key sections include the timing name field at the top, a Flexible timing toggle switch, Start time and End time selectors (defaulting to 09:00 AM and 06:00 PM), and a Half-day toggle. Below these core fields, you'll find attendance calculation parameters: Late arrival tolerance, Early departure threshold, and Absent day threshold (enabled by default). Additional options control break allowances, out-of-office check-ins, and check-in restrictions before work start time. At the bottom, the Extra Hours calculation method dropdown allows selection between calculation approaches.

After entering the configuration details, clicking the "Save" button creates the work timing record. The system validates all required fields and time logic (ensuring end time is after start time). Once saved, the new configuration appears in the work timings table and becomes available for assignment to employees. The configuration immediately affects how attendance is tracked for any employees assigned to this timing, including check-in/check-out validation, late arrival detection, and extra hours calculation.

Subtask: View Work Timing Configuration Details

Viewing existing work timing details allows administrators to review all parameters of a configuration without making changes. This is essential for verification, auditing, and understanding how attendance is being calculated for employees assigned to specific timings.

From the work timings table, locate the configuration you want to review. Each row in the table displays summary information including the timing name, start and end times, and key thresholds. Special indicators appear for configurations with flexible timing (shown as a badge) or overnight shifts (indicated by an icon). The table supports searching by name using the search textbox at the top, and pagination controls allow navigation through multiple pages of configurations.

To view full details, click the Edit button (pencil icon) in the actions column for the desired configuration. This opens the same modal dialog used for editing, but serves as a detailed view of all settings. The form displays the complete configuration including the timing name, flexible timing status, precise start and end times, half-day settings, late arrival tolerance, early departure threshold, absent day threshold (e.g., "12 Minutes"), break allowances, check-in restrictions, and the selected extra hours calculation method (e.g., "Total hours").

The detail view provides complete visibility into how attendance will be calculated for employees on this timing. Administrators can verify that thresholds align with company policy, confirm that flexible timing is enabled or disabled as intended, and understand the extra hours calculation approach. This view does not require any changes to be made; you can simply review the information and close the dialog. The configuration remains unchanged and continues to govern attendance tracking for assigned employees.

Subtask: Update an Existing Work Timing Configuration

Updating work timing configurations is necessary when business requirements change, such as adjusting working hours, modifying attendance thresholds, or enabling flexible timing options. Updates affect how attendance is calculated going forward for all employees assigned to the timing.

To update a configuration, locate it in the work timings table and click the Edit button (pencil icon) in the actions column. The edit form opens as a modal dialog, pre-populated with all current values. All fields are editable, allowing you to modify the timing name, toggle flexible timing on or off, adjust start and end times using the time selectors, enable or disable half-day timing, and change attendance calculation parameters including late arrival tolerance, early departure threshold, and absent day threshold. You can also modify break allowances, out-of-office check-in permissions, check-in time restrictions, and the extra hours calculation method.

After making the necessary changes, click the "Update" button at the bottom of the form. The system validates the modified configuration, ensuring time logic is sound and required fields are completed. Once updated, the changes take effect immediately for future attendance tracking. Employees assigned to this timing will have their attendance calculated using the new parameters starting from the next check-in event. Historical attendance records remain unchanged; only future attendance events use the updated configuration. If the timing is assigned to employees for future dates, those assignments will use the updated parameters when those dates arrive.

Subtask: Delete a Work Timing Configuration

Deleting work timing configurations is necessary when a schedule is no longer used or was created in error. This action requires careful consideration as it affects employees assigned to the timing for future dates.

To delete a configuration, locate it in the work timings table and click the Delete button (trash icon) in the actions column. The system immediately displays a confirmation dialog to prevent accidental deletion. This dialog contains a warning message explaining the impact of deletion: "Are you sure you want to delete this work timing? Employees assigned to this work timing for the future will not be able to check-in." The dialog provides two options: "Cancel" to abort the deletion, or "Delete" to proceed with removing the configuration.

If you click "Delete", the system permanently removes the work timing configuration from the platform. The configuration immediately disappears from the work timings table and is no longer available for assignment to employees. Critically, any employees who were assigned to this timing for future dates will lose their work timing assignment, preventing them from checking in until a new timing is assigned. Historical attendance records for employees who previously used this timing remain intact and unchanged. The deletion cannot be undone; if the configuration is needed again, it must be recreated manually with all parameters re-entered. Before deleting, administrators should verify that no active employees are currently assigned to the timing or reassign them to alternative configurations.

Task: Manage Multi-Level Attendance Policies

Work timing configurations support sophisticated attendance policy enforcement through multi-level late arrival systems, early departure thresholds, and absent day calculations. Administrators must configure these parameters to align with organizational policies and ensure accurate attendance tracking.

Subtask: Configure Late Arrival Tolerance Levels

Late arrival tolerance defines how much time employees can arrive after the scheduled start time before being marked late. The system supports multi-level configurations (e.g., "1 level") that can apply progressive consequences or tracking for different degrees of lateness.

Within the work timing creation or edit form, locate the "Late arrival" field. This field accepts time duration inputs that define the grace period before an employee is considered late. For example, setting this to "15 minutes" means employees can check in up to 15 minutes after the scheduled start time without penalty. The multi-level system visible in the work timings table (showing "1 level" for some configurations) indicates that the platform supports progressive late arrival tracking, though the specific level configuration may be managed through related attendance policy settings.

When an employee checks in after the start time plus the late arrival tolerance, the system marks the attendance event as late. This late marking feeds into attendance reports, payroll deduction calculations, and compliance tracking. The tolerance setting balances operational flexibility with policy enforcement, allowing reasonable grace periods while maintaining accountability. Changes to late arrival tolerance affect future check-ins immediately; employees checking in after the update will be evaluated against the new threshold.

Subtask: Define Early Departure Thresholds

Early departure thresholds determine when an employee leaving before the scheduled end time is flagged as departing early. This setting works in conjunction with extra hours calculation to ensure accurate tracking of actual working time.

In the work timing form, the "Early departure" field allows you to specify the duration before the scheduled end time that triggers an early departure flag. For instance, setting this to "30 minutes" means that if an employee checks out more than 30 minutes before the scheduled end time, the system records an early departure. This threshold helps distinguish between minor variations in departure time and significant early exits that may require policy intervention or payroll adjustment.

The early departure setting interacts with the extra hours calculation method. If extra hours are calculated based on "Total hours", early departures reduce the total hours worked and may result in negative extra hours or deductions. The threshold provides a buffer that accommodates normal end-of-day variations while capturing genuinely early exits. Employees who depart within the threshold of the scheduled end time are not flagged, maintaining morale and flexibility while preserving policy enforcement for significant deviations.

Subtask: Set Absent Day Calculation Rules

The absent day threshold defines the minimum duration an employee must be present to avoid being marked absent for the entire day. This is critical for payroll accuracy and attendance compliance, as it determines when partial attendance is insufficient to count as a working day.

Within the work timing form, the "Absent day" field is enabled by default and accepts a time duration input. For example, setting this to "12 Minutes" (as shown in the validation evidence) means that if an employee's total checked-in time is less than 12 minutes, they are marked as absent for the entire day. This threshold prevents situations where an accidental or brief check-in incorrectly registers as a working day. The default enablement ensures that all work timing configurations have absent day protection unless explicitly disabled.

When an employee's total attendance duration falls below this threshold, the system marks them as absent in attendance reports and payroll calculations. This absent marking triggers any configured deduction policies and affects leave balances if applicable. The threshold must be set thoughtfully: too high and legitimate partial attendance may be marked absent; too low and the protection against accidental check-ins is weakened. Changes to the absent day threshold take effect immediately for future attendance calculations, ensuring that the next day's attendance is evaluated against the updated rule.

Task: Configure Flexible and Overnight Work Schedules

Modern work environments require support for flexible timing arrangements and overnight shifts. Work timing configurations provide toggles and indicators to handle these non-standard scheduling scenarios while maintaining accurate attendance tracking.

Subtask: Enable Flexible Work Timing

Flexible work timing allows employees to check in and out within a broader time window rather than adhering to strict start and end times. This accommodates modern work arrangements while still tracking attendance and calculating working hours.

In the work timing creation or edit form, locate the "Flexible timing" toggle switch near the top of the form. Enabling this toggle changes how the system interprets the configured start and end times. Instead of treating these as rigid boundaries, the system uses them as reference points for calculating expected working hours while allowing employees to check in and out at variable times. Configurations with flexible timing enabled display a "Flexible timing" badge in the work timings table, making them easily identifiable.

When flexible timing is enabled, the late arrival and early departure thresholds function differently. Rather than marking employees late for arriving after a specific time, the system focuses on total hours worked and adherence to the expected duration. The extra hours calculation method becomes particularly important in flexible arrangements, as it determines whether employees are credited for working beyond the expected duration regardless of when they start and finish. Flexible timing configurations are ideal for roles with variable schedules, remote workers, or organizations adopting flexible work policies. Employees assigned to flexible timings experience less rigid attendance enforcement while the organization maintains visibility into total working hours and productivity.

Subtask: Configure Overnight Shift Schedules

Overnight shifts span across midnight, requiring special handling to ensure attendance tracking correctly attributes working hours to the appropriate day and calculates durations accurately when the end time is earlier (in clock terms) than the start time.

When creating or editing a work timing, configure the start and end times such that the end time occurs on the following calendar day. For example, setting start time to "11:00 PM" and end time to "07:00 AM" creates an overnight shift. The system automatically detects this configuration and displays an overnight shift indicator icon in the work timings table, helping administrators quickly identify schedules that span midnight.

The overnight shift detection ensures that attendance calculations correctly compute working hours across the day boundary. When an employee checks in at 11:00 PM and checks out at 07:00 AM the next morning, the system calculates 8 hours of work rather than a negative duration. This automatic handling prevents common errors in overnight shift tracking. The late arrival, early departure, and absent day thresholds function normally within the overnight context, with the system understanding that the shift's "day" is defined by the start time. Payroll integration correctly attributes the working hours to the shift start date, ensuring accurate pay period calculations and compliance reporting.

Subtask: Configure Half-Day Timing Options

Half-day timing configurations allow organizations to define specific schedules for partial work days, such as during Ramadan, training days, or reduced-hour arrangements. This provides flexibility while maintaining structured attendance tracking.

In the work timing form, locate the "Half-day" toggle switch below the start and end time fields. Enabling this toggle designates the configuration as a half-day schedule. The system uses this designation to adjust attendance calculations, payroll integration, and reporting to reflect that the timing represents a partial work day rather than a full shift. Half-day configurations appear in the work timings table with their reduced duration clearly visible in the Start Time and End Time columns.

When employees are assigned to a half-day timing, the system adjusts expected working hours accordingly. This affects extra hours calculations (employees are not penalized for working fewer hours than a full-day timing) and ensures that payroll processes correctly calculate compensation for partial days. Half-day timings are particularly useful for organizations with regular partial work days or special circumstances requiring reduced schedules. The configuration can be combined with other settings like flexible timing or overnight shifts, providing comprehensive coverage of diverse scheduling needs. Administrators can create multiple half-day configurations for different scenarios (morning half-day, afternoon half-day, etc.) and assign them to employees as needed.

Task: Configure Advanced Attendance Controls

Work timing configurations include advanced controls that govern employee check-in behavior, break allowances, and extra hours calculation methods. These settings provide granular control over attendance policy enforcement and ensure accurate tracking of working time.

Subtask: Manage Break and Out-of-Office Check-In Allowances

Organizations need to control whether employees can take breaks during shifts and whether check-ins from outside designated work locations are permitted. These settings balance flexibility with policy compliance and location-based attendance requirements.

Within the work timing form, locate the "Allow breaks" and "Allow out of office check-ins" checkboxes. The "Allow breaks" option, when enabled, permits employees to check out temporarily during their shift and check back in, with the system tracking multiple check-in/check-out pairs throughout the day. This is essential for organizations that require break tracking or have policies around paid versus unpaid breaks. The "Allow out of office check-ins" option controls whether employees can check in from locations outside defined work centers or geofenced areas.

When breaks are allowed, the system calculates total working hours by summing all check-in periods and subtracting break durations. This ensures accurate payroll calculation and compliance with labor regulations regarding break times. When out-of-office check-ins are allowed, employees can check in from any location, which is useful for remote workers, field staff, or flexible work arrangements. Disabling this option enforces location-based attendance, requiring employees to be at designated work centers to check in. These settings interact with biometric devices and work center configurations; if out-of-office check-ins are disabled, employees attempting to check in from unauthorized locations will receive an error. The combination of break and location controls provides comprehensive attendance policy enforcement tailored to organizational needs.

Subtask: Configure Check-In Time Restrictions

Preventing early check-ins helps organizations enforce shift schedules, prevent time theft, and ensure that employees do not begin work before authorized. The check-in time restriction setting provides this control at the work timing level.

In the work timing form, locate the "Disallow check-in before work start time" checkbox. When enabled, this setting prevents employees from checking in before the configured start time. For example, if the start time is set to 09:00 AM and this restriction is enabled, employees attempting to check in at 08:45 AM will receive an error message, and the check-in will not be recorded. This ensures that working hours are tracked only during authorized periods and prevents situations where employees accumulate extra hours by arriving significantly early.

The restriction interacts with the late arrival tolerance setting. If late arrival tolerance is set to 15 minutes, employees can check in from 09:00 AM to 09:15 AM without being marked late, but they still cannot check in before 09:00 AM if the restriction is enabled. This creates a defined check-in window that balances flexibility with control. The setting is particularly useful for shift-based operations, security-sensitive environments, or organizations with strict labor cost management requirements. Disabling the restriction allows employees to check in at any time before their shift, which may be appropriate for flexible work arrangements or roles where early arrival is encouraged. The check-in restriction applies consistently across all check-in methods, including mobile app, biometric devices, and web-based check-ins.

Subtask: Select Extra Hours Calculation Method

The extra hours calculation method determines how the system computes overtime or additional working time beyond the scheduled shift. This setting is critical for payroll accuracy and ensuring employees are compensated correctly for extra work.

At the bottom of the work timing form, locate the "Extra Hours" dropdown menu. This field offers multiple calculation methods, with "Total hours" being one documented option. The selected method defines the algorithm used to calculate extra hours. For example, "Total hours" may calculate extra hours as the total time worked minus the scheduled shift duration, crediting employees for all time beyond the expected hours. Other methods might calculate extra hours differently, such as only counting time beyond a daily threshold or applying different rates for different periods.

The calculation method interacts with all other attendance parameters. If an employee checks in late but works beyond the scheduled end time, the extra hours calculation determines whether the late arrival is offset against extra hours or treated separately. The method also affects how early departures are handled; some methods may reduce extra hours for early exits, while others may calculate them independently. The work timings table displays the selected calculation method in the "Extra Hours" column, allowing administrators to quickly verify which method is applied to each configuration. Choosing the appropriate calculation method requires understanding organizational overtime policies, labor regulations, and payroll system integration requirements. Changes to the calculation method take effect immediately for future attendance events, ensuring that extra hours are calculated consistently according to the current policy.

  • Employee shift is created: Triggers when a new shift is assigned to an employee
  • Employees are marked as absent for past day: Triggers when bulk absence marking occurs for historical dates

These triggers operate on shift assignments and attendance events, which are downstream from work timing configurations. Work timings define the rules and parameters, while shifts and attendance events are the actual instances where those rules are applied.

The workflow system was searched for work timing-specific triggers using the event search functionality, with no results found for "work timing" as a triggerable event. This confirms that work timing configuration management is a setup activity rather than an operational event suitable for workflow automation.

Frequently Asked Questions

How many work timing configurations can I create?

The system supports multiple work timing configurations (current production environment shows 78 total items across 16 pages). There is no documented hard limit. Create as many configurations as needed to support your organizational structure, shifts, and policies. Use clear naming conventions to maintain organization as your configuration library grows.

Can I assign different work timings to the same employee for different time periods?

Yes. Work timings can be assigned to employees with future effective dates, supporting advance scheduling and shift rotations. When assigning a new work timing, specify the effective date. The system will use the appropriate work timing based on the date of attendance. This enables seasonal schedules, temporary assignments, and planned shift changes.

What happens to attendance data if I modify a work timing?

Modifying a work timing affects how future attendance is calculated but does not retroactively change historical attendance records. Past attendance remains calculated according to the work timing rules that were active at the time. This preserves audit trail integrity. If you need to recalculate past attendance, you must do so manually or contact support.

How does flexible timing affect attendance violations?

When flexible timing is enabled, the system adjusts how late arrival and early departure violations are calculated. Flexible timing provides a grace period or tolerance range for check-in times. The specific tolerance depends on your configuration. Employees assigned to flexible work timings will see the 'Flexible timing' badge, indicating relaxed punctuality rules apply to their schedule.

Can I recover a deleted work timing?

No. Work timing deletion is permanent and cannot be undone. This is why the system displays a confirmation dialog with explicit warning about impact on future employee check-ins. Before deleting, document the configuration details or take a screenshot. If you accidentally delete a work timing, you must recreate it manually with the same settings and reassign affected employees.

Why don't work timing changes trigger workflows?

Work timings are configuration settings, not operational events. Bayzat Workflows are designed to automate responses to employee actions and shift events (shift created, updated, deleted, absence marked). Configuration changes are administrative actions that require human decision-making. However, the attendance events that result from work timing rules (such as marking employees absent) can trigger workflows.

How do I handle employees who work split shifts?

Split shifts (working two separate periods in one day with a long break in between) are not directly supported by a single work timing configuration. Workarounds include: (1) Create a work timing with 'Allow breaks' enabled and configure break duration to match the split; (2) Use Multiple Visits settings in Attendance module to allow multiple check-in/check-out pairs per day; (3) Assign two separate shifts per day using shift scheduling. Consult with support to determine best approach for your use case.

What's the difference between 'Absent day' threshold and late arrival levels?

'Absent day' threshold (default 12 minutes) determines when an employee is marked entirely absent for the day - they checked in too late to receive any attendance credit. Late arrival levels track tardiness that doesn't result in full absence - the employee is present but late. You can configure multiple late arrival levels (e.g., '1 level') to create graduated consequences. Both settings work together to enforce punctuality policies.

Can I set different work timings for different days of the week?

Work timings define a single daily schedule (start time, end time, rules). To implement different schedules for different days (e.g., shorter Friday hours), you must create separate work timing configurations and use shift scheduling to assign the appropriate work timing to each day. Alternatively, use the Half-day toggle for days with reduced hours, though this applies the same reduced schedule whenever the work timing is assigned.
