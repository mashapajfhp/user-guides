üè† Leave Salary

Tagline: Automate paid leave compensation with flexible calculation policies

Key Benefits:

  • Automatically calculate and process leave salary based on customizable policies

  • Support multiple salary components including basic salary and allowances

  • Streamline approval workflows with multi-stage request management

  • Generate comprehensive reports for payroll reconciliation and compliance

üìã Feature Overview

What Leave Salary Does

Leave Salary enables organizations to compensate employees for approved paid leave periods by calculating salary amounts based on configurable policies. The feature automates the entire lifecycle from policy configuration through payroll processing.

Who Uses This Feature

Primary Users:

  • Super Admins: Configure leave salary policies and system-wide settings

  • Payroll Managers: Process leave salary transactions and manage payroll integration

  • Leave Managers: Review and approve leave salary requests

  • HR Administrators: Monitor leave salary accruals and generate reports

    • Key Capabilities


  • Policy Configuration: Create flexible leave salary policies with customizable calculation formulas

  • Request Management: Track leave salary requests through pending, approved, rejected, and processed stages

  • Calculation Engine: Automatically compute leave salary based on salary components, working days, and leave duration

  • Payroll Integration: Seamlessly add approved leave salary amounts to payroll tables

  • Reporting: Generate accrual reports and leave salary breakdowns for audit and compliance

Business Value

Leave Salary reduces manual calculation errors, ensures consistent policy application across the organization, and provides full transparency into leave-related compensation. The feature supports compliance with regional labor laws while giving HR teams the flexibility to configure policies that match their organizational needs.

üìù Core Tasks and Procedures

Configuration Tasks

Task 1: Enable Leave Salary for a Leave Policy

Prerequisites:

  • Super Admin or Payroll Manager role

  • Existing leave policy configured

    • Steps:

  • Navigate to Settings > Leave > Leave Policies

  • Locate and click on the leave policy you want to enable for leave salary

  • In the policy settings, toggle Leave Salary Settings to ON

  • Click Save to apply changes

    • Result: The leave policy is now eligible for leave salary configuration.

    ---

    Task 2: Create a Leave Salary Policy

    Prerequisites:

  • Leave salary enabled for at least one leave policy

  • Access to Payroll Settings

    • Steps:

    Step 1: Setup Policy

  • Navigate to Settings > Payroll > Leave salary policy

  • Click Add new button

  • The policy creation wizard opens with 3 steps

    • !Policy Creation Form

    Policy Details Section:

  • Enter Policy Name (e.g., "Annual Leave Salary Policy")

  • Add Policy Description to help employees understand conditions

  • Select Leave Types from dropdown (Required field)

  • Only leave types with leave salary enabled will appear

    • !Leave Types Dropdown

    Leave Salary Formula Section:

    The formula displayed is: (Salary component / Working days) * number of leaves

  • Select Salary Component:

  • Basic salary: Uses only basic salary in calculation

  • Basic salary + allowances: Includes selected allowances

    • !Salary Component Dropdown

  • If "Basic salary + allowances" selected:

  • Click Allowances dropdown

  • Select applicable allowances (multiple selection supported)

  • Available allowances include: Housing, Transportation, Food, Car Allowance, etc.

    • !Allowances Multiselect

  • Select Working Days calculation method:

  • Calendar days: Uses total days in month (30 or 31)

  • Working days: Uses actual working days per month

  • Custom days: Specify a fixed number of days

    • !Working Days Dropdown

  • If "Custom days" selected:

  • Enter the number of days in the Days field

  • Warning: "Total net pay for the leave month may differ when using this setting"

    • !Custom Days Field

    Optional: Define Minimum Leave Days

  • Toggle Define minimum number of leave days required for leave salary

  • If enabled, enter Minimum days value

    • !Minimum Days Field

    Policy Restrictions Section:

  • Toggle Restrict during probation if needed

  • When enabled: Employees cannot apply for leave salary during probation period

  • Click Next to proceed to Step 2

    • Step 2: Add Employees

  • Select employees to include in this policy

  • Use search or filters to find employees

  • Select individual employees or bulk select

  • Click Next to proceed to Step 3

    • Important: Employees can only be assigned to one leave salary policy at a time.

    Step 3: Review & Save

  • Review all policy settings

  • Verify employee list

  • Click Save to activate the policy

    • Result: Policy becomes active immediately and applies to future leave requests.

    Task 14: Resolve "No Eligible Leave" Error

    Problem: Employee cannot create leave salary request due to "No approved paid leave" error.

    Steps:

  • Verify employee is assigned to a leave salary policy:

  • Go to Settings > Payroll > Leave salary policy

  • Check if employee appears in policy's employee list

  • Check leave policy configuration:

  • Go to Settings > Leave > Leave Policies

  • Verify leave salary is enabled for the employee's leave policy

  • Verify employee has approved leave:

  • Navigate to employee profile > Leave tab

  • Check policy assignment date:

  • Employee must be assigned to policy BEFORE leave request creation

  • If assigned after, delete and recreate the leave request

    Task 15: Handle Cross-Cycle Leave Salary Requests

    Problem: Leave request spans multiple payroll cycles.

    Limitation: System cannot process leave salary requests with start date before current active payroll cycle.

    Workaround:

  • Split the leave request into two separate requests:

  • Request 1: Covers days in first payroll cycle

  • Request 2: Covers days in second payroll cycle

  • Process each request separately:

  • Approve Request 1 for first cycle

  • Add to payroll for first cycle

  • Approve Request 2 for second cycle

  • Add to payroll for second cycle

    • Note: This is a known system limitation documented in ticket TSSD-4887.

    Task 16: Move Request Back to Pending

    Problem: Need to modify an approved leave salary request.

    Steps:

  • Navigate to Payroll > Leave Salary

  • Go to Approved tab

  • Select the request

  • Click Move to Pending action

  • Request returns to Pending tab

  • Make necessary modifications

  • Re-approve the request

Note: This is only possible if the request has not been added to payroll table yet.

üìú Business Rules & Limitations

Business Rules

Policy Assignment Rules


  • Policy Eligibility Timing: Leave salary eligibility is determined at the time of leave request creation based on policy assignment date. Employees must be assigned to a leave salary policy before creating their leave request.

  • Single Policy Assignment: Employees can be added to only one leave salary policy at a time. Multiple policy assignments are not supported.

  • Probation Restrictions: Organizations can configure policies to restrict leave salary during probation periods. This setting prevents probationary employees from accessing leave salary benefits.

  • Policy Activation: Leave salary policies become active from the date of activation. Past leave requests are not affected by new policy activation.

    • Leave Salary Request Generation


  • Automatic Request Creation: Leave salary requests are automatically generated only for employees eligible under a specific leave salary policy when they submit approved paid leave requests.

  • Payroll Cycle Constraints: Leave salary requests cannot have a starting date before the current active payroll cycle's start date. This prevents processing of requests that fall into closed payroll periods.

  • Cross-Cycle Leave Handling: When a leave request spans multiple payroll cycles, the system generates separate leave salary requests for each cycle that must be approved independently.

  • Multiple Leave Cycles: For employees on individual leave cycles, leave requests spanning two different cycles result in two separate leave salary approval requests that require independent processing.

    • Calculation Rules


  • Salary Components: Leave salary can be calculated based on:

  • Basic salary only

  • Basic salary + selected allowances

  • Working Days Configuration: The system supports three working day calculation methods:

  • Calendar days (30 days)

  • Working days (actual working days in the month)

  • Custom days (user-defined number)

  • Daily Wage Variations: When using 'working days' setting, the daily wage rate changes based on the number of working days in each month, leading to monthly calculation variations.

  • Custom Days Warning: When 'Custom days' is selected for working days, a warning displays: "Total net pay for the leave month may differ when using this setting"

  • Minimum Days Requirement: Policies can define a minimum number of leave days required for leave salary eligibility.

    • Approval and Processing Rules


  • Amount Modification: Once a leave salary amount is manually overwritten and approved, it cannot be changed unless moved back to pending state.

  • Request States: Leave salary requests move through defined stages:

  • Pending: Awaiting approval

  • Approved: Approved but not yet added to payroll

  • Rejected: Request denied

  • At Payroll: Added to payroll table

  • Processed: Completed and paid

  • Multi-Month Processing: Leave salary can be processed in the current month or a different month, with automatic salary adjustments across pay cycles.

    • System Limitations

    Reporting Limitations


  • Leave Salary Accrual Report:

  • Cannot customize default sorting (no hire date sorting available)

  • Lacks number formatting precision (displays as text/general format)

  • Missing columns: comments, nationality, gender, cost segment

  • Report is generated on-the-go, limiting real-time customization

  • Does not show month-by-month breakdown for multi-month leave requests

  • Leave Salary Report Download:

  • Known issue prevents downloading reports for specific staff members through standard reporting interface

  • Workaround: Use Payroll table reports to access leave salary data

  • Payroll Template Data:

  • Missing absent days count (only absent amount available)

  • Lacks complete leave salary policy breakdown in custom templates

  • Cannot view leave encashment data within Leave salary accrual report (separate report required)

    • Currency and Localization


  • Currency Label Display: The landing page displays a hardcoded currency label (AED) that does not automatically adapt to the company's configured currency. While actual leave salary requests use the correct currency, the static landing page label cannot be customized or removed.

    • Request Processing Limitations


  • Closed Payroll Cycle Restriction: Leave salary requests cannot be automatically triggered if the leave start date falls in a closed payroll cycle. Manual splitting of leave requests is required.

  • Cross-Cycle Leave Processing: The system lacks capability to automatically handle leave requests that span multiple payroll cycles. Workaround requires manual splitting of leave requests aligned with each payroll cycle.

  • Related Request Tracking: No feature exists to link, flag, or consolidate leave salary requests that are split across different leave cycles, requiring manual tracking by administrators.

    • Configuration Constraints


  • Retroactive Policy Assignment: The system does not support retroactive leave salary policy assignments. Policy eligibility is fixed at the time of leave request creation.

  • Policy Modification: Employees not automatically included in leave salary policy from their leave policy must be manually added. Past leave requests remain unaffected by policy changes.

    • Data Synchronization


  • Approval Queue Population: Known issue where not all approved leave requests consistently appear in the leave salary payroll approval queue, potentially causing incomplete processing.

    • Workflow Limitations


  • Leave Encashment Requests: Cannot create encashment requests directly from the Leave Salary page. Separate workflow required.

  • Payroll Table Filtering: Limited filtering options in payroll table for leave salary specific data.

    • Unsupported Scenarios

    Multi-Policy Assignment

    Employees cannot be assigned to multiple leave salary policies simultaneously. Organizations with complex compensation structures requiring different leave salary calculations must manage employees in separate policy groups.

    Retrospective Leave Salary

    Leave requests created before an employee is assigned to a leave salary policy cannot be retrospectively processed for leave salary. The employee must:

  • Be assigned to the leave salary policy

  • Delete the existing leave request

  • Create a new leave request

    • Consolidated Multi-Cycle Approvals

    When leave spans multiple cycles, administrators must approve each generated leave salary request separately. No consolidated approval mechanism exists.

    Advanced Report Customization

    The leave salary accrual report cannot be customized in real-time for:

  • Custom column additions

  • Dynamic sorting preferences

  • Cell formatting modifications

  • Integrated encashment data

    • Known Issues

    Report Generation

    Issue: Unable to download leave salary report for specific staff members

  • Impact: Users cannot generate individual staff reports through standard interface

  • Workaround: Access data through Payroll table reports path

  • Status: Known issue, no timeline for resolution

    • Currency Display

    Issue: Landing page shows hardcoded AED currency label regardless of company configuration

  • Impact: Creates confusion for non-AED currency users

  • Workaround: None available; actual transactions use correct currency

  • Status: Configuration not currently possible

    • Approval Queue Synchronization

    Issue: Approved leave requests may not consistently appear in leave salary approval queue

  • Impact: Potential incomplete payroll processing

  • Workaround: Manual verification of approved leaves against leave salary queue

  • Status: Under investigation

    • Cross-Cycle Processing

    Issue: Leave requests spanning closed payroll cycles cannot auto-generate leave salary requests

  • Impact: Requires manual intervention and request splitting

  • Workaround: Split leave requests to align with open payroll cycles

  • Status: System limitation, not planned for change

    • Compliance Considerations

    Audit Trail


  • All leave salary calculations and approvals are logged with timestamps and user details

  • Remarks field available for documenting calculation adjustments

  • Transaction history visible in employee profile Leave salary tab

    • Data Privacy


  • Leave salary processing requires access to sensitive payroll data

  • Role-based access controls limit who can view and process leave salary requests

  • Recommended: Separate payroll and leave management permissions where possible

    • Regional Requirements


  • Daily wage calculation methods support various regional compliance requirements

  • Custom days option accommodates different labor law interpretations

  • Formula transparency enables verification against local regulations

üîß Troubleshooting & Edge Cases

Common Issues and Solutions

Leave Salary Request Not Generated

Symptom: Employee submitted approved leave but no leave salary request appears in the queue

Possible Causes and Solutions:

  • Employee Not Assigned to Leave Salary Policy

  • Check: Navigate to Settings > Payroll > Leave salary policy and verify employee is in the policy

  • Solution: Add employee to appropriate leave salary policy

  • Important: Employee must be assigned BEFORE creating leave request

  • Policy Assignment Timing Issue

  • Check: Compare leave request creation date with policy assignment date

  • Solution: If policy was assigned after leave request creation:

  • Delete the existing leave request

  • Ensure employee is in leave salary policy

  • Create new leave request

  • Prevention: Always assign employees to leave salary policy before they submit leave requests

  • Leave Start Date in Closed Payroll Cycle

  • Symptom: Leave request spans October and November, but October payroll is closed

  • Check: Verify leave start date against current active payroll cycle

  • Solution: Split the leave request into two separate requests:

  • First request: Covers days in closed cycle (manual processing may be required)

  • Second request: Covers days in open cycle (will auto-generate leave salary)

  • Workaround: Contact payroll team to reopen cycle if absolutely necessary

  • Leave Type Not Eligible

  • Check: Verify the leave type is included in the leave salary policy configuration

  • Solution: Edit leave salary policy to include the leave type, or create new policy

  • Minimum Days Requirement Not Met

  • Check: Review policy configuration for minimum days setting

  • Solution: If leave duration is below minimum, request will not generate. Employee must request longer leave or policy must be adjusted

    • Leave Salary Amount Incorrect

    Symptom: Calculated leave salary amount doesn't match expectations

    Diagnostic Steps:

  • Verify Salary Components

  • Navigate to leave salary request details

  • Check if calculation uses:

  • Basic salary only, or

  • Basic salary + allowances

  • Solution: If incorrect, review policy configuration under Settings > Payroll > Leave salary policy

  • Check Working Days Configuration

  • Review the "Working days" setting in policy:

  • Calendar days (30)

  • Working days (varies by month)

  • Custom days (fixed number)

  • Issue: Using "Working days" causes monthly variations

  • Solution: For consistent calculations, consider switching to "Custom days" with fixed value (e.g., 30)

  • Verify Allowances Selection

  • If policy uses "Basic salary + allowances", check which allowances are selected

  • Common Issue: Not all allowances included in calculation

  • Solution: Edit policy to add/remove allowances as needed

  • Review Manual Overrides

  • Check if amount was manually overwritten during approval

  • Look for remarks explaining the adjustment

  • Note: Once approved with override, amount cannot be changed unless moved back to pending

  • Multi-Month Calculation

  • For leaves spanning multiple months, verify calculation breakdown

  • Known Limitation: System shows total days √ó daily rate, not month-by-month breakdown

  • Workaround: Manual calculation required for month-specific amounts

    • Employee Not Appearing in Leave Salary List

    Symptom: Employee is assigned to policy but doesn't appear in leave salary management interface

    Troubleshooting:

  • Verify Policy Assignment Date

  • Check when employee was added to leave salary policy

  • Confirm leave request was created AFTER policy assignment

  • Solution: If timing is wrong, delete and recreate leave request

  • Check Leave Request Status

  • Verify leave request is approved (not pending or rejected)

  • Solution: Approve the leave request first

  • Confirm Leave Type Eligibility

  • Ensure leave type is included in employee's leave salary policy

  • Solution: Update policy or create new request with eligible leave type

  • Probation Status

  • Check if employee is on probation and policy restricts probation leave salary

  • Solution: Either wait until probation ends or modify policy settings

    • Cannot Approve Leave Salary Request

    Symptom: Approve button is disabled or action fails

    Possible Causes:

  • Insufficient Permissions

  • Required Roles: Super Admin, Payroll Table Manager, or Transaction Manager

  • Solution: Contact admin to grant appropriate permissions

  • Request Already Processed

  • Check request status in different tabs (Approved, At Payroll, Processed)

  • Solution: Request may have already moved to next stage

  • Payroll Cycle Closed

  • Verify the target payroll cycle is open

  • Solution: Reopen payroll cycle or process in next cycle

    • Leave Salary Not Appearing in Payroll Table

    Symptom: Approved leave salary request not showing in employee's payroll table

    Resolution Steps:

  • Check Request Status

  • Navigate to Payroll > Leave Salary

  • Verify request is in "At Payroll" or "Processed" status

  • Solution: If still in "Approved", click "Add to Payroll Table"

  • Verify Payroll Cycle

  • Confirm you're viewing the correct payroll cycle

  • Leave salary may be added to different month than leave occurrence

  • Solution: Check payroll table for the month specified in leave salary request

Key Terms and Definitions

A

Accrual Report: A report showing cumulative leave salary amounts calculated based on approved leave requests over a specified period. Displays employee-wise breakdown of leave salary entitlements.

Allowances: Additional salary components beyond basic salary (e.g., housing allowance, transportation allowance, food allowance) that can be included in leave salary calculations based on policy configuration.

At Payroll: A leave salary request status indicating the approved amount has been added to the employee's payroll table and is ready for payment processing.

B

Basic Salary: The fundamental component of an employee's compensation, excluding allowances and bonuses. Can be used as the sole basis for leave salary calculation.

Basic Salary + Allowances: A leave salary calculation method that includes both basic salary and selected allowances in the daily wage computation.

C

Calendar Days: A working days calculation method using 30 days as the divisor for daily wage calculation, regardless of actual working days in the month.

Closed Payroll Cycle: A payroll period that has been finalized and locked, preventing new transactions from being added. Leave salary requests cannot be created with start dates in closed cycles.

Cumulative Total: The sum of all leave salary amounts accrued over multiple periods, as shown in accrual reports.

Custom Days: A working days calculation method allowing organizations to specify a fixed number of days (e.g., 26, 28) as the divisor for daily wage calculation.

D

Daily Wage: The per-day salary rate calculated by dividing salary components by the configured number of working days. Used to compute leave salary amounts.

Daily Wage Calculator: System configuration determining how daily wage is calculated, including salary components and working days divisor settings.

E

Eligible Leave: Approved paid leave requests that qualify for leave salary processing based on policy configuration and employee assignment.

Encashment: The process of converting unused leave days into monetary payment. Separate from leave salary, which compensates for approved leave taken.

L

Leave Cycle: The period during which leave accrual and usage are tracked for an employee, which may differ from the payroll cycle.

Leave Salary: Monetary compensation provided to employees for approved paid leave days, calculated based on configured salary components and working days.

Leave Salary Formula: The calculation method defined in a leave salary policy, typically expressed as: (Salary component / Working days) √ó Number of leave days.

Leave Salary Policy: A configuration defining which employees are eligible for leave salary, which leave types qualify, and how leave salary amounts are calculated.

Leave Salary Request: An automatically generated or manually created request for leave salary payment, triggered by approved leave requests for eligible employees.

Leave Type: The category of leave (e.g., Annual Leave, Sick Leave, Maternity Leave) that determines eligibility for leave salary based on policy configuration.

M

Minimum Days: A policy setting requiring a minimum number of leave days before leave salary eligibility is triggered.

Multi-Month Leave: A leave request spanning more than one calendar month, which may generate separate leave salary requests for each month or payroll cycle.

O

Override Amount: A manually entered leave salary amount that replaces the system-calculated value, requiring remarks to document the reason for adjustment.

P

Paid Leave: Leave types where employees continue to receive salary during absence. Only paid leave types can be configured for leave salary.

Payroll Cycle: The recurring period for salary processing and payment, typically monthly. Leave salary requests must align with open payroll cycles.

Payroll Table: The system interface displaying all salary components, additions, and deductions for employees in a specific payroll cycle.

Pending: A leave salary request status indicating the request is awaiting review and approval by authorized personnel.

Policy Assignment Date: The date when an employee is added to a leave salary policy, which determines eligibility for future leave requests.

Probation Period: The initial employment period during which an employee's performance is evaluated. Leave salary policies can be configured to restrict leave salary during probation.

Processed: A leave salary request status indicating the amount has been paid to the employee and the transaction is complete.

Proration: The calculation method for adjusting salary amounts based on partial periods, such as when leave spans only part of a month.

R

Remarks: Text field allowing administrators to document reasons for amount overrides, special circumstances, or calculation adjustments in leave salary requests.

Request Status: The current stage of a leave salary request in the approval and processing workflow (Pending, Approved, Rejected, At Payroll, Processed).

Retroactive Assignment: Attempting to apply a leave salary policy to leave requests created before the policy assignment date. Not supported by the system.

S

Salary Component: Elements of employee compensation (basic salary, allowances) that can be included in leave salary calculations.

SIF File: Salary Information File used in Wage Protection System (WPS) for processing salary payments to employees.

Split Request: The practice of dividing a single leave period into multiple leave salary requests, typically required when leave spans multiple payroll cycles.

T

Transaction: A payroll entry representing a leave salary payment, visible in the payroll table and employee profile.

V

Variable Pay File (VPF): A payroll file format used to add one-time or variable payments, including leave salary amounts, to employee compensation.

W

Working Days: A calculation method for daily wage using the actual number of working days in a month as the divisor. Causes monthly variations in daily wage rates.

WPS (Wage Protection System): A government-mandated system in some regions for electronic salary transfer, ensuring timely and full payment of wages.

Role Definitions

Payroll Manager: User role with permissions to process payroll transactions, including leave salary requests.

Payroll Table Manager: User role with specific permissions to manage entries in the payroll table.

Super Admin: User role with full system access and permissions across all modules, including leave salary configuration and processing.

Transaction Manager: User role with permissions to create, edit, and process payroll transactions.

Status Definitions

Approved: Leave salary request has been reviewed and approved, ready to be added to payroll table.

At Payroll: Leave salary amount has been added to the employee's payroll table for the relevant cycle.

Pending: Leave salary request is awaiting review and approval by authorized personnel.

Processed: Leave salary has been paid to the employee and the transaction is complete.

Rejected: Leave salary request has been denied and will not be processed for payment.
