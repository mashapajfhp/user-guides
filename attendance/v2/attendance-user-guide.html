What is Attendance?

Overview

Attendance tracks employee work hours through mobile check-ins, biometric devices, or manual entry. Access it from Time ‚Üí Employee Attendance or via the Bayzat mobile app.

The system captures check-in/check-out times, calculates work hours, identifies violations (late arrivals, early departures, absences), and generates data for payroll processing.

Key Benefits

  • Eliminate manual timesheet collection and data entry
  • Enforce work timing policies with automated violation detection
  • Calculate overtime, deductions, and days in lieu automatically
  • Prevent buddy punching with facial recognition or biometric verification
  • Generate audit-ready attendance reports for payroll and compliance
  • Support flexible work arrangements including shifts, remote work, and split schedules

Product Foundation Overview

How Attendance Fits

Attendance is a transactional data layer that captures employee work hours through check-ins and check-outs. It sits between configuration (work timings, offices) and payroll processing.

Mental model: Work Timing (template) ‚Üí Assigned to Employee ‚Üí Check-in/Check-out recorded ‚Üí Attendance record created ‚Üí Feeds Time & Pay Adjustments ‚Üí Flows to Payroll

One work timing configuration affects all employees assigned to it. Attendance records are created daily and feed downstream processes like overtime calculations and deductions.

Key Decisions Before Setup

Answer these questions before configuring Attendance:

  • Which tracking method fits your workforce? ‚Äî Mobile app for field workers, biometric machines for on-site staff, or manual entry for mixed environments
  • What work patterns exist? ‚Äî Fixed schedules, rotating shifts, split shifts, or flexible hours
  • How will you handle violations? ‚Äî Define late arrival thresholds, early departure rules, and absence penalties before employees start checking in
  • What compensation policies apply? ‚Äî Decide overtime rates, days in lieu conversion, and deduction formulas that align with labor law

Related Features

  • Work Timings ‚Äî Defines the schedule templates (start/end times, break rules) that attendance records are validated against
  • Shift Scheduler ‚Äî Assigns specific shifts to employees when work patterns vary daily or weekly
  • Time & Pay Adjustments ‚Äî Processes attendance data into overtime, deductions, and days in lieu for payroll
  • Payroll ‚Äî Receives finalized attendance impacts (overtime pay, absence deductions) for salary calculations
  • Leave Management ‚Äî Marks employees on approved leave as exempt from attendance requirements

Initial Setup

Before employees can track attendance, configure these core settings. This setup creates the foundation for all attendance tracking.

Step 1: Define Office Locations

Create office entries with geographical coordinates for location-based check-ins.

Settings ‚Üí Attendance ‚Üí Offices
  • Add office name and address
  • Set geographical radius for check-in validation
  • Enable or disable out-of-office check-ins

Step 2: Configure Work Timings

Define shift schedules with start/end times, break durations, and attendance rules.

Settings ‚Üí Attendance ‚Üí Work Timings
  • Set standard or flexible work hours
  • Define late arrival and early departure thresholds
  • Configure break allowances and maximum durations
  • Set "Absent after" timing rules

Note: The default work timing cannot be deleted. Create additional timings for different shift patterns.

Step 3: Define Workweeks

Specify working days and rest days for your organization.

Settings ‚Üí Attendance ‚Üí Workweeks
  • Select working days (e.g., Sunday to Thursday)
  • Mark weekend days
  • Configure public holiday handling

Step 4: Enable Image Capture (Optional)

Turn on photo verification for check-ins to prevent proxy attendance.

Settings ‚Üí Attendance ‚Üí Image Capture
  • Enable image capture toggle
  • Employees will be prompted to take a photo during check-in
  • Photos are stored for audit and verification

Step 5: Assign Employees to Settings

Link employees to their office, work timing, and workweek configurations.

People ‚Üí Employees ‚Üí [Employee] ‚Üí Work Tab
  • Select office location
  • Assign work timing schedule
  • Set workweek pattern

Required: Employees must be assigned to an office, work timing, and workweek before they can check in via the mobile app.

Step 6: Configure Work Centers (For Shift Workers)

If using shift scheduling, create work centers to group employees by shift patterns.

Settings ‚Üí Attendance ‚Üí Work Centers
  • Create work center (shift group)
  • Assign schedulers who can manage shifts
  • Add office locations and work timings
  • Assign employees to the work center

Note: An employee can only belong to one work center at a time.

Step 7: Set Minimum Extra Hours Threshold (Optional)

Define the minimum overtime hours required before employees can request compensation.

Settings ‚Üí Attendance ‚Üí Policies
  • Set minimum threshold (e.g., 1 hour)
  • Employees cannot request overtime below this threshold

Step 8: Enable Line Manager Permissions (Optional)

Control whether line managers can modify compensation impacts for attendance adjustments.

Settings ‚Üí Attendance ‚Üí Policies
  • Toggle ON: Line managers can modify compensation amounts
  • Toggle OFF: Line managers can only review and reject requests

Alternative Automation Options

While direct workflow integration is not available, attendance processes can be automated through:

  • Attendance Policies ‚Äî Overtime, Days in Lieu, and Deduction policies automatically generate time and pay adjustments based on attendance records
  • Approval Flows ‚Äî Attendance regularization tickets follow configured approval chains for check-in/check-out corrections
  • Scheduled Extra Hours ‚Äî Line managers can pre-assign overtime that automatically appears in employee schedules
  • Shift Publishing ‚Äî Published shifts trigger push notifications to employees via the mobile app

Integration Points

Attendance data flows to other modules automatically:

  • Payroll Module ‚Äî Approved time and pay adjustments (overtime, deductions) automatically sync to payroll processing
  • Time Off Module ‚Äî Days in Lieu policy creates leave balances; approved leave requests block attendance tracking
  • Employee Records ‚Äî Attendance configurations (office, work timing, work center) are stored in employee profiles
  • Biometric Systems ‚Äî Third-party biometric devices can push check-in/check-out data via API, CAMS, or vendor integrations

Business Rules & Limitations

KSA Labor Law Compliance

Attendance deduction policies must align with KSA Ministry of Human Resource and Social Development labor law articles. Penalty escalation is based on occurrence count (e.g., 15 min late: 1st=warning, 2nd=5% daily wage). The system enables customer decisions coherent with labor law rather than strict implementation.

Edge Cases

  • Multiple violation types same day: Late arrival and early departure on same day are tracked separately. Each violation counted independently for occurrence-based penalties.
  • Retroactive policy changes: Policy edits only affect future attendance. Historical violations remain under original policy rules. Manual adjustments required for retroactive changes.
  • Cross-border employees: UAE vs KSA labor law rules may differ. Configure separate policies per region. Verify compliance with local regulations.
  • Grace period handling: Minor lateness within grace period does not trigger violations. Grace period configured in work timing settings.
  • Public holidays and leave: Attendance calculations automatically exclude approved leave days and public holidays. Verify holiday calendar is up to date.
  • Back-to-back shift checkout: Configurable window between shift end and next start determines checkout behavior. Within window: checkout allowed. At next shift start: check-in plus auto-checkout for previous shift.
  • Kiosk mode for non-shift employees: Kiosk mode works for all employees without smartphones. Image capture mandatory when enabled. Managers can mark attendance on behalf of employees.
  • Attendance record creation date errors: Timestamp validation issues may cause incorrect date assignments. Verify time zone settings and system clock synchronization.
  • Line manager override requests: Line managers can review and approve/reject violations. Documentation required for audit. Escalate to HR for final approval.
  • Employee disputes on penalties: Use attendance regularization tickets for corrections. Provide evidence and justification. Follow approval workflow for resolution.
  • Partial attendance days: Half-day leave prevents check-in for full day. Configure half-day leave policies separately. Use attendance regularization for exceptions.
  • Overtime on days off: Only shift employees eligible. Weekends and public holidays considered full extra hours. Verify employee assigned to shift schedule.
  • Missing reference photo: Admin prompted to upload passport photo before enabling facial recognition. Cannot use facial recognition without reference photo.
  • Concurrent edit conflicts: System lacks robust concurrent edit handling. Last save wins. Coordinate with other admins to avoid conflicts.
  • Approval flow creator is final approver: Ticket remains in Pending if creator is last approver. Requires separate approver or manual intervention.

FAQs & Support

Frequently Asked Questions

How do I enable attendance tracking for my organization?

Navigate to Settings ‚Üí Attendance, then configure work timings, offices, and workweeks. Assign employees to specific offices and work timings in their employee profiles. Enable image capturing or biometric integration if needed for identity verification.

Can employees check in from outside the office location?

By default, check-ins are restricted to the office radius defined in Settings ‚Üí Offices. You can enable "Allow check-in from anywhere" in work timing settings to permit remote check-ins, or use the Out of Office feature for temporary remote work.

What happens if an employee forgets to check out?

The system will not calculate worked hours for that day. Admins or line managers can manually edit the attendance record from Time ‚Üí Employee Attendance to add the missing check-out time.

How do I configure overtime policies?

Go to Settings ‚Üí Payroll ‚Üí Overtime Policy, click Add New, name the policy, select hourly rate type (Single or Custom), configure rate multipliers and salary components, then assign employees to the policy.

Can I schedule split shifts for employees?

Yes. Navigate to Time ‚Üí Shift Scheduler, select a work center and date, then create multiple shifts for the same employee on the same day. Maximum of 2 shifts per employee per day. Shifts cannot overlap or be scheduled on days off.

How do attendance deductions work?

Configure deduction policies in Settings ‚Üí Attendance ‚Üí Deduction Policy. Set separate formulas for absent days, late arrivals, and early departures. The system automatically calculates deductions based on attendance records and assigns them to Time & Pay Adjustments for approval.

Can employees request attendance regularization?

Yes, if enabled. Admins configure attendance regularization in Settings ‚Üí Tickets ‚Üí Attendance Regularization, assign employees, and set up approval flows. Employees can then submit tickets to correct check-in/check-out times.

How do I mass upload attendance data from biometric devices?

Navigate to Settings ‚Üí Attendance ‚Üí Biometric Data. Choose manual entry (add rows individually) or Excel upload (download template, fill in employee names, dates, times, then upload). Uploading data for the same employee on the same date overwrites existing records.

What is the difference between Days in Lieu and Overtime policies?

Overtime policies compensate extra hours with monetary payment added to payroll. Days in Lieu policies convert extra hours into time off (default: 8 hours = 1 day) credited to the employee's leave balance.

Can line managers see attendance deduction amounts?

Only if the super admin enables the visibility toggle in Settings ‚Üí Attendance ‚Üí Policies. When off, line managers can review and select work disruption status but cannot see calculated amounts.

How do I publish shift schedules to employees?

In Shift Scheduler, create shifts and save as draft. Review the schedule, then click Publish Shifts (available only when a specific work center is selected). Employees receive push notifications on the Bayzat mobile app.

Can I copy shifts to multiple employees at once?

Yes. Use Schedule Planner under Shift Scheduler to select shift type, office, and timings, then assign to multiple employees simultaneously. Alternatively, use the Copy function to copy one employee's shifts and paste to others for a date range up to 90 days.
