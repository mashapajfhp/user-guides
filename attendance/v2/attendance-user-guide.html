What is Attendance?

Overview

Attendance tracks employee work hours through mobile app check-ins or biometric devices. Admins access it via Time > Employee Attendance to view daily records, generate reports, and manage time-based pay adjustments.

Key actions: View attendance records, edit check-in/out times, generate custom reports, manage deductions and overtime, export data for payroll.

Key Benefits

  • Eliminate manual timesheet entry and calculation errors
  • Prevent proxy attendance through location verification and optional facial recognition
  • Automate overtime and deduction calculations based on configurable policies
  • Generate audit-ready reports showing worked hours, late arrivals, early departures, and absences
  • Integrate attendance data directly with payroll processing

Product Foundation Overview

How Attendance Fits

Attendance is transactional data that captures employee work hours in real-time through check-ins and check-outs. It sits at the operational core of Time & Pay management, feeding data to payroll, overtime calculations, and deduction policies.

Mental model: Work Timing (template) ‚Üí Assigned to Employee ‚Üí Employee checks in/out ‚Üí System records hours ‚Üí Attendance data flows to payroll and reports

Every attendance record impacts payroll calculations, compliance reporting, and workforce analytics across the organization.

Key Decisions Before Setup

Answer these questions before configuring Attendance:

  • What tracking method fits your workforce? ‚Äî Mobile app for flexible/remote workers, biometric devices for on-site staff, or kiosk mode for employees without smartphones
  • How do you define work hours? ‚Äî Fixed schedules vs. flexible hours vs. shift-based patterns determine which work timing templates you need
  • What attendance violations matter? ‚Äî Decide which behaviors trigger deductions (late arrival, early departure, absence) and configure thresholds before employees start checking in
  • How will overtime be compensated? ‚Äî Choose between monetary payment, days in lieu, or no compensation to set up the right policies upfront

Related Features

These Bayzat features work together with Attendance. Where available, links point to the published user guide.

Initial Setup

Before employees can check in, configure the foundational attendance settings. This section covers the minimum required setup to start tracking time.

1. Create Offices

Define physical locations where employees work. Each office requires a name, address, and geographical coordinates for geo-fencing.

  1. Navigate to Settings ‚Üí Attendance ‚Üí Offices
  2. Click Add Office
  3. Enter office name and address
  4. Set geo-fence radius (determines check-in proximity)
  5. Save office

Geo-fencing: Employees can only check in when physically within the defined radius of the office location.

2. Define Work Timings

Create shift templates that specify start/end times, break durations, and late arrival thresholds.

  1. Navigate to Settings ‚Üí Attendance ‚Üí Work Timings
  2. Click Add Work Timing
  3. Enter shift name (e.g., "Morning Shift", "9-5 Standard")
  4. Set start and end times
  5. Configure break duration (e.g., 1 hour lunch)
  6. Set late arrival threshold (grace period before marking late)
  7. Set early departure threshold
  8. Choose flexible hours option if applicable
  9. Save work timing

Default work timing: The system creates a default work timing that cannot be deleted. You can edit it or create additional timings.

3. Configure Work Weeks

Define which days are working days versus weekends for your organization.

  1. Navigate to Settings ‚Üí Attendance ‚Üí Work Weeks
  2. Select working days (e.g., Sunday-Thursday for UAE)
  3. Mark weekend days
  4. Save work week

4. Assign Employees to Settings

Connect employees to their office, work timing, and work week. This determines where and when they can check in.

  1. Navigate to People ‚Üí Employees
  2. Select an employee
  3. Go to Work tab
  4. Assign office location
  5. Assign work timing
  6. Assign work week
  7. Save employee settings

Prerequisite: Employees must have all three assignments (office, work timing, work week) before they can check in via the mobile app.

5. Enable Image Capture (Optional)

Verify employee identity during check-in by requiring photo capture.

  1. Navigate to Settings ‚Üí Attendance ‚Üí Image Capture
  2. Toggle image capture ON
  3. System will prompt employees to take a photo at check-in/check-out

For advanced identity verification, configure Biometric Attendance with facial recognition and liveness detection.

6. Configure Attendance Policies (Recommended)

Set up policies for overtime compensation, deductions, and days in lieu to automate payroll impacts.

Overtime Policy

  1. Navigate to Settings ‚Üí Attendance ‚Üí Overtime Policy
  2. Click Add New
  3. Name the policy
  4. Choose hourly rate type (Single Rate or Custom Rate)
  5. Set rate multiplier (e.g., 1.5x for overtime)
  6. Select salary component (Basic Salary or Basic + Allowances)
  7. Assign employees to policy
  8. Save policy

See the Overtime guide for detailed policy configuration.

Deduction Policy

  1. Navigate to Settings ‚Üí Attendance ‚Üí Deduction Policy
  2. Click Add New
  3. Name the policy
  4. Configure formulas for late arrivals, early departures, and absences
  5. Set deduction amounts or percentages
  6. Assign employees to policy
  7. Save policy

7. Set Up Work Centers (For Shift Workers)

If your organization uses shift scheduling, create work centers to group employees by shift patterns.

  1. Navigate to Settings ‚Üí Attendance ‚Üí Work Centers
  2. Click Add Work Center
  3. Name the work center (e.g., "Retail Floor", "Customer Service")
  4. Assign schedulers (who can create shifts)
  5. Assign offices
  6. Assign employees to this work center
  7. Save work center

For shift scheduling workflows, see the Shift Scheduling guide.

8. Configure Line Manager Permissions

Control what line managers can see and modify in attendance records.

  1. Navigate to Settings ‚Üí Attendance ‚Üí Policies
  2. Toggle "Allow Line Managers to impact compensation" ON or OFF
  3. When ON: Line managers can modify overtime amounts and deductions
  4. When OFF: Line managers can only review and reject entries

Workflow Integration

Not Available: Attendance does not currently support workflow automation.

While Attendance does not currently have workflow automation capabilities, you can still automate related processes using the Workflows feature for connected modules like Employee Tickets (for attendance regularization requests) and payroll adjustments.

Approval Flow

Note: Attendance tracking itself does not require approval.

However, attendance-related requests such as attendance regularization tickets route through approval flows. Employees submit correction requests via Employee Tickets, which are then routed to the appropriate approvers based on your organization's Approval Flow configuration.

Attendance Regularization Routing

When employees need to correct their check-in or check-out times, they create an attendance regularization ticket. The approval routing is determined by your organization's approval flow settings, typically involving line managers and HR administrators.

Routing Examples

Example 1: Employee submits attendance regularization ticket ‚Üí Routes to Line Manager for review ‚Üí Line Manager approves or rejects ‚Üí If approved, attendance record is automatically updated

Example 2: Employee requests correction for multiple days ‚Üí Routes to Line Manager first ‚Üí Escalates to HR if pattern indicates policy violation ‚Üí HR reviews and makes final decision

Example 3: Shift employee requests time adjustment ‚Üí Routes to Shift Scheduler for verification ‚Üí Then to Line Manager for approval ‚Üí Attendance record updates upon final approval

For detailed approval flow configuration, see the Approval Flow guide.

Business Rules & Limitations

Attendance Policy Constraints

Attendance policies follow specific configuration rules that affect how compensation and deductions are calculated.

KSA Labor Law Compliance

Penalty escalation for attendance violations must align with KSA Ministry of Human Resource and Social Development labor law articles. The system enables flexible interpretation rather than strict implementation.

By Design: Penalty amounts vary based on occurrence count (e.g., 15 min late: 1st=warning, 2nd=5% daily wage). Documentation is required for audits. Automatic updates maintain accuracy when occurrences are rejected.

Overtime Policy Rules

Rule Constraint Impact
Employee Assignment One overtime policy per employee at a time Employees already assigned to another policy show as 'Unavailable'
Rate Multiplier Minimum 0.1, no maximum, decimals rounded to 2 places Default multiplier is 1.5
Salary Component Basic Salary OR Basic Salary + Allowances Default is Basic Salary
Days Calculation Actual Calendar Days OR Actual Working Days OR Custom (30 days default) Affects hourly rate calculation
Working Hours Default 8 hours per day Used in hourly rate formula

Deduction Policy Editing

Remarks are mandatory when editing deduction amounts. Once a deduction is added to payroll, editing is locked and only removal is possible.

Deduction amounts can be set to any value (2 decimal places). The original calculated amount remains visible for audit purposes.

Shift Scheduling Limitations

Split Shifts

Phase 1 allows maximum 2 shifts per employee per day. Additional constraints apply:

  • Cannot schedule second shift if day off already exists on that day
  • Cannot schedule second shift if day off with check-in allowed exists
  • No overlap allowed with shifts on same day, day before, or day after
  • No minimum gap requirement between multiple shifts
  • Cannot schedule extra hours if multiple shifts exist
  • Each shift tracks late/early/absent status independently

Copy and Paste Restrictions

Action Limitation
Copy shifts Up to 90 days in advance
Work center selection Copy disabled when 'all' work center is selected
Paste restrictions Cannot paste on Day Off, Public Holiday, Published Shifts, or On Leave periods (unless overriding drafts)
Undo window 1 minute after pasting shifts

Attendance Regularization Constraints

Attendance regularization tickets have validation rules that prevent invalid corrections:

  • Check-in time cannot be later than check-out time
  • Cannot regularize attendance for future dates
  • Cannot modify records with processed time and pay adjustments
  • Cannot regularize when employee is on leave or not scheduled to work
  • Cannot modify break details or visit location timestamps
  • Cannot reject an already approved ticket

Note: Once a ticket is approved, it cannot be reverted. Further changes must be made manually in the attendance record.

Time and Pay Adjustments

Processing Order

Entries must be processed in occurrence order. Preceding entries must be handled before acting on subsequent entries.

Split Shift Handling

Scenario Rule
Extra hours scheduling Restricted for multiple shifts
Deductions Split shift deductions are halved
Entry modification Cannot modify entries once processed

Biometric and Image Capture

Facial Recognition Requirements

When facial recognition is enabled, specific constraints apply:

  • Facial recognition requires image capture to be enabled (cannot disable image capture if FR is on)
  • Retry attempts: configurable 1-5, default is 3
  • Reference photo must be employee primary passport photo from documents
  • Only Admins can view or update reference photos
  • Employees must accept consent prompt before first use
  • Location validation: Check-in only allowed if geo-fence rules are satisfied

Fallback Behavior

Condition Options
Retry limit exceeded Either fallback to image capture (default) OR block check-in and notify admin
Reference photo update Also updates the Primary Passport Photo in Employee Documents

Module Integration Constraints

Attendance-Payroll Integration

Policy edits only affect future attendance actions, not existing records. Deactivated policies stop impacting records immediately.

Note: Trade license filter not available in V1. Deductions from Time Off not included in V1.

Leave Interaction

When leave is approved, the system automatically removes attendance records for the leave period. For split shifts, this may result in partial day attendance removal.

Reporting Limitations

  • Cannot exclude specific employees from attendance reports
  • Line Managers can only view reports for employees who directly report to them
  • Custom report tab not available for admins and line managers
  • Offboarded employees created before offboarding feature may still appear in reports

Known Technical Issues

Browser Compatibility

Safari 10.1 Compatibility: The platform has inconsistent URLSearchParams behavior in Safari 10.1. Use a modern browser for best results.

Attendance Record Validation

The platform currently lacks automated validation for attendance records during payroll status transitions. Manual verification is recommended before finalizing payroll.

Permission Granularity

Current architecture has tightly coupled modules (Attendance, Shift Scheduler, Work Center, Time-off), preventing independent configuration. Granular permission settings are not available.

Shift Scheduling Edge Cases

Split Shift Scenarios

  • Back-to-back shifts checkout handling (Phase 2): Configurable window between shift end and next start determines checkout behavior. Within window: checkout allowed; At next shift start: check-in plus auto-checkout for previous shift; Beyond window: normal checkout.
  • Deduction policy complexity: Hourly rate vs daily wage for absenteeism calculations across multiple shifts needs consideration.
  • Day off presence blocks split shift creation: Cannot create second shift if day off exists on that day.

Repeating Shifts

  • Undo window: Undo option only visible for one minute after pasting shifts
  • Weekend coverage: Paste option includes weekends automatically
  • Predefined periods: Cannot paste shifts into holidays or days off without override

Overtime and Deduction Edge Cases

Multiple Edits and Workflow

  • Multiple edits before payroll: Each edit tracked separately in activity history
  • Editing to zero amount: Full waiver allowed; original amount remains visible for audit
  • Editing to higher amount: Stricter penalty than calculated can be applied
  • Closing modal without saving: Retains "Assigned to Payroll" status; changes not saved

Access and Visibility

  • Line manager visibility: Depends on super admin toggle setting
  • Attendance manager access: Needs payroll role or visibility toggle enabled to view amounts
  • Role changes mid-process: May affect ability to complete pending actions

Facial Recognition Edge Cases

  • Missing reference photo: Admin must upload passport photo before enabling FR for that employee
  • Liveness detection failure: Ensures image is from live human, not a photo of a photo; retry or use fallback
  • Retry limit exceeded: System falls back to configured behavior (block or image capture)
  • Kiosk multi-employee scans: Supports back-to-back sequential check-ins for multiple employees
  • Consent not accepted: Employee cannot proceed with FR until consent is given
  • Geo-fence violation: Check-in blocked if location rules not satisfied even with successful FR

Attendance Regularization Edge Cases

  • Ticket creator is final approver: System may block ticket progression; requires workflow review
  • Multiple shifts on regularization date: Cannot specify which shift to regularize; applies to entire day
  • Regularization during processed pay period: Cannot modify records with finalized time and pay adjustments

Reporting Edge Cases

  • Multiple visits per day: Location report shows cumulative or breakdown format based on selection
  • Split shift status in daily report: If leave request created for one shift, status may not update to "on leave" for both rows
  • Confidence column visibility: Shows only when both image comparison AND liveness detection are enabled, not just comparison alone

Integration Edge Cases

Biometric Machine Integration

  • Duplicate check-in records: Lack of unique index may allow redundant mobile attendance records
  • API integration complexity: May require developer expertise for CAMS or vendor systems

Payroll Integration

  • Pay adjustment blocking: Cannot edit or delete pay adjustment requests linked to attendance records
  • Attendance ON/OFF sync: Lacks standardized API for managing attendance status across payroll-related companies

User Experience Edge Cases

  • Raw error messages: System may expose technical error messages directly to users
  • Non-shift user on leave: Check-in screen shows generic message instead of differentiated messaging
  • Missing office in override dialog: System cannot automatically guide users to resolve missing employee workspace information
  • Double-click on Remove From Payroll: Lacks robust prevention mechanism; rapid clicks may trigger unintended actions

Data Integrity Edge Cases

  • Inactive employee mentions: System allows mentioning inactive or off-boarded employees in comments

FAQs & Support

Frequently Asked Questions

Can employees check in from outside the office location?

This depends on your attendance settings. Admins can configure geo-fencing to allow or restrict out-of-office check-ins. If enabled, employees can check in remotely; if disabled, they must be within the office radius.

What happens if an employee forgets to check out?

The system will not count work hours for that day until a check-out is recorded. Admins or line managers can manually edit the attendance record to add the missing check-out time.

How do I handle split shifts for employees working multiple times per day?

Attendance supports up to 2 shifts per day. Each shift is tracked independently with separate check-in/check-out times, and late/early/absent status is calculated per shift.

Can I modify attendance records after they've been approved for payroll?

Once attendance records are approved and added to payroll, they cannot be moved back to pending. Further changes must be made manually in the employee's attendance record.

How does facial recognition prevent proxy attendance?

Facial recognition uses AWS Rekognition to compare the employee's live camera image against their reference photo from documents. Liveness detection ensures the image is from a real person, not a photo of a photo.

What's the difference between overtime policies and days in lieu?

Overtime policies compensate extra hours with additional pay added to payroll. Days in lieu convert extra hours into time-off leave credits (default: 8 hours = 1 day). Employees can be assigned to either policy type.

How do I bulk upload attendance data from biometric devices?

Navigate to Settings > Attendance > Biometric Data. Download the Excel template, fill in employee names, check-in/check-out dates and times, then upload the completed file. Uploading data for the same employee on the same date will override existing records.

Can line managers approve attendance regularization requests?

Yes, if the attendance regularization policy is configured with an approval flow that includes line managers. Requests route through the defined approval chain before updating attendance records.

How do I exclude specific employees from attendance reports?

Currently, the system does not support excluding individual employees from attendance reports. You can use filters to focus on specific departments, offices, or teams, but all employees within those groups will appear.

What happens to attendance records when an employee is on leave?

Employees on approved leave are exempt from check-in/check-out requirements for those dates. The attendance report will show their status as "On Leave" and no absence deductions will be applied.
