What is Attendance?

Overview

Attendance tracks employee work hours through mobile check-ins, biometric devices, or manual entry. Admins access it via Time > Attendance > Employee attendance to view daily reports, edit records, and manage time and pay adjustments.

Key actions: View attendance reports, edit check-in/out times, create leave requests, approve time and pay adjustments, export data, and configure attendance policies.

Key Benefits

  • Eliminate manual timesheet collection and data entry errors
  • Automatically calculate overtime, deductions, and days in lieu based on configured policies
  • Track attendance across multiple locations with geo-fencing and biometric verification
  • Route attendance corrections through approval workflows before payroll processing
  • Generate compliance reports showing late arrivals, early departures, and absences
  • Integrate attendance data with payroll for automatic salary adjustments

Product Foundation Overview

Attendance is transactional data that records when employees check in and out of work. It sits at the intersection of time tracking, payroll processing, and compliance monitoring.

Mental model: Work Timing (template) ‚Üí Assigned to Employee ‚Üí Employee checks in/out ‚Üí Attendance record created ‚Üí Hours calculated ‚Üí Flows to payroll

Initial Setup

Before employees can track attendance, configure the foundational settings that define work schedules, locations, and tracking rules.

1. Define Work Timings

Work timings are templates that specify when employees work. Navigate to Settings > Attendance > Work Timings.

Click Add new to create a work timing. Configure:

  • Name: Descriptive label (e.g., "Morning Shift 8-5")
  • Working hours: Start and end time
  • Late arrival threshold: Grace period before marking late
  • Early departure threshold: Minimum time before shift end to avoid penalty
  • Breaks: Total break duration (deducted from worked hours)
  • Extra hours calculation: How overtime is tracked (e.g., "All Hours Worked" for hourly workers)

Flexible timing: Enable this to allow employees to check in/out within a window instead of fixed times. Look for the "Flexible timing" badge on work timing rows.

2. Create Office Locations

Offices define where employees work and enable geo-fencing for mobile check-ins. Navigate to Settings > Attendance > General accordion.

For each office:

  • Office name: Location identifier
  • Address: Physical location
  • Geo-fence radius: Distance within which check-ins are allowed
  • Allow out-of-office check-in: Toggle to permit check-ins outside the radius

3. Configure Attendance Policies

Set up policies that translate attendance data into payroll impacts. Navigate to Settings > Attendance.

Overtime Policy (Recommended)

Defines how extra hours are compensated. Expand Overtime Policies accordion.

  • Choose rate type: Single Rate (one multiplier) or Custom Rate (different rates for weekdays/weekends/holidays)
  • Set hourly rate multiplier (e.g., 1.5x for time-and-a-half)
  • Select salary component: Basic Salary or Basic Salary + Allowances
  • Assign employees to the policy

Days In Lieu Policy (Recommended)

Converts extra hours into time-off credits. Expand Days In Lieu Policies accordion.

  • Set hours-to-day conversion (default: 8 hours = 1 day)
  • Assign employees to the policy

Deductions Policy (Recommended)

Applies penalties for late arrivals, early departures, or absences. Expand Deductions Policies for Payroll accordion.

  • Configure separate formulas for: Absent days, Early departures, Late arrivals
  • Set deduction calculation method (percentage of daily wage or fixed amount)
  • Assign employees to the policy

Policy assignment: Each employee can belong to only ONE policy per type (one overtime policy, one days in lieu policy, one deduction policy). Employees already assigned to a policy will show as "Unavailable" when creating a new policy of the same type.

4. Enable Biometric Integration (Optional)

If using biometric devices or API-based attendance systems, configure data sources. Navigate to Settings > Attendance.

  • Biometric Devices: Expand accordion to register fingerprint readers or facial recognition devices
  • Attendance API Settings: Expand accordion to generate API credentials for third-party integrations
  • Biometric Data: Expand accordion to manually upload attendance records from external systems

For detailed biometric setup, see the Biometric Attendance guide.

5. Set Up Work Centers for Shift Workers (Conditional)

Required only for shift-based scheduling: Work centers group employees who share similar shift patterns.

Navigate to Settings > Attendance > Work Centers for Shift Scheduling accordion.

  • Create work center with a descriptive name
  • Assign schedulers (users who can create shifts for this group)
  • Add office locations where this group works
  • Assign employees to the work center

Note: An employee can belong to only one work center. Work centers are required to use the Shift Scheduling feature.

6. Assign Employees to Settings

After creating work timings, offices, and policies, assign employees to their configurations. Navigate to People > Employees.

For each employee, edit their profile and set:

  • Office: Primary work location
  • Work timing: Assigned schedule template
  • Work center: (If using shift scheduling)

Required for attendance tracking: Employees MUST have an assigned office and work timing before they can check in/out. Records will not be generated for employees missing these assignments.

7. Configure Approval Flow for Regularization (Recommended)

If employees will request attendance corrections via tickets, set up the approval routing. Navigate to Settings > Approval Flow > Attendance Regularization.

  • Define approval steps (e.g., Line Manager ‚Üí HR)
  • Set conditions for advanced routing (e.g., escalate to Super Admin if correction exceeds 2 hours)
  • Assign default approvers for the Else (fallback) path

For detailed approval setup, see the Approval Flow guide.

Subtask: View Location Report

Task: Manage Time and Pay Adjustments

Review and approve overtime, deductions, and other pay impacts generated from attendance data.

  • Click Approve for Payroll to move to Approved tab

Subtask: Reject Adjustments

  • Select entries to reject
  • Click Reject and add reason

Attendance supports workflow automation through event-based triggers. The system currently monitors shift scheduling events to enable automated responses.

  • Trigger: Employee shift is deleted, updated, or created

Approval Flow

How Attendance Requests Route

Attendance Regularization requests require approval before updating employee attendance records. When an employee submits a regularization ticket to correct check-in or check-out times, the request routes through a configured approval chain. The default approver is Super Admin, but organizations can configure advanced routing rules with conditional logic based on employee attributes, request amount, or other criteria.

Types Requiring Approval

Attendance Regularization is the primary ticket type requiring approval in this module. When employees need to correct their attendance records (missed check-in, incorrect check-out time, or other attendance discrepancies), they submit a regularization ticket that includes the corrected check-in date/time and check-out date/time. The system validates the request against attendance policies before routing to the approval chain.

Routing Examples

How different Attendance requests route through approvers:

Example 1: Standard Attendance Regularization

Employee submits regularization ticket to correct a missed check-out ‚Üí Request routes to Super Admin (default approver) ‚Üí Super Admin reviews attendance record and ticket details ‚Üí Approves or rejects ‚Üí System updates attendance record if approved

Example 2: Line Manager First Review

Employee submits regularization for late arrival correction ‚Üí Advanced flow routes to Line Manager first (IF employee reports to a Line Manager) ‚Üí Line Manager reviews and approves ‚Üí Request then routes to Super Admin for final approval ‚Üí Attendance record updates upon final approval

Example 3: High-Impact Corrections

Employee submits regularization that changes status from Absent to Present (high impact on payroll) ‚Üí Advanced flow detects impact level ‚Üí Routes to Line Manager AND HR Manager in parallel (both must approve) ‚Üí Then routes to Super Admin for final sign-off ‚Üí Attendance record and Time and Pay Adjustments update only after all approvals complete

Example 4: Self-Approval Blocked

Line Manager submits their own regularization ticket ‚Üí System detects ticket creator is in the approval chain ‚Üí Automatically skips Line Manager step and routes directly to Super Admin to prevent self-approval

For detailed approval flow configuration, see the Approval Flow guide.

Business Rules & Limitations

High-Impact Limitations

These limitations require immediate attention and may block critical workflows:

Backend Technical Constraints: The attendance module has API version compatibility issues that may prevent editing employee attendance information. The attendance controller lacks robust null parameter validation, and the attendance calculation backend has performance limitations in caching daily available actions. These issues may manifest under high load or specific conditions.

Approval Workflow Blockage with Self-Approval: When a ticket creator is also the final approver in the authorization chain, the ticket remains stuck in Pending status and the associated attendance record fails to update. Configure approval flows to avoid self-approval scenarios.

Attendance Data Entry Timestamp Issues: The platform lacks robust timestamp validation during manual attendance record creation, resulting in incorrect date assignments and work hour computations. Concurrent edits may cause race conditions. Always verify timestamps after manual entry.

Biometric Security Implementation Gap: AWS Rekognition currently lacks a Flutter SDK for face liveness detection, creating a technical barrier that prevents implementation of comprehensive biometric security. The biometric settings focus on device configuration and data import, not facial recognition liveness.

UAE Overtime Policy Compliance Gap: The platform lacks comprehensive overtime policy configuration that meets specific UAE Federal labor law requirements for different employee classifications and overtime rate calculations. Overtime policies and work timings exist but lack UAE-specific compliance options or employee classification-based overtime rates.

Low-Impact Limitations

These limitations have minimal impact on daily operations:

Notifications and Policy Visibility: The platform currently lacks visibility of policy tabs for employees, which prevents meaningful push notifications about attendance policy processes. My Attendance (employee view) shows only a calendar with check-in/out times and totals.

Attendance Policy Dialog UI: The current Attendance policy dialog has visibility and interaction constraints that prevent smooth user experience. The dialog's design does not adequately support intuitive user interaction or clear exit strategies.

Edge Cases

Special scenarios requiring attention:

  • Multiple violation types in same day: When an employee has both late arrival and early departure on the same day, each violation is tracked and processed separately. Deductions may apply to both if configured in the Deductions Policy.
  • Retroactive policy changes: Policy edits only affect future attendance actions, not existing records. To apply new policy rules to historical data, manually review and adjust records in Time and Pay Adjustments.
  • Split shift deductions: For employees with split shifts, deductions are halved per shift. If an employee is late for the first shift and absent for the second, both violations are recorded but calculated independently.
  • Employee transferred between departments: If an employee moves to a department with different attendance policies mid-period, the new policy applies from the transfer date forward. Historical records remain under the old policy.
  • Concurrent attendance record edits: The system lacks robust concurrent edit handling. If two users edit the same attendance record simultaneously, the last save wins. Coordinate edits to avoid data loss.
  • Overtime on public holidays: Overtime calculations for public holidays depend on the Overtime Policy configuration. Verify the 'Overtime rate for days off' setting applies to shift employees working holidays.
  • Biometric device data vs mobile app check-in: When both biometric device data and mobile app check-ins exist for the same employee on the same day, the system prioritizes the earliest check-in and latest check-out. Review Attendance Raw Data to see all sources.
  • Inactive employee mentioned in comments: The system allows mentioning inactive or off-boarded employees in attendance comments. Verify employee status before tagging to avoid confusion.
  • Scheduling extra hours for multiple shifts: Scheduling extra hours is restricted when an employee has multiple shifts on the same day. Use Time and Pay Adjustments > Scheduled tab to assign extra hours individually per shift.
  • Leave approval removes second shift: When leave is approved for a day with split shifts, the system may remove attendance records for both shifts. Verify attendance records after leave approval and manually restore if needed.

FAQs & Support

Frequently Asked Questions

Can I edit attendance records after they've been approved?

Once an attendance regularization ticket is approved, it cannot be reverted. Any further changes must be made manually in the employee's attendance record via the Edit Attendance dialog.

Why can't I see the Edit option for some attendance records?

The Edit option is only available for records that have actual attendance data (check-in/out times). Records with no attendance data cannot be edited.

What happens to attendance records when leave is approved?

The system automatically removes attendance records when an employee is on leave. For half-day leave periods, employees cannot check in during the leave portion.

Can I schedule multiple shifts for the same employee in one day?

Yes, split shifts allow up to 2 shifts per employee per day. Shifts cannot overlap and cannot be scheduled on days off or during leave periods.

Why are some employees marked as 'Unavailable' when assigning overtime policies?

Employees already assigned to another overtime policy show as 'Unavailable'. Each employee can only be assigned to one overtime policy at a time.

How far in advance can I copy and paste shifts?

You can copy shifts up to 90 days in advance. The minimum date range is one day, and shifts cannot be pasted into previous dates.

Can line managers see attendance deduction amounts?

Line managers see deduction amounts only if the super admin enables the visibility toggle in attendance settings.

What's the difference between 'Clear' and 'Delete' in the attendance row menu?

Clear resets the day's attendance data while keeping the record. Delete removes the attendance record entirely. Both options are only active when the record has attendance data.

Why can't I create leave requests from the Daily Attendance Report?

The 'Create leave request' option in the three-dot menu is available, but may only appear for certain employee records depending on their configuration and current status.

Can I customize the attendance report columns?

The Daily Report offers Department, Location, and Employment type filters, but does not support per-employee exclusion or custom column configuration. Use the Custom Report tab for date range exports.

How do I handle attendance for employees working from multiple locations?

Use the Location Report tab to view employee visits across different offices. The Multiple Visits feature tracks all location check-ins throughout the day.

What happens if I forget to publish draft shifts?

Draft shifts remain saved but employees won't receive notifications until you publish them. You can publish shifts for specific employees by applying filters before clicking Publish.
