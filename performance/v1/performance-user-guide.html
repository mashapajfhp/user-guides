What is Performance Management?

Overview

Bayzat Performance Management is a comprehensive system for managing employee performance cycles, goals, and reviews. It enables HR administrators and managers to structure performance evaluation processes, track employee development, and gather multi-directional feedback through 360° reviews.

The platform centers around Performance Cycles - structured periods during which organizations conduct goal setting, self-assessments, manager reviews, and calibration sessions. Each cycle can be customized with specific timelines, rating scales, and review types to match your organization's performance management methodology.

Key Capabilities

  • Performance Cycles: Create multi-step review cycles with configurable stages including goal setting, self-assessment, manager review, and calibration
  • Goal Management: Set, track, and approve employee goals with three-state workflow (Draft, Pending, Approved)
  • 360° Reviews (Beta): Gather comprehensive feedback from peers, direct reports, and managers through structured nomination and review processes
  • Catch-ups: Schedule and document one-on-one meetings between managers and employees
  • Custom Configuration: Define organization-specific goal types and rating scales
  • Employee Inclusion Management: Control which employees participate in each performance cycle

Who Uses This Feature

HR Administrators
Configure performance cycles, rating scales, and goal types. Manage organization-wide performance settings and monitor cycle progress across the company.
Managers
Conduct performance reviews, provide feedback on employee goals, and participate in calibration sessions. Write evaluations and track team member development.
Employees
Set personal goals, complete self-assessments, view performance feedback, and participate in 360° reviews as nominees or reviewers.

Business Value

Performance Management transforms ad-hoc review processes into structured, data-driven evaluation cycles. By centralizing goal tracking, review workflows, and feedback collection, organizations can:

  • Ensure consistent performance evaluation standards across teams
  • Maintain historical records of employee development and achievements
  • Identify high performers and development opportunities through calibration
  • Foster continuous feedback culture through catch-ups and 360° reviews
  • Align individual goals with organizational objectives

Feature Usage with Common Workflows

This section covers the essential tasks for managing Performance Cycles, Goals, and 360° Reviews in Bayzat Performance Management. Tasks are organized by workflow type to help you efficiently manage your organization's performance processes.

Performance Cycle Management

Creating a New Performance Cycle

Performance Cycles are the foundation of your performance management process. Follow these steps to create a new cycle:

  1. Navigate to Performance → Settings → Cycles
  2. Click the Add new cycle button
  3. Complete the 5-step creation wizard:
    • Step 1: Cycle details - Enter cycle name, start/end dates, and description
    • Step 2: Goal setting - Configure goal-related settings and deadlines
    • Step 3: Self-assessment - Set up employee self-review parameters
    • Step 4: Manager review - Configure manager review settings and deadlines
    • Step 5: Calibration - Set calibration meeting parameters
  4. Review your settings and click Create cycle
Best Practice: Plan your cycle dates carefully. While cycles can be edited, date modifications for active cycles require backend support intervention (see limitations section).

Viewing Performance Cycle Details

To view comprehensive information about a performance cycle:

  1. Go to Performance → Settings → Cycles
  2. Browse the cycles list showing Current and Past cycles
  3. Click on any cycle name to view its detail page

The cycle detail view displays:

  • Cycle overview with status (Current/Past)
  • Activity timeline showing all review stages
  • Employees included and excluded from the cycle
  • Rating scale configuration
  • Review types enabled (Self-review, Manager review, etc.)
  • Goal setting parameters

Editing a Performance Cycle

You can modify cycle settings for Current cycles:

  1. Navigate to Performance → Settings → Cycles
  2. Locate the cycle you want to edit (must be in Current status)
  3. Click the Edit icon (pencil) next to the cycle name
  4. Update the necessary fields in the cycle wizard
  5. Save your changes
⚠️ Date Modification Limitation

Impact: Active performance cycle dates (including self-review and manager review deadlines) cannot be modified directly through the user interface. Changes require backend intervention from Bayzat support staff, creating potential delays.

Workaround: Contact Bayzat support with specific date change requests, including cycle name and desired new dates for each review stage.

Reference: TSSD-464

Reopening a Past Cycle

If you need to reactivate a completed cycle:

  1. Go to Performance → Settings → Cycles
  2. Find the Past cycle you want to reopen
  3. Click Reopen cycle option
  4. Confirm the action

The cycle status will change from Past to Current, allowing employees and managers to access and complete reviews.

Deleting a Performance Cycle

You can delete Current cycles that are no longer needed:

  1. Navigate to Performance → Settings → Cycles
  2. Locate the cycle to delete (must be Current status)
  3. Click the Delete icon (trash) next to the cycle name
  4. Confirm the deletion in the dialog box
⚠️ Deletion Restriction: Past cycles cannot be deleted. The delete button is disabled with the tooltip "Past cycle cannot be deleted". This protects historical performance data.

Goal Management

Viewing Goals

Goals are organized into three status categories:

  1. Navigate to Performance → Goals
  2. Use the tab system to filter goals:

  3. Click on any goal to view its details

The goal detail view shows:

  • Goal title and description
  • Goal type and category

  • Progress tracking
  • Associated employees
  • Edit and Delete controls

Editing Goals

To modify an existing goal:

  1. Navigate to the goal detail page
  2. Click the Edit button
  3. Update goal information as needed
  4. Save your changes
Note: Goal editing permissions depend on the goal status and your role. Approved goals may require re-approval after editing.

Deleting Goals

To remove a goal:

  1. Open the goal detail page
  2. Click the Delete button
  3. Confirm the deletion
Caution: Deleting a goal removes it permanently and cannot be undone. Ensure you have proper authorization before deleting goals.

Business Rules & Limitations

Business Rules

The Performance Management module operates under the following business rules:

  • Cycle Status: Performance cycles can have two states: Current or Past. Only Current cycles can be edited or deleted through the UI.
  • Past Cycle Management: Past cycles cannot be deleted but can be reopened. Once reopened, they become Current cycles and can be managed normally.
  • Employee Inclusion: Administrators can include or exclude specific employees from performance cycles on a per-cycle basis.
  • Review Stage Sequencing: Performance cycles follow a structured workflow: Goal setting → Self-assessment → Manager review → Calibration. Each stage must be configured during cycle creation.
  • 360° Reviews (Beta): The 360° feedback feature is in Beta status and follows a separate workflow from standard performance cycles.
  • Goal Approval Workflow: Goals progress through three states: Draft → Pending → Approved. Goals must be approved before they can be evaluated in performance reviews.
  • Rating Scale Assignment: Each performance cycle must be associated with a rating scale. Rating scales are configured globally and applied at the cycle level.

Known Limitations

⚠️ Performance Cycle Date Modification Requires Backend Support

Impact: Administrators cannot modify active performance cycle dates (such as self-review deadlines or manager review end dates) through the user interface. Changes to cycle dates require manual backend configuration by Bayzat support staff.

Workaround: Contact Bayzat support with the specific cycle and date changes required. Support will process the request through backend configuration tools.

Reference: TSSD-464
Note: No additional product limitations have been documented for this feature at this time. As the Performance Management module evolves, particularly with the 360° Reviews Beta feature, additional constraints may be identified.

Configuration Constraints

  • Rating Scales: Rating scales must be configured before creating performance cycles. Cycles cannot be created without an assigned rating scale.
  • Goal Types: Goal types must be defined in Settings before employees can create goals. Custom goal types can be created to match organizational objectives.
  • Workflow Integration: Performance Management does not currently integrate with Bayzat Workflows. Automation of performance-related tasks is not available through the Workflows module.
  • Approval Automation: The module does not have built-in approval flow automation. Goal approvals and review completions are managed manually within the Performance interface.

Troubleshooting & Edge Cases

Common Issues and Solutions

Issue: Cannot Delete a Performance Cycle

Symptom: The delete button is disabled or shows a tooltip stating "Past cycle cannot be deleted."

Cause: Only Current cycles can be deleted. Once a cycle's end date has passed, it automatically becomes a Past cycle and deletion is disabled.

Solution:

  1. If you need to remove a Past cycle, first reopen it using the "Reopen cycle" option
  2. The cycle will return to Current status
  3. You can then delete it using the delete (trash) icon
  4. Note: Reopening a cycle may affect historical reporting data

Issue: Need to Extend Performance Cycle Deadlines

Symptom: Review deadlines are approaching but employees need more time to complete self-assessments or managers need additional time for reviews.

Cause: Cycle dates cannot be modified through the UI once the cycle is created and active.

Solution:

  1. Contact Bayzat support immediately with your request
  2. Provide the cycle name and specific dates that need to be changed
  3. Include the new target dates for each review stage
  4. Support will process the backend configuration change
  5. Allow 1-2 business days for the change to be implemented

Reference: TSSD-464

Issue: Goals Not Appearing in Performance Review

Symptom: Employee goals are not visible during the performance review process.

Possible Causes and Solutions:

  • Goal Status: Goals must be in "Approved" status to appear in reviews. Check the Goals tab and ensure goals have moved from Draft → Pending → Approved.
  • Cycle Configuration: Verify that goal setting was enabled when creating the performance cycle. If disabled, goals won't be included in the review process.
  • Employee Inclusion: Confirm the employee is included in the performance cycle. Check the cycle's employee inclusion list.

Issue: 360° Review Nominations Not Received

Symptom: Participants haven't received nomination requests for 360° feedback.

Solution:

  1. Navigate to Performance → 360° reviews Beta
  2. Locate the specific review in the list
  3. Verify the review status - it should show as "Active"
  4. Check the "Pending nominations" count on the overview dashboard
  5. Confirm participant email addresses are correct in the review setup
  6. Check if nomination emails are in spam/junk folders
  7. If issues persist, you may need to resend nominations through the review detail view

Edge Cases

Edge Case: Employee Leaves During Active Performance Cycle

Scenario: An employee included in a performance cycle is terminated or resigns before the cycle completes.

Handling:

  • The employee remains in the cycle even after termination
  • Managers should complete the review based on performance during employment
  • Alternatively, exclude the employee from the cycle using the employee inclusion/exclusion management feature
  • Historical data is preserved for reporting purposes

Edge Case: Manager Changes Mid-Cycle

Scenario: An employee's reporting manager changes while a performance cycle is in progress.

Handling:

  • Update the employee's reporting structure in HR → Employees
  • The new manager will automatically inherit review responsibilities
  • Previous manager's draft reviews are not automatically transferred
  • Consider having both managers collaborate on the final review if significant work was completed under the previous manager

Edge Case: Multiple Active Cycles

Scenario: Organization wants to run multiple performance cycles simultaneously (e.g., quarterly reviews for some departments, annual for others).

Handling:

  • Multiple Current cycles can exist simultaneously
  • Use employee inclusion/exclusion to assign employees to the appropriate cycle
  • Ensure clear communication about which employees are in which cycle
  • Be mindful of overlapping review periods which may create workload issues for managers

Edge Case: Rating Scale Changes After Cycle Creation

Scenario: Organization modifies a rating scale that's already assigned to an active performance cycle.

Impact:

  • Changes to rating scales may affect active cycles using that scale
  • In-progress reviews may show updated rating options
  • Completed reviews retain their original ratings

Best Practice: Create a new rating scale version rather than modifying an existing one if cycles are active.

Frequently Asked Questions

General Questions

Q: What's the difference between Performance Cycles and 360° Reviews?

A: Performance Cycles are comprehensive review processes that include goal setting, self-assessment, manager review, and calibration stages. They follow a structured timeline and are typically conducted annually or semi-annually. 360° Reviews (Beta) are separate feedback collection processes where an employee receives feedback from multiple sources including peers, direct reports, and managers. They can be run independently of performance cycles.

Q: Can I run multiple performance cycles at the same time?

A: Yes, you can have multiple Current cycles running simultaneously. Use the employee inclusion/exclusion feature to assign different employee groups to different cycles. This is useful for organizations with different review schedules for different departments or employee levels.

Q: How do I write effective performance reviews?

A: Follow these best practices:

  • Use simple, clear language that's easy to understand
  • Provide specific examples of work and behaviors
  • Balance constructive feedback with recognition of strengths
  • Focus on future development and growth opportunities
  • Avoid absolute terms like "always" or "never"
  • Maintain ongoing notes throughout the review period to avoid recency bias
  • Make feedback actionable with clear suggestions for improvement

Reference: Writing an effective Performance evaluation

Q: What happens to goals when an employee changes roles?

A: Goals remain associated with the employee regardless of role changes. When an employee changes positions, you should:

  • Review existing goals for continued relevance
  • Edit or archive goals that no longer apply
  • Create new goals aligned with the new role
  • Discuss the transition with the employee and new manager
Q: Can employees see each other's performance reviews?

A: No. Performance reviews are confidential and only visible to:

  • The employee being reviewed
  • The employee's direct manager
  • HR administrators with appropriate permissions
  • Senior managers involved in calibration (if applicable)

Configuration Questions

Q: How do I create custom rating scales?

A: Navigate to Performance → Settings → Rating scales. Click "Add new rating scale" and define:

  • Scale name (e.g., "5-Point Performance Scale")
  • Rating levels (e.g., 1-5 or Exceeds/Meets/Below Expectations)
  • Description for each rating level
  • Color coding for visual differentiation

Rating scales must be created before you can assign them to performance cycles.

Q: Can I change a performance cycle after it's started?

A: Limited changes are possible:

  • Employee inclusion/exclusion: Can be modified at any time
  • Cycle dates: Cannot be changed through UI - requires support request (TSSD-464)
  • Rating scale: Cannot be changed once cycle is created
  • Review stages: Cannot be added/removed after creation

Plan your cycle configuration carefully before creation.

Q: What's the recommended frequency for performance cycles?

A: Best practices vary by organization:

  • Annual cycles: Traditional approach, comprehensive reviews once per year
  • Semi-annual cycles: More frequent feedback, typically every 6 months
  • Quarterly cycles: Agile approach for fast-paced organizations

Consider your organization's size, industry, and culture when deciding. Many organizations combine formal cycles with informal check-ins using the Catch-ups feature.

Workflow and Automation Questions

Q: Can I automate performance review reminders?

A: Performance Management does not currently integrate with Bayzat Workflows. Reminders must be sent manually or through external calendar systems. Consider:

  • Setting calendar reminders for key milestone dates
  • Using the activity timeline in cycle details to track progress
  • Communicating deadlines clearly during cycle kickoff
  • Monitoring completion rates through the Reviews tab
Q: Is there an approval workflow for performance reviews?

A: Performance reviews do not have a formal approval workflow. However:

  • Goals follow a Draft → Pending → Approved workflow
  • Manager reviews are submitted directly without additional approval
  • Calibration stage allows senior leaders to review and adjust ratings
  • HR administrators can monitor review completion through the dashboard

Glossary of Terms

Glossary

360° Review
A comprehensive feedback process where an employee receives performance input from multiple sources including managers, peers, direct reports, and sometimes external stakeholders. In Bayzat, this is a Beta feature separate from standard performance cycles.
Calibration
The final stage of a performance cycle where senior leaders and HR review performance ratings across teams to ensure consistency, fairness, and alignment with organizational standards. Helps prevent rating inflation or deflation.
Catch-ups
Informal one-on-one meetings between employees and managers for ongoing feedback and development discussions. Accessible through the "My performance" tab and separate from formal performance reviews.
Current Cycle
A performance cycle that is currently active and has not yet reached its end date. Current cycles can be edited and deleted through the user interface.
Goal Setting
The first stage of a performance cycle where employees define objectives and key results for the review period. Goals progress through Draft, Pending, and Approved statuses.
Goal Type
A category or classification for goals (e.g., "Individual Development," "Team Objective," "Company Goal"). Goal types are configured in Performance Settings and help organize goals by purpose or scope.
Line Manager Review
The stage where an employee's direct manager evaluates performance, provides ratings, and writes feedback. Also called "Manager Review" or "Manager-to-Employee Review."
Nomination
In 360° reviews, the process of selecting individuals who will provide feedback about an employee. Participants can nominate peers, direct reports, or other colleagues to contribute feedback.
Past Cycle
A performance cycle that has completed and reached its end date. Past cycles cannot be deleted but can be reopened to become Current cycles again. Used for historical reference and reporting.
Performance Cycle
A structured, time-bound process for evaluating employee performance. Typically includes stages for goal setting, self-assessment, manager review, and calibration. Cycles can be annual, semi-annual, or quarterly.
Primary Entity
The main object or item managed by a feature. For Performance Management, the primary entity is the Performance Cycle.
Rating Scale
A standardized system for evaluating performance, defining the possible ratings (e.g., 1-5 or Exceeds/Meets/Below Expectations) and their meanings. Rating scales are configured globally and assigned to performance cycles.
Recency Bias
The tendency to give disproportionate weight to recent events when evaluating performance over a longer period. Best practices recommend maintaining ongoing notes throughout the review period to avoid this bias.
Reopen Cycle
The action of converting a Past cycle back to Current status, allowing it to be edited or deleted. Used when cycle dates need to be extended or corrections need to be made to completed cycles.
Review Stage
A distinct phase within a performance cycle (e.g., Goal Setting, Self-Assessment, Manager Review, Calibration). Each stage has its own timeline and required actions.
Self-Assessment
The stage where employees evaluate their own performance against goals and provide self-ratings. Completed before the manager review stage to give employees a voice in the evaluation process.
