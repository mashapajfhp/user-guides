Feature Overview

What Work Expenses Does

Work Expenses is a comprehensive expense management module within Bayzat's payroll system that enables employees to submit reimbursement requests for work-related expenditures. The feature automates the entire expense lifecycle—from initial submission with receipt uploads to final settlement through payroll processing.

Who Uses This Feature

Work Expenses serves multiple user personas across your organization:

    • Employees: Submit expense requests for travel, office supplies, mileage, and other work-related costs
    • Line Managers: Review and approve/reject expense requests from direct reports
    • Expense Managers: Oversee all company expense requests and manage approval workflows
    • Payroll Table Managers: Add approved expenses to payroll cycles for reimbursement processing
    • Transaction Processors: Manage expense settlements and payment tracking
    • Super Admins: Configure approval flows, manage categories, and oversee the entire expense system

    Key Capabilities

    • Dual Submission Methods: Create standard expense reimbursements or mileage-based reimbursements with automatic distance calculations
    • OCR Technology: Upload receipts and automatically extract expense details including amount, date, and vendor information
    • Flexible Approval Flows: Configure multi-level approval hierarchies with line manager routing and role-based permissions
    • Status Tracking: Monitor expenses through five stages: Pending Requests, Approved, Rejected, Added to Payroll, and Settled
    • Bulk Operations: Select and process multiple expense requests simultaneously
    • Comprehensive Reporting: Download expense lists as Excel files and attachments as ZIP archives
    • Activity Feed: View complete approval history and comments for each expense request
    • Multi-Currency Support: Submit expenses in various currencies with automatic conversion tracking

    Business Value

    Work Expenses delivers measurable benefits to organizations:

    • Reduced Processing Time: OCR automation and streamlined approvals eliminate manual data entry and reduce approval cycles from days to hours
    • Enhanced Compliance: Structured approval workflows ensure proper authorization and create audit trails for every expense transaction
    • Improved Employee Experience: Mobile-friendly submission and transparent status tracking empower employees with self-service capabilities
    • Financial Accuracy: Direct payroll integration eliminates manual data transfer errors and ensures timely reimbursements
    • Cost Visibility: Comprehensive reporting and categorization provide insights into expense patterns and spending trends

User Journey: End-to-End Expense Management Workflow

The Work Expenses feature supports a complete lifecycle for managing employee reimbursement requests, from initial submission through final settlement. This journey involves multiple stakeholders and progresses through five distinct stages.

Journey Overview

The expense management workflow accommodates two primary user paths:

    • Employee-Initiated Requests: Employees submit their own expense reimbursement requests through the platform
    • Admin-Initiated Requests: Administrators or managers submit expense requests on behalf of employees

    Workflow Stages

    Every expense request progresses through a defined status lifecycle:

    1. Pending Requests: Initial submission awaiting approval from designated approvers in the approval flow
    2. Approved: Request has been approved by all required approvers and is ready for payroll processing
    3. Rejected: Request was declined by one or more approvers in the approval chain
    4. Added to Payroll: Approved expense has been added to the payroll table for payment processing
    5. Settled: Payment has been processed and the expense is fully reimbursed

    Key Decision Points

    1. Submission Method Selection

    Users choose between two submission types based on their needs:

    • Expense Reimbursement: For general work-related expenses requiring receipt documentation
    • Mileage Reimbursement: For travel-related expenses calculated based on distance traveled

    2. Receipt Upload Approach

    When creating an expense request, users can:

    • Start with a receipt image (OCR technology automatically extracts expense details)
    • Start without a receipt and manually enter all expense information
    • Add or update receipts after initial submission

    3. Approval Flow Routing

    The system routes requests through a predefined approval hierarchy that may include:

    • Line Managers (first-level approval)
    • Expense Managers (specialized expense reviewers)
    • Super Admins (final approval authority)
    • Multi-level approvers (sequential or parallel approval steps)

    Critical Rule: If any approver in the approval flow rejects the request, it is immediately moved to Rejected status and the approval process stops.

    4. Payroll Processing Decision

    Once approved, authorized users decide when to add expenses to the payroll table:

    • Individual expense selection for immediate processing
    • Bulk selection for batch processing
    • Alignment with specific payroll cycles

    Typical Use Case Scenarios

    Scenario 1: Employee Travel Expense

    1. Employee attends a business conference and incurs hotel and meal expenses
    2. Employee takes photos of receipts using mobile device
    3. Employee submits expense request via mobile app or web platform
    4. OCR technology extracts amount, date, and merchant information
    5. Employee reviews auto-filled data and adds description
    6. Request routes to Line Manager for initial approval
    7. Line Manager reviews receipt and approves
    8. Request automatically moves to Expense Manager for final approval
    9. Expense Manager approves and adds to next payroll cycle
    10. Employee receives reimbursement in next payroll payment

    Scenario 2: Admin-Submitted Expense for Blue Collar Worker

    1. Blue collar employee submits paper receipt to site supervisor
    2. Supervisor forwards receipt to HR administrator
    3. HR admin navigates to employee's profile in Bayzat
    4. Admin clicks "Add Expense Reimbursement" on behalf of employee
    5. Admin uploads receipt image and system auto-fills details
    6. Admin submits request (automatically approved due to admin privileges)
    7. Request appears in Approved tab immediately
    8. Admin adds expense to payroll table for next payment cycle
    9. Employee receives reimbursement without needing platform access

    Scenario 3: Mileage Reimbursement

    1. Employee uses personal vehicle for client meetings
    2. Employee selects "Add Mileage Reimbursement" option
    3. Employee enters starting location, destination, and distance traveled
    4. System automatically calculates reimbursement amount based on configured mileage rate
    5. Employee adds client name and trip description
    6. Request routes through approval flow
    7. Approvers review trip details and approve
    8. Mileage reimbursement added to payroll and settled

Initial Setup

This section guides you through the first-time configuration of the Work Expenses feature. Complete these steps in order to prepare the system for expense submissions.

Step 1: Access Work Expenses Settings

    • Log in to Bayzat as a Super Admin
    • Click Settings in the left navigation menu
    • Locate and click Work Expenses in the settings list

You will see the Work Expenses configuration dashboard with the following sections:

  • Expense Categories
  • Approval Flows
  • Mileage Rate
  • General Settings

Step 2: Create Expense Categories

Expense categories organize and classify different types of expenses in your organization.

  1. In the Work Expenses settings, click Expense Categories
  2. Click + Add Category button
  3. Enter the following information:
    • Category Name: Descriptive name (e.g., "Travel Expenses", "Office Supplies", "Client Entertainment")
    • Description: Optional explanation of what expenses belong in this category
    • Require Receipt: Toggle ON if receipts are mandatory for this category
  4. Click Save to create the category
  5. Repeat steps 2-4 to create additional categories
Recommended Categories: Start with these common expense types: Travel & Transportation, Meals & Entertainment, Office Supplies, Communication Expenses, Professional Development, Client Expenses, Mileage Reimbursement.

Category Configuration Best Practices

  • Create 5-10 categories to balance specificity with simplicity
  • Use clear, descriptive names that employees will easily understand
  • Enable "Require Receipt" for categories involving significant amounts
  • Consider your reporting needs when structuring categories

Step 3: Configure Mileage Rate (Optional)

If your organization reimburses employees for using personal vehicles for work purposes:

  1. In Work Expenses settings, click Mileage Rate
  2. Toggle Enable Mileage Reimbursement to ON
  3. Enter the following details:
    • Rate per Unit: Amount to reimburse per kilometer or mile (e.g., 0.50 AED/km)
    • Unit Type: Select Kilometers or Miles
    • Category: Select which expense category will use this mileage rate
  4. Click Save Settings
⚠️ Limitation: Only ONE expense category can be configured with a mileage rate. If you need different rates for different vehicle types or scenarios, you will need to manage this manually or use separate expense categories with fixed amounts.

Step 4: Define Approval Flows

Approval flows determine who reviews and approves expense requests before they can be processed for payment.

  1. In Work Expenses settings, click Approval Flows
  2. Click + Create Approval Flow
  3. Configure the approval flow:
    • Flow Name: Descriptive name (e.g., "Standard Expense Approval", "Manager + Finance Approval")
    • Description: Optional explanation of when this flow applies
  4. Add approval levels:
    • Click + Add Level
    • Select approver type:
      • Line Manager: Employee's direct manager (automatically determined from employee profile)
      • Specific User: Choose a specific user by name (e.g., Finance Manager)
      • Role: Choose users with a specific role (e.g., all Expense Managers)
    • Configure level settings:
      • Approval Type: Sequential (one after another) or Parallel (all at once)
      • Required Approvers: All approvers or Any one approver (for parallel levels)
  5. Add additional levels by repeating step 4
  6. Review the approval flow sequence
  7. Click Save Approval Flow

Example Approval Flow Configurations

Simple Flow (2 levels):

  1. Level 1: Line Manager (Sequential)
  2. Level 2: Expense Manager (Sequential)

Complex Flow (3 levels with parallel approval):

  1. Level 1: Line Manager (Sequential)
  2. Level 2: Department Head OR Finance Manager (Parallel - any one approver)
  3. Level 3: Super Admin (Sequential)
Best Practice: Start with a simple 2-level approval flow (Line Manager → Expense Manager) and adjust based on your organization's needs. Complex flows can slow down reimbursement processing.

Step 5: Configure General Settings

  1. In Work Expenses settings, click General Settings
  2. Configure the following options:
    • Allow employees to submit expenses: Toggle ON to enable employee self-service submissions
    • Default currency: Select the primary currency for expense reporting
    • Enable multi-currency expenses: Toggle ON if employees travel internationally
    • Require expense description: Toggle ON to make description field mandatory
    • Enable OCR for receipts: Toggle ON to automatically extract data from receipt images
  3. Click Save Settings

Step 6: Assign User Roles and Permissions

Ensure the right people have access to manage expenses:

  1. Navigate to Settings → Role Management
  2. Assign the following roles as needed:
    • Expense Manager: Users who will approve and manage expense requests
    • Payroll Table Manager: Users who will add approved expenses to payroll
    • Transaction Processor: Users who will process expense settlements
  3. For each role:
    • Click Edit next to the role name
    • Click + Add User
    • Search for and select the user
    • Click Save

Step 7: Configure Line Manager Relationships

If your approval flow includes Line Manager approval:

  1. Navigate to People → Employees
  2. For each employee, click on their profile
  3. Click Edit in the Organization section
  4. Select the employee's Line Manager from the dropdown
  5. Click Save
  6. Repeat for all employees who will submit expense requests
⚠️ Critical: If an employee's Line Manager is not set and your approval flow requires Line Manager approval, expense requests from that employee will fail to route correctly.

Step 8: Test the Configuration

Before rolling out to all employees, validate your setup:

  1. Create a test expense request:
    • Navigate to Finance → Work Expenses
    • Click + Add Expense Reimbursement
    • Select a test employee
    • Fill in expense details with test data
    • Upload a sample receipt
    • Click Submit
  2. Verify the approval flow:
    • Check that the request appears in Pending Requests tab
    • Confirm the first approver receives a notification
    • Log in as the approver and approve the request
    • Verify the request moves to the next approval level
    • Complete the full approval cycle
  3. Test payroll integration:
    • Navigate to the Approved tab
    • Select the test expense
    • Click Add to Payroll
    • Verify the expense appears in Added to Payroll tab
    • Check that the expense is visible in the payroll table
  4. Delete the test expense after validation

Step 9: Communicate to Employees

Once configuration is complete, inform employees about the new expense submission process:
