Daily Wage Calculator

Calculate accurate daily wages for salary proration, leave encashment, and unpaid leave deductions

What is Daily Wage Calculator?

The Daily Wage Calculator is a flexible and customizable method for calculating daily wages, including salary proration, leave encashment, and unpaid leave deductions. This feature enables HR teams to accurately adjust employee compensation based on actual days worked and leave taken.

Who Uses This Feature?

Superadmin, Payroll table managers, and transaction processors use the Daily Wage Calculator to manage payroll operations. HR personnel and payroll administrators responsible for managing employee leave and salary calculations rely on this feature for daily operations.

Key Benefits

  • Accurate Calculations: Calculate daily pay by dividing total salary by number of actual working days
  • Flexible Calculation Methods: Choose from calendar days, working days, or custom days based on your company's requirements
  • Automated Deductions: Automatically generate pay deductions and reflect them on the payroll table
  • Compliance Ready: Most UAE and KSA companies use 30 custom days to align with labor law requirements
  • Holiday Consideration: Public holidays are automatically deducted from working days

7. Setup Process

Configuring Daily Wage Calculation

The Daily Wage Calculator allows you to select a calculation base (calendar days, working days, or custom days) and choose which salary components to factor into calculations.

Step-by-Step Configuration

  1. Access the Daily Wage Calculator

    Navigate to Settings → Payroll → Daily Wage Calculation (as described in Feature Discovery section)

  2. Review Configuration Options

    The configuration table displays three calculation types. Each row shows the current calculation method and salary components used.

  3. Select Calculation Basis for Salary Proration

    Click on the Salary Proration row to select your calculation basis: Custom days, Calendar days, or Working days. A modal will appear with your options.

    Calculation Basis Explained:

    • Calendar Days: Uses the actual number of days in the month (28-31 days)
    • Working Days: Excludes weekends and public holidays from the calculation
    • Custom Days: Most UAE and KSA companies use 30 custom days to align with labor law
  4. Confirm Month Length Handling

    The system automatically accounts for varying month lengths (e.g., 31 days in January, 28 days in February). When you select your calculation basis, the system will show how it handles these variations.

  5. Select Salary Components

    Choose which salary components to include in daily wage calculations. Options typically include:

    • Basic Salary Only - Uses only the employee's base salary
    • Basic Salary + Allowances - Includes base salary and all allowances

    The selection applies to all three calculation types (Salary Proration, EOS Leave Encashment, and Unpaid Leave Deduction).

  6. Save Configuration

    Click the Save button to apply your daily wage calculation settings.

    Important: If the payroll month is open and prorated salary transactions exist, you must reject existing prorated salary transactions before saving new proration settings. The system will prompt you if this action is required.

Configuring End-of-Service Leave Encashment

Super admins can configure and customize leave encashment policies for end-of-service calculations by selecting salary components and choosing a calculation method.

  1. Access EOS Configuration

    Navigate to Settings → Payroll → End of Service Eligibility → Configure

  2. Select Salary Components

    Choose between Basic Only or Basic + Allowances for salary components.

  3. Choose Calculation Method

    Select a calculation method: Calendar days, Working days, or Custom days. This determines how the daily rate is calculated for leave encashment.

  4. Configure Individual Leave Types

    Configure specific leave types for end-of-service calculations. Each leave type can have its own calculation settings.

    Note: Super admins can see which eligible paid leave-type policies will be overwritten by the global Daily Wage Calculator settings. The system displays an info banner when payroll settings take precedence.
  5. Save EOS Configuration

    Click Save to apply your end-of-service leave encashment settings.

Configuring Unpaid Leave Deductions

Unpaid leave deductions are calculated and managed on a daily basis for employee payroll. Configuration occurs at the leave policy level.

  1. Navigate to Leave Policies

    Go to Settings → Leaves → Leave Policies

  2. Add or Edit a Leave Policy

    Click Add New Policy or select an existing policy to edit. Select the Unpaid Leave option when creating or editing.

  3. Choose Calculation Method

    Select a calculation method: Basic Only or Basic + Allowances. This determines which salary components are used for daily deduction calculations.

  4. Select Day Calculation Type

    Specify the day calculation type: Calendar days, Working days, or Custom days.

  5. Review Daily Rate Formula

    The system displays the daily rate formula: Basic salary / Day calculation type / Divisor. Unpaid leave deductions must be calculated daily and set off against the employee's monthly pay.

    Important: If the global Daily Wage Calculator settings are configured, they take precedence over policy-level settings. The system will display a message indicating "Configured in daily wage calculation setting" with a link to the global configuration.
  6. Save Leave Policy

    Click Save to apply your unpaid leave deduction settings.

Common Calculation Examples

Example 1: Daily Pay for Unpaid Leave (Custom Days)

Scenario: Employee with AED 3,000 basic salary + AED 500 allowance takes 2 days unpaid leave. Company uses 30 custom days.

Calculation:

  • Total Salary: AED 3,500 (Basic + Allowances)
  • Daily Rate: AED 3,500 ÷ 30 = AED 116.67 per day
  • Unpaid Leave Deduction: AED 116.67 × 2 days = AED 233.34

Example 2: Salary Proration (Calendar Days)

Scenario: Employee joins on January 15th with AED 3,000 monthly salary. January has 31 calendar days.

Calculation:

  • Days Worked: 17 days (January 15-31)
  • Daily Rate: AED 3,000 ÷ 31 = AED 96.77 per day
  • Prorated Salary: AED 96.77 × 17 = AED 1,645.16

Example 3: Leave Encashment (Working Days)

Scenario: Employee separates with 10 remaining vacation days. Basic salary is AED 3,000. Company uses 22 working days per month.

Calculation:

  • Daily Rate: AED 3,000 ÷ 22 = AED 136.36 per day
  • Leave Encashment: AED 136.36 × 10 days = AED 1,363.60
Edge Case Alert: Public holidays must be considered and subtracted from working days when calculating daily pay. Ensure your company's public holiday calendar is up-to-date in the system before processing payroll.

Calculation Logic by Use Case

Unpaid Leave Deductions

Unpaid leave deductions must be calculated daily and set off against the employee's monthly pay. The daily rate is multiplied by the number of unpaid leave days taken.

Formula: Daily Rate × Number of Unpaid Leave Days = Total Deduction

Salary Proration

Salary proration adjusts and calculates an employee's salary proportionally based on the actual days worked, affecting their first salary and end-of-service settlement.

Formula: (Monthly Salary ÷ Total Days in Period) × Actual Days Worked = Prorated Salary

Leave Encashment

Leave encashment policies can be customized to select salary components and calculation methods for end-of-service calculations.

Formula: Daily Rate × Remaining Leave Days = Leave Encashment Amount

Troubleshooting & Edge Cases

Common Issues and Solutions

Issue: Payroll Transactions Fail After Changing Calculation Settings

Cause: When payroll month is open and prorated salary transactions exist, changes to calculation settings cannot be saved without rejecting existing transactions.

Solution:

  1. Navigate to Settings → Payroll → Daily Wage Calculation
  2. Note the changes you want to make
  3. Go to your Payroll table and locate all active (unpaid) prorated salary transactions
  4. Reject each transaction individually
  5. Return to Daily Wage Calculation settings and save your changes
  6. The system will automatically recalculate and regenerate the transactions

Issue: Daily Wage Rate Appears Incorrect

Cause: Different calculation methods can impact final wage calculations. Verify that the correct calculation base and salary components are selected.

Solution:

  1. Confirm your selected calculation base (calendar days, working days, or custom days)
  2. Verify the salary components included (Basic Only vs. Basic + Allowances)
  3. Check that public holidays are correctly configured and deducted
  4. Manually calculate: Monthly Salary ÷ Total Working Days = Daily Rate
  5. Compare your manual calculation with the system-generated rate
  6. If discrepancies exist, contact support with your calculation details

Issue: Leave Encashment Amount Doesn't Match Expected Value

Cause: Leave encashment calculations depend on configured salary components and calculation methods.

Solution:

  1. Navigate to Settings → Payroll → End of Service Eligibility
  2. Verify the salary components selected for the specific leave type
  3. Confirm the calculation method (calendar days, working days, or custom days)
  4. Check that the leave type is marked as eligible for encashment
  5. Recalculate manually: Daily Rate × Remaining Leave Days
  6. If the issue persists, review the leave policy configuration

Issue: Unpaid Leave Deduction Not Applied to Payroll

Cause: Unpaid leave deduction calculations must be properly configured in the leave policy.

Solution:

  1. Verify the leave request was approved (not just submitted)
  2. Navigate to Settings → Leaves → Leave Policies
  3. Confirm the leave type is configured as "Unpaid Leave"
  4. Check that the calculation method and day calculation type are selected
  5. Ensure the payroll month is still open (deductions apply during payroll processing)
  6. Regenerate the payroll to recalculate deductions

Issue: Salary Proration Not Reflecting in First Month

Cause: Salary proration affects the first salary and end-of-service settlement based on actual days worked.

Solution:

  1. Confirm the employee's joining date is correctly set in their profile
  2. Navigate to Settings → Payroll → Daily Wage Calculation
  3. Verify the calculation basis is set to your intended method
  4. Check the payroll table for the prorated salary transaction
  5. If the transaction shows as rejected, approve it to apply the proration
  6. Ensure the payroll month is processed to finalize the prorated amount

Edge Cases and Special Scenarios

Edge Case: February Payroll with Calendar Days

Calculation varies depending on month length (e.g., 31 days in January, 28 days in February).

Impact: When using calendar days, February will have fewer days (28 or 29), resulting in a higher daily rate compared to other months.

Mitigation: Consider using custom days (30) or working days to maintain consistent daily rates across all months.

Edge Case: Multiple Public Holidays in One Month

Public holidays are deducted from working days.

Impact: Months with multiple public holidays will have fewer working days, increasing the daily rate.

Mitigation: Ensure your public holiday calendar is accurate and up-to-date. Review payroll calculations for months with multiple holidays.

Edge Case: Employee with Partial Month Service

Scenario: Employee joins mid-month or leaves mid-month.

Impact: Salary proration adjusts compensation based on actual days worked.

Mitigation: Ensure the employee's joining/leaving date is accurately recorded. The system will automatically calculate the prorated amount based on actual days worked.

Edge Case: Changing Calculation Base Mid-Payroll Cycle

Scenario: You need to change from calendar days to working days while payroll is being processed.

Impact: Changes will affect all active (unpaid) amounts in the payroll table.

Mitigation: Complete the current payroll cycle before making calculation base changes. If changes are urgent, reject all active transactions, update settings, and regenerate payroll.

Edge Case: Leave Encashment with Mixed Leave Types

Scenario: Employee has multiple leave types with different encashment policies at end-of-service.

Impact: Different leave types can be configured with different calculation methods and salary components.

Mitigation: Configure each leave type individually in End of Service Eligibility settings. Verify the total encashment amount by summing individual leave type calculations.

Known Limitations

While the Daily Wage Calculator is fully functional, be aware of these inherent limitations:

Limitation 1: Month Length Variation

Calculation varies depending on month length (e.g., 31 days in January, 28 days in February). When using calendar days, daily rates will fluctuate monthly. Mitigation: Use custom days (30) for consistent rates.

Limitation 2: Public Holiday Dependency

Accurate daily wage calculations depend entirely on your public holiday calendar being current. If holidays are missing or incorrectly dated, calculations will be inaccurate. Mitigation: Review and update your public holiday calendar annually.

Limitation 3: Payroll Month Lock

When payroll month is open and prorated salary transactions exist, super admin must manually reject existing prorated salary transactions before saving new proration settings. This can create workflow delays if urgent configuration changes are needed. Mitigation: Plan configuration changes before opening the payroll month.

Limitation 4: Salary Component Constraints

Can choose between Basic Only or Basic + Allowances for salary components. More granular component selection (e.g., Basic + Specific Allowances) is not available. Mitigation: Configure leave policies to use the most appropriate component option for your company.

Limitation 5: Manual Transaction Rejection Required

When configuration changes impact active payroll transactions, they must be manually rejected and regenerated. Bulk rejection tools are not available. Mitigation: Batch process rejections during scheduled payroll maintenance windows.
