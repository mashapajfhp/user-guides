What is Leave Management?

Overview

Leave Management in Bayzat provides a centralized system for managing employee time off, from initial request submission through approval and balance tracking. The feature enables organizations to configure leave policies, process leave requests within defined cycles, and integrate leave data with payroll and end-of-service calculations. It supports multiple leave types and automates balance calculations while maintaining compliance with organizational policies and labor regulations.

Key Benefits

  • Automate leave request submission and approval workflows
  • Track leave balances and entitlements across multiple leave types
  • Configure leave encashment for end-of-service calculations
  • Integrate leave data with payroll processing

Product Foundation Overview

Core Concepts

Leave Cycles

Time periods that define when leave entitlements are allocated and when leave balances are calculated. Leave requests are restricted to current, last, and upcoming cycles.

Leave Types

Categories of time off defined by the organization, such as annual leave, sick leave, or unpaid leave. Each type has different entitlement rules and approval requirements.

Leave Balances

Real-time tracking of available leave entitlements for each employee across all leave types. Balances update automatically when leave is approved.

Leave Encashment

The conversion of unused leave entitlements into monetary compensation, typically calculated during end-of-service settlements.

Approval Workflows

Sequential approval processes that route leave requests through predefined organizational roles based on reporting structure.

Leave Request Status

The current state of a leave request: pending, approved, rejected, or cancelled. Status changes trigger automatic email notifications.

Related Features

  • Payroll Processing: Leave data integrates directly with payroll calculations for salary deductions and end-of-service settlements
  • Attendance Management: Leave requests affect attendance records and reporting
  • Employee Self-Service: Employees access leave balances and submit requests through their self-service portal
  • Organizational Structure: Reporting hierarchies determine leave approval routing and manager visibility
  • End of Service Calculations: Leave encashment rules directly impact final settlement amounts

Setup Process

Configuring Leave Policies (Admin)

Before employees can submit leave requests, HR administrators must configure leave policies, types, and cycles.

1

Navigate to Leave Settings

Go to Settings → Leave Policies. This opens the leave configuration page where you can manage leave types and cycles.

2

Create Leave Types

Click '+ New Leave Type' to create leave categories (e.g., Annual Leave, Sick Leave, Unpaid Leave). Define entitlement rules for each type.

3

Configure Leave Cycles

Define the leave cycle period (typically annual). Set start and end dates that determine when balances reset or carry over.

4

Set Accrual Rules

Configure how leave accrues over time. Options include monthly accrual, annual allocation, or custom accrual schedules.

5

Configure Approval Flow

Set up the approval workflow for leave requests. Approvers are determined by the organizational reporting structure.

6

Save and Publish

Review all settings and click 'Save'. The leave policy is now active for employees.

Tip: Test your leave configuration by submitting a test request to verify the approval workflow works as expected.

Approving Leave Requests (Manager Flow)

  1. Navigate to Leave → Team Leave Requests
  2. Click 'Pending' tab to view requests awaiting approval
  3. Click on a request row to open the detail panel
  4. Review the request details and team availability
  5. Click 'Approve' or 'Reject' (add reason if rejecting)

Note: Approval routing is determined by the organizational reporting structure. The system follows a strict sequential approval flow.

Modifying Leave Requests

Employees can modify pending leave requests before approval.

  1. Navigate to Leave and locate your pending request
  2. Click 'Edit' or 'Modify'
  3. Update the dates, leave type, or notes as needed
  4. Resubmit for approval

Important: Leave requests can only be modified for current, last, and upcoming leave cycles. Historical changes require backend support.

System Constraints

Approval Workflow Limitations: The system only supports predefined roles for leave approval and enforces a strict sequential approval flow without flexible routing or custom role support.

Non-Configurable Email Notifications: Administrators cannot disable or configure automatic leave expiry and other notification emails. Notifications are sent automatically.

Half-Day Leave Attendance Logging: Employees cannot record attendance during half-day leaves, and HR cannot manage these entries. Manual attendance adjustments may be required.

Cross-Cycle Leave Requests: Leave requests overlapping two leave cycles cause incorrect leave balance calculations. Split leave requests manually to avoid this issue.

Audit Trail Visibility: The platform lacks clear, user-friendly audit trails for leave request changes, deletions, and status modifications. Backend log searches are required for verification.

Edge Cases

  • Employee Transfer During Pending Leave: If an employee transfers to a new department while a leave request is pending, cancel and resubmit under the new reporting structure.
  • Leave Cycle Change During Active Leave: Leave balance may not reflect correctly in both cycles. Manual verification may be required.
  • Bulk Requests Near Cycle End: Submit leave requests with adequate time before cycle boundaries to avoid processing issues.
  • Approval Delegation: No automated delegation mechanism exists. Requests may be delayed until the approver returns.
  • Encashment Timing: Leave encashment must be configured before processing final payroll. Changes cannot be retroactively applied.

Performance Note: Employees with extensive leave history may experience slower load times. Large report generation may take several minutes.

Support Resources

Frequently Asked Questions

Can I customize the leave approval workflow to include custom roles?

No, the leave approval workflow only supports predefined system roles such as Line Manager and Department Head. Custom role assignments are not currently supported.

How do I view the complete history of changes to a leave request?

The user interface does not provide a comprehensive audit trail view. Contact the Bayzat support team for backend log searches to retrieve the complete audit trail.

What happens if I submit a leave request that spans two leave cycles?

Leave requests spanning multiple leave cycles may result in incorrect balance calculations. Submit separate leave requests for each leave cycle to ensure accurate balance tracking.

Can I disable automatic leave expiry notification emails?

No, automatic leave expiry notifications cannot be disabled or configured by administrators. These notifications are sent automatically to all affected employees.

How do I record attendance for half-day leave?

The system does not support attendance logging during half-day leave. You will need to manually adjust attendance records outside the system.

Can I modify a leave request from a previous year?

Leave request modifications are limited to current, last, and upcoming leave cycles. For changes to older requests, contact the Bayzat support team for backend assistance.

How is leave encashment calculated for end-of-service?

Leave encashment depends on your payroll settings. Navigate to Payroll Settings → End of Service Eligibility to configure salary components and calculation method. These must be configured before processing final payroll.
