What is Leave Management?

Leave Management in Bayzat provides a centralized system for managing employee time off, from initial request submission through approval and balance tracking. The feature enables organizations to configure leave policies, process leave requests within defined cycles, and integrate leave data with payroll and end-of-service calculations. It supports multiple leave types and automates balance calculations while maintaining compliance with organizational policies and labor regulations.

Key Benefits

  • Automated Leave Processing: Streamline leave request workflows from submission to approval with automated notifications and status tracking
  • Accurate Balance Management: Maintain real-time leave balance calculations across multiple leave types and cycles
  • Payroll Integration: Connect leave data directly to payroll processing for accurate salary calculations and end-of-service settlements
  • Policy Compliance: Enforce organizational leave policies and labor law requirements through configurable rules and approval workflows
  • Encashment Configuration: Define leave encashment rules for end-of-service calculations with flexible component selection

Product Foundation

Core Concepts

Leave Cycles

Time periods that define when leave entitlements are allocated and when leave balances are calculated. Leave requests and modifications are restricted to current, last, and upcoming cycles to maintain data integrity and accurate balance tracking.

Leave Types

Categories of time off defined by the organization, such as annual leave, sick leave, or unpaid leave. Each type can have different entitlement rules, approval requirements, and encashment policies configured independently.

Leave Balances

Real-time tracking of available leave entitlements for each employee across all leave types. Balances are automatically updated when leave requests are approved and when new entitlements are allocated at the start of each cycle.

Leave Encashment

The conversion of unused leave entitlements into monetary compensation, typically calculated during end-of-service settlements. Encashment rules define which salary components are included and which calculation method is used (calendar days, working days, or custom days).

Approval Workflows

Sequential approval processes that route leave requests through predefined organizational roles. The system enforces a strict linear approval flow where requests move through approvers in a fixed sequence based on reporting structure.

Leave Request Status

The current state of a leave request in the approval workflow: pending, approved, rejected, or cancelled. Status changes trigger automatic email notifications to relevant stakeholders throughout the request lifecycle.

Related Features

  • Payroll Processing: Leave data integrates directly with payroll calculations for salary deductions and end-of-service settlements
  • Attendance Management: Leave requests affect attendance records and reporting, though half-day leave attendance logging has limitations
  • Employee Self-Service: Employees access leave balances and submit requests through their self-service portal
  • Organizational Structure: Reporting hierarchies determine leave approval routing and manager visibility
  • End of Service Calculations: Leave encashment rules directly impact final settlement amounts when employees exit the organization

Core Tasks

Leave Management in Bayzat enables HR administrators and employees to handle leave requests, track balances, and manage leave policies. This section covers the essential tasks for working with leave in the system.

Note: The leave-management feature does not provide direct CRUD operations for leave items through the standard interface. Leave management is handled through specialized workflows for leave requests, policy configuration, and balance tracking.

Managing Leave Policies

HR administrators configure leave policies that govern how leave is accrued, requested, and calculated across the organization.

Configure Leave Encashment Settings

  1. Navigate to Payroll Settings
  2. Select End of Service Eligibility
  3. Choose salary components for leave encashment:
    • Basic Only - Calculate encashment based on basic salary only
    • Basic + Allowances - Include allowances in encashment calculation
  4. Select calculation method:
    • Calendar Days - Count all days including weekends
    • Working Days - Count only working days
    • Custom Days - Define custom calculation rules
  5. Configure individual leave types for end-of-service calculations
  6. Save the configuration

Leave encashment configuration determines how unused leave is calculated and paid out during end-of-service processing.

Upload Trade License for Payroll Processing

  1. Access Payroll Settings
  2. Navigate to Company Documents
  3. Select Trade License
  4. Upload the current trade license document
  5. Verify the document is accepted and linked to payroll processing

The trade license is required for certain payroll and end-of-service calculations to comply with local labor regulations.

Managing Leave Requests

Employees and managers work together to submit, review, and approve leave requests within the system's constraints.

Submit a Leave Request

  1. Navigate to Leave section in the employee portal
  2. Click Request Leave
  3. Select the leave type from available options
  4. Choose start and end dates for the leave period
  5. Add any required notes or documentation
  6. Submit the request for approval

⚠️ Cross-Cycle Leave Requests Not Supported

Impact: Leave requests overlapping two leave cycles cause incorrect leave balance calculations and require manual intervention.

Workaround: Split leave requests manually or avoid cross-cycle leave requests.

Reference: AV-755

Modify an Existing Leave Request

  1. Navigate to Leave section
  2. Locate the leave request to modify
  3. Click Edit or Modify
  4. Update the dates, leave type, or notes as needed
  5. Resubmit for approval if required

Restricted leave request modifications for historical leave cycles: Users can only modify leave requests for current, last, and upcoming leave cycles. Plan leave modifications within allowed cycles; historical changes require backend support. (AV-318)

View Leave Balance

  1. Access the Leave dashboard
  2. Review current leave balances by leave type
  3. Check accrual rates and upcoming accruals
  4. View leave history and pending requests

Leave balances are calculated based on the configured leave policies and accrual rules for each leave type.

Approval Workflows

Leave requests follow a sequential approval process based on organizational hierarchy and configured approval roles.

Review and Approve Leave Requests (Manager)

  1. Navigate to Approvals or Leave Requests
  2. Review pending leave requests from team members
  3. Check leave balance availability
  4. Approve or reject the request with comments
  5. System notifies the employee of the decision

Known Limitations

Audit Trail Visibility: The system does not provide a user-friendly interface for viewing complete audit trails of leave request changes, deletions, and status modifications. For compliance or investigation purposes, backend log searches are required.

Cross-Cycle Leave Requests: Leave requests that span multiple leave cycles may result in incorrect balance calculations. Always submit separate requests for each leave cycle to ensure accurate balance tracking.

Workflow Flexibility: The approval workflow system does not support flexible routing, parallel approvals, or custom role assignments. All approvals follow a strict sequential flow based on predefined system roles.

Edge Cases

  • Employee Transfer During Pending Leave: If an employee transfers to a new department or reporting manager while a leave request is pending, the approval may become stuck. The leave request should be cancelled and resubmitted under the new reporting structure.
  • Leave Cycle Change During Active Leave: If the leave cycle changes while an employee is on approved leave, the leave balance may not reflect correctly in both cycles. Manual verification and adjustment may be required.
  • Bulk Leave Requests Near Cycle End: Submitting multiple leave requests near the end of a leave cycle may cause processing delays or balance calculation issues. Submit leave requests with adequate time before cycle boundaries.
  • Retroactive Leave Requests: Requesting leave for dates in the past (beyond the current cycle) is restricted. For legitimate retroactive leave needs, contact support for backend assistance.
  • Partial Day Leave with Attendance: If an employee takes a half-day leave and needs to record attendance for the working portion, manual attendance tracking outside the system is required.
  • Approval Delegation: When an approver is unavailable (vacation, sick leave), there is no automated delegation mechanism. Approval requests may be delayed until the approver returns or manual reassignment occurs.
  • Leave Encashment Calculation Timing: Leave encashment calculations for end-of-service must be configured before processing final payroll. Changes to encashment rules after payroll processing cannot be retroactively applied.

Performance Considerations

  • Large Leave History: Employees with extensive leave history (multiple years) may experience slower load times when viewing leave balance details
  • Bulk Approval Processing: Approving large batches of leave requests simultaneously may cause temporary delays in notification delivery
  • Report Generation: Generating leave reports for large organizations or extended date ranges may take several minutes to process

Frequently Asked Questions

Can I customize the leave approval workflow to include custom roles?

No, the leave approval workflow only supports predefined system roles such as Line Manager and Department Head. Custom role assignments are not currently supported. Use the predefined roles and ensure your organizational structure aligns with these role definitions.

How do I view the complete history of changes to a leave request?

The user interface does not provide a comprehensive audit trail view. For detailed change history, including deletions and status modifications, contact the Bayzat support team who can perform backend log searches to retrieve the complete audit trail.

What happens if an employee submits a leave request that spans two leave cycles?

Leave requests spanning multiple leave cycles may result in incorrect balance calculations and require manual intervention. To avoid this issue, submit separate leave requests for each leave cycle. If a cross-cycle request has already been submitted, cancel it and create two separate requests.

Can I disable automatic leave expiry notification emails?

No, automatic leave expiry notifications and other system emails cannot be disabled or configured by administrators. These notifications are sent automatically to all affected employees and cannot be customized.

How do I record attendance for an employee on half-day leave?

The system does not support attendance logging during half-day leave. You will need to manually adjust attendance records outside the system or consider using full-day leave allocation for attendance tracking purposes.

Can I modify a leave request from a previous year?

Leave request modifications are limited to current, last, and upcoming leave cycles. For changes to leave requests in older cycles, contact the Bayzat support team for backend assistance.

What happens to pending leave approvals when an employee's manager changes?

Approval routing is tied to the reporting structure at the time of submission. If the approver changes roles or the employee transfers, the approval may become stuck. Cancel the pending leave request and resubmit it under the new reporting structure, or contact support to manually reassign the approval.

How is leave encashment calculated for end-of-service?

Leave encashment calculation depends on your payroll settings configuration. Navigate to Payroll Settings → End of Service eligibility to configure salary components (Basic Only or Basic + Allowances) and calculation method (calendar days, working days, or custom days). These settings must be configured before processing final payroll.
