Daily Wage Calculator

Calculate accurate daily wages for salary proration, leave encashment, and unpaid leave deductions

What is Daily Wage Calculator?

The Daily Wage Calculator is a flexible and customizable method for calculating daily wages, including salary proration, leave encashment, and unpaid leave deductions. This feature enables HR teams and payroll administrators to ensure accurate compensation calculations based on actual days worked and leave policies.

Who Uses This Feature?

Primary users include Superadmin, Payroll table managers, and transaction processors, along with HR personnel and payroll administrators responsible for managing employee leave and salary calculations.

Key Benefits

  • Flexible Calculation Methods: Choose from calendar days, working days, or custom days to align with your company's labor law requirements.
  • Accurate Deductions: Automatic pay deduction reflected on the payroll table with calculated daily pay rate for unpaid leave.
  • Compliance-Ready: Most UAE and KSA companies use 30 custom days to align with labor law.
  • Customizable Components: Select which salary components (basic salary, allowances, or both) factor into daily wage calculations.

Product Foundation Overview

Business Problem Solved

Calculate and deduct daily pay for employees taking unpaid leave by determining the appropriate salary reduction based on the number of working or calendar days. Eliminates manual calculation errors and ensures consistent payroll processing.

Key Benefits & Impact

  • Accurate salary proration for new joiners and departing employees
  • Automated leave encashment calculations for end-of-service
  • Reduced payroll processing time and human error
  • Compliance with UAE and KSA labor regulations

How It Works (Brief)

Access Daily Wage Calculator from Settings → Payroll → Daily Wage Calculation, select calculation base (calendar days, working days, or custom days), and choose salary components to factor into calculations. The system then automatically calculates daily wage rates.

7. Setup Process

Configuring Daily Wage Calculation

The Daily Wage Calculator provides a flexible and customizable method for calculating daily wages, including salary proration, leave encashment, and unpaid leave deductions.

Step 1: Select Calculation Base

  1. Navigate to Settings → Payroll → Daily Wage Calculation
  2. Locate the Calculation Base section
  3. Choose one of the following options:
    • Calendar Days: Uses 30 days as the standard month
    • Working Days: Excludes weekends and public holidays
    • Custom Days: Most UAE and KSA companies use 30 custom days to align with labor law
  4. Click to select your preferred calculation method

Step 2: Choose Salary Components

  1. In the same Daily Wage Calculation settings, locate Salary Components
  2. Select which salary elements to include in daily wage calculations:
    • Basic Only: Uses only the basic salary
    • Basic + Allowances: Includes basic salary and all allowances
  3. This selection affects all daily wage calculations (proration, leave encashment, unpaid leave deductions)
Important: Changes will affect all active (unpaid) amounts in the payroll table. Review existing payroll transactions before making changes.

Step 3: Configure Public Holidays

  1. Ensure your company's public holiday calendar is configured
  2. Public holidays are deducted from working days when calculating daily pay
  3. Navigate to Settings → Holidays to add or update public holidays for your organization
  4. Save your holiday configuration

Step 4: Save Configuration

  1. Review all selected options (calculation base and salary components)
  2. Click Save button at the bottom of the form
  3. System will confirm the configuration has been saved
Payroll Month Open: When payroll month is open and prorated salary transactions exist, super admin must manually reject and resave transactions after updating calculation settings.

Configuring Unpaid Leave Deductions

Unpaid leave deductions must be calculated daily and set off against the employee's monthly pay.

  1. Navigate to Settings → Leaves → Leave Policies
  2. Click Add New Policy or edit an existing policy
  3. Select Unpaid Leave as the leave type
  4. Choose the Calculation Method:
    • Basic Only
    • Basic + Allowances
  5. Select the Day Calculation Type:
    • Calendar Days
    • Working Days
    • Custom Days
  6. Click Save to apply the unpaid leave deduction configuration

Configuring Leave Encashment for End-of-Service

  1. Navigate to Settings → Payroll → End of Service Eligibility
  2. Click Configure for the leave type you want to set up (e.g., Vacation)
  3. Select Salary Components for leave encashment:
    • Basic Only
    • Basic + Allowances
  4. Choose the Calculation Method:
    • Calendar Days
    • Working Days
    • Custom Days
  5. Review which leave type policies will be affected
  6. Click Save to apply the configuration
Configuration Tip: Super admins can see which eligible paid leave-type policies will be overwritten when configuring leave encashment settings.

Understanding Daily Pay Calculations

Daily pay is calculated by dividing total salary by number of actual working days.

Calculation Formula

Daily Pay = Total Monthly Salary ÷ Number of Working Days

Example (using 22 working days in a month):

  • Employee monthly salary: AED 3,300
  • Working days in month: 22 days
  • Daily pay rate: AED 3,300 ÷ 22 = AED 150 per day
  • If employee takes 2 days unpaid leave: 2 × AED 150 = AED 300 deduction
Important Calculation Rules:
  • Public holidays are deducted from working days
  • Can use calendar days (30 days) or working days (e.g., 22 days)
  • Calculation varies depending on month length (e.g., 31 days in January, 28 days in February)

Viewing Calculated Daily Wages

  1. Navigate to Payroll → Payroll Table
  2. Select the relevant payroll month
  3. View employee records with calculated daily wage deductions
  4. Daily wage calculations are displayed in the salary breakdown section
  5. Verify calculations match your company's leave and proration policies

Edge Cases & Troubleshooting

February Calculations: Daily rate calculations for February must account for the month having 28 days (or 29 in leap years). The system automatically adjusts calculations based on the actual calendar month.
Public Holiday Handling: Public holidays must be considered and subtracted from working days when calculating daily pay. Ensure your public holiday calendar is up-to-date before processing payroll.
Stored Daily Rate: The calculated daily rate is stored in the system and used consistently for all wage calculations throughout the payroll month. Changes to calculation settings only affect future payroll periods.

Calculation Logic by Use Case

Unpaid Leave Deductions

Unpaid leave deductions must be calculated daily and set off against the employee's monthly pay.

Daily Rate = Total Salary ÷ Number of Working Days
Deduction = Daily Rate × Number of Unpaid Leave Days

Salary Proration

Salary is adjusted proportionally based on actual days worked, affecting first salary and end-of-service settlement.

Prorated Salary = (Total Salary ÷ Total Days in Period) × Actual Days Worked

Leave Encashment

Leave encashment is calculated using the same daily wage methodology as unpaid leave deductions.

Encashment Amount = Daily Rate × Number of Eligible Leave Days

System Constraints

  • Changes to calculation base will affect all active (unpaid) amounts in the payroll table.
  • Super admins can see which eligible paid leave-type policies will be overwritten when updating leave encashment configuration.
  • Daily wage calculations are applied automatically once configured; manual overrides are not available at the transaction level.
  • Calculation settings are company-wide; individual employee exceptions require separate leave policy configurations.

Troubleshooting & Edge Cases

Common Issues and Solutions

Issue: Payroll Month is Open and Cannot Update Calculation Settings

Cause: When the payroll month is open with existing prorated salary transactions, the system prevents direct updates to proration settings.

Solution:

  1. Navigate to your payroll table
  2. Identify all active (unpaid) prorated salary transactions
  3. Reject each transaction individually
  4. Update your Daily Wage Calculation settings in Settings > Payroll > Daily Wage Calculation
  5. Resave the transactions with the new calculation basis
  6. Confirm the payroll month reflects updated amounts

Issue: Daily Wage Rate Seems Incorrect

Cause: Mismatch between selected calculation base and actual working days, or public holidays not properly configured.

Solution:

  1. Verify your calculation base setting: Settings > Payroll > Daily Wage Calculation
  2. Confirm the number of working days for the month (should be 22 for most months if using working days)
  3. Check that public holidays are correctly entered in your company's holiday calendar
  4. Recalculate: Daily Rate = Total Salary ÷ (Working Days - Public Holidays)
  5. If using custom days, verify the custom day value matches your labor agreement

Issue: Leave Encashment Amount Differs from Expected Value

Cause: Salary components selected for encashment may differ from unpaid leave deduction settings, or calculation base mismatch.

Solution:

  1. Navigate to Settings > Payroll > End of Service Eligibility
  2. Verify salary components are set correctly (Basic Only vs. Basic + Allowances)
  3. Confirm the calculation method matches your Daily Wage Calculation settings
  4. Check that the leave type is configured as eligible for encashment
  5. Recalculate manually: Daily Rate × Eligible Leave Days = Encashment Amount

Issue: Unpaid Leave Deduction Not Appearing in Payroll

Cause: Leave policy may not be configured for unpaid leave deductions, or leave request not properly approved.

Solution:

  1. Verify the leave policy is configured as "Unpaid Leave" in Settings > Leaves > Leave Policies
  2. Confirm the unpaid leave deduction calculation method is selected (Basic Only or Basic + Allowances)
  3. Check that the leave request has been approved by the appropriate manager
  4. Verify the leave dates fall within the current payroll month
  5. Confirm the payroll month is still open (not finalized)

Issue: Salary Proration Not Applied to First Salary

Cause: Salary proration setting may be disabled, or employee join date not properly configured.

Solution:

  1. Navigate to Settings > Payroll > Daily Wage Calculation
  2. Confirm a calculation base is selected (calendar days, working days, or custom days)
  3. Verify the employee's join date is correctly entered in their profile
  4. Check that the first salary transaction is still in "unpaid" status
  5. If settings were just changed, reject and resave the salary transaction

Edge Cases and Special Scenarios

Edge Case: February Payroll with Different Day Count

Calculation varies depending on month length (e.g., 31 days in January, 28 days in February).

Impact: If using calendar days, February will have fewer days, resulting in a higher daily wage rate.

Mitigation: Use custom days (30) to maintain consistent daily rates across all months, which aligns with most UAE and KSA labor practices.

Edge Case: Multiple Public Holidays in One Month

Public holidays are deducted from working days.

Impact: Months with multiple public holidays will have fewer working days, increasing the daily wage rate and reducing total deductions for unpaid leave.

Mitigation: Ensure your public holiday calendar is accurately maintained and reviewed annually before the new calendar year.

Edge Case: Employee with Partial Month Service

Scenario: Employee joins mid-month or leaves before month-end.

Impact: Salary proration automatically adjusts compensation based on actual days worked.

Mitigation: Ensure salary proration is enabled in your Daily Wage Calculation settings. The system will automatically calculate the prorated amount based on the employee's actual service days.

Edge Case: Changing Calculation Base Mid-Year

Scenario: Company decides to switch from calendar days to custom days calculation.

Impact: Changes will affect all active (unpaid) amounts in the payroll table.

Mitigation: Make calculation base changes only when the payroll month is closed. If changes are needed during an open month, reject all affected transactions, update settings, and resave transactions with new calculations.

Edge Case: Leave Policy Overwrite During End-of-Service Configuration

Super admins can see which eligible paid leave-type policies will be overwritten when updating leave encashment configuration.

Impact: Updating end-of-service eligibility settings may change how specific leave types are encashed.

Mitigation: Review the list of affected leave policies before saving changes. Document any intentional policy changes for audit purposes.

Feature Gaps and Known Limitations

Current Limitations

Limited Calculation Flexibility

The Daily Wage Calculator supports three calculation bases (calendar days, working days, custom days) but does not allow for employee-specific calculation methods. All employees must use the same company-wide calculation base.

Workaround: Create separate leave policies with different calculation methods if you need to apply different daily wage calculations to different employee groups.

Manual Transaction Management Required

When payroll month is open with existing prorated salary transactions, manual rejection and resaving of transactions is required to apply new calculation settings.

Impact: Changing calculation settings during an open payroll month requires additional manual steps and cannot be automated.

No Mid-Month Calculation Changes

Calculation base changes apply to the entire month and cannot be applied retroactively to transactions already processed in the same payroll period.

Workaround: Plan calculation changes for closed payroll months or use the manual transaction rejection process during open months.

Public Holiday Calendar Dependency

Accurate daily wage calculations depend on maintaining a current public holiday calendar. Missing or incorrect holidays will result in incorrect daily wage rates.

Mitigation: Establish a process to review and update the public holiday calendar annually before the new calendar year begins.

Limited Salary Component Options

Daily wage calculations support only two salary component options: Basic Only or Basic + Allowances. Custom component combinations are not available.

Workaround: Configure allowances as either included or excluded; partial component inclusion requires manual calculation outside the system.
