What is Employee Structures?

Overview

Employee Structures organize employees into groups for policy assignment and data access control. Navigate to Settings > Company > Employee Structures to create structures with automatic or manual grouping.

Create structures, define groups using criteria or manual selection, and assign policies to groups instead of individual employees.

Key Benefits

  • Automatic policy assignment based on employee attributes eliminates manual configuration
  • Consistent access control across leave policies, loan policies, and custom roles
  • Bulk management reduces administrative overhead when policies change
  • Audit trail shows which employees belong to which groups and why

Product Foundation Overview

How Employee Structures Fits

Employee Structures is a configuration feature that creates custom groupings of employees based on organizational criteria. These groups serve as access control boundaries for role-based permissions and policy assignments.

Mental model: Employee Structure (container) ‚Üí Groups employees by criteria ‚Üí Assigned to Roles/Policies ‚Üí Controls what admins see and what policies apply

Structures impact role visibility (which employees an admin can manage) and policy assignment (which employees receive specific leave, loan, or other policies).

Key Decisions Before Setup

Answer these questions before creating Employee Structures:

  • What access boundaries do you need? ‚Äî Determine which groups of employees should be visible to specific admin roles (e.g., by department, location, or seniority)
  • Manual or automatic grouping? ‚Äî Decide whether to manually select employees or use system fields (department, location, etc.) for automatic assignment
  • Which policies will reference this structure? ‚Äî Identify which leave policies, loan policies, or other settings need group-level assignment
  • How many groups are needed? ‚Äî Each structure requires at least two groups to function properly

Feature Discovery

Navigation Paths

Employee Structures is accessed through the Company Settings module.

Settings > Company > Employee Structures

This path leads to the main Employee Structures management page where you can create, view, edit, and delete structures.

From the Employee Structures list, you can:

  • View all existing structures and their group counts
  • Create new structures using the "Add New" button
  • Edit or delete existing structures using the action icons
  • View detailed group membership by clicking "View Groups"

Initial Setup

Employee Structures must be configured before they can be used to organize employees or control data access. This section covers first-time setup for admins.

Before You Begin

Ensure you have Super Admin access and have completed the Role Management setup if you plan to restrict data access by group.

Access the Configuration Page

Settings ‚Üí Company ‚Üí Employee Structures

This page displays all existing structures and allows you to create new ones.

Create Your First Structure

Click Add New to start the structure creation wizard.

Step 1: Name and Describe the Structure

  • Enter a descriptive name (e.g., "Regional Teams", "Department Structure")
  • Add a description explaining the purpose of this structure
  • Click Next

Step 2: Add Groups

Define at least two groups within the structure. Each group represents a category of employees.

Choose a grouping method:

Option A: Manual Grouping
  • Click Add Group
  • Enter group name and description
  • Select employees individually from the list
  • Click Save
  • Repeat until you have at least two groups
Option B: Automatic Grouping
  • Click Add Group and select Automatic
  • Enter group name
  • Select criteria from available employee data points (Department, Location, Job Title, etc.)
  • Choose specific values for each criterion
  • The system automatically assigns employees matching all selected criteria
  • Click Save

Minimum Requirement: You must create at least two groups before proceeding. The system will display an error if you attempt to continue with fewer than two groups.

Step 3: Review and Confirm

  • Review structure name, description, and all groups
  • Verify employee counts for each group
  • Click Create Structure to finalize

Configure Data Access Restrictions (Optional)

After creating a structure, you can restrict which admins see which employee groups in payroll and other sensitive modules.

Settings ‚Üí Role Management ‚Üí [Select Custom Role]
  • Open the custom role you want to restrict
  • Scroll to Employee Structure Restrictions
  • Select the structure from the dropdown
  • Choose which groups this role can view
  • Save the role

Users assigned this role will only see employees in the selected groups when accessing payroll, reports, and other restricted data.

Next Steps

  • Assign the structure to Leave Policies for group-specific leave rules
  • Use the structure in custom roles to control data visibility
  • Review group assignments periodically as your organization changes

Key Tasks

Task: Create Employee Structure

Set up a custom organizational grouping to control data access and policy assignments.

Subtask: Create Automatic Structure

  1. Go to Settings > Company > Employee Structures
  2. Click Add New
  3. Enter structure name and description
  4. Select Automatic grouping method
  5. Click Next
  1. Click Add Group to create first group
  2. Enter group name
  3. Click Add Criteria and select data point (Department, Location, Job Title, etc.)
  4. Select matching values for the criteria
  5. Save group
  6. Repeat to create at least one more group (minimum two required)
  1. Review all groups
  2. Click Next
  3. Review structure summary
  4. Click Create

Expected outcome: Structure appears in the Employee Structures list. Employees automatically assigned to groups based on criteria. Structure available for selection in Role Management and policy assignment screens.

Subtask: Create Manual Structure

  1. Go to Settings > Company > Employee Structures
  2. Click Add New
  3. Enter structure name and description
  4. Select Manual grouping method
  5. Click Next
  1. Click Add Group
  2. Enter group name
  3. Select employees from the list
  4. Save group
  5. Create at least one more group (minimum two required)
  6. Review all groups
  7. Click Next
  8. Review structure summary
  9. Click Create

Expected outcome: Structure created with manually assigned employees. Changes to employee data do not affect group membership. Structure available for role restrictions and policy assignments.

Task: View and Manage Structures

Review existing structures, check group membership, and verify employee assignments.

Subtask: View Structure Details

  1. Go to Settings > Company > Employee Structures
  2. Locate structure in the list
  3. Click View Groups to see all groups in the structure
  1. Review group names and criteria
  2. Click on a group to see assigned employees

Expected outcome: See complete structure configuration including all groups and their members.

Subtask: Check Employee Assignments

  1. From Employee Structures list, click View Employees & Roles
  2. Review which employees are in which groups
  3. Verify structure assignments are correct

Expected outcome: Confirm employees are grouped correctly. For automatic structures, verify criteria are working as intended. For manual structures, verify manual assignments are up to date.

Task: Edit Employee Structure

Modify structure name, description, or group configuration after creation.

Subtask: Edit Structure Details

  1. Go to Settings > Company > Employee Structures
  2. Click Edit icon on the structure row
  3. Update structure name or description
  4. Click Next to proceed to groups

Grouping method cannot be changed: You cannot switch between Automatic and Manual after creation. Create a new structure if you need a different grouping method.

Subtask: Edit Automatic Groups

  1. On the Groups step, click Edit on a group
  2. Modify group name if needed
  3. Add or remove criteria
  4. Change selected values for existing criteria
  5. Save group
  6. Review changes
  7. Click Next, then Update

Expected outcome: Employees automatically reassigned based on updated criteria. Changes take effect immediately.

Subtask: Edit Manual Groups

  1. On the Groups step, click Edit on a group
  2. Modify group name if needed
  3. Add or remove employees from the selection list
  4. Save group
  5. Review changes
  6. Click Next, then Update

Expected outcome: Employee assignments updated per manual changes. No automatic reassignment occurs.

Task: Delete Employee Structure

Remove a structure that is no longer needed.

Subtask: Delete Structure

  1. Go to Settings > Company > Employee Structures
  2. Click Delete icon on the structure row
  3. Review the warning about role restrictions and policy assignments
  4. Click Delete to confirm

Cannot delete if in use: You cannot delete a structure that is currently used in Role Management for access restrictions or assigned to policies. Remove all references first, then delete.

Expected outcome: Structure removed from the list. Employees no longer grouped under this structure. Structure no longer available for selection in role or policy screens.

Task: Assign Structure to Leave Policy

Use structure groups to automatically assign leave policies to employees.

Subtask: Enable Automatic Assignment

  1. Go to Settings > Leaves > Leave Policies
  2. Click Edit on a leave policy
  3. Scroll to Employee Assignment section
  4. Toggle Enable Automatic Assignment ON
  1. Select an Employee Structure from the dropdown
  1. Review the groups in the mapping table
  2. Select which groups should receive this policy
  3. Save the policy

Expected outcome: Employees in selected groups automatically assigned the leave policy. New employees joining those groups receive the policy automatically. Manual assignments no longer needed for these groups.

For detailed leave policy configuration, see the Leave Policies guide.

Task: Restrict Role Access by Structure

Limit which employees a custom role can view or manage using structure groups.

Subtask: Configure Role Restrictions

  1. Go to Settings > Role Management
  2. Click Edit on a custom role
  3. Scroll to Access Restrictions section
  1. Select an Employee Structure
  2. Choose which groups this role can access
  3. Save the role

Expected outcome: Users with this role can only view and manage employees in the selected groups. Payroll data, attendance records, and other sensitive information restricted to assigned groups only. Admins cannot see employees outside their assigned groups.

For detailed role configuration, see the Role Management guide.

Business Rules & Limitations

Known Limitations

Employee Structures currently has no documented limitations or edge cases from validation testing. The feature operates as designed with standard configuration rules.

Configuration Rules:

  • Structures must contain at least two groups to be valid
  • Automatic grouping requires predefined data points (Department, Location, Job Title, etc.)
  • Manual groups require explicit employee selection
  • Each employee can belong to only one group within a structure

Policy Assignment Constraints

Constraint Description Impact
Structure Selection Only one structure can be selected per policy at a time Policies using structures cannot mix manual and structure-based assignment
Group Mapping Each group within a structure must be explicitly mapped to policy configuration Unmapped groups will not receive policy assignment
Role Restrictions Custom roles can restrict data access to specific groups within a structure Admins with restricted roles see only employees in their assigned groups

Related Feature Limitations

The following limitations from related features may affect Employee Structures usage:

Leave Policy Sorting (Medium): Leave policies are automatically sorted by update time in the employee view. Manual prioritization or pinning of policies is not available. See Leave Policies guide for policy management details.

Deleted Policy Recovery (Medium): Deleted leave policies cannot be restored through self-service. Recovery requires manual backend intervention by support teams. See Leave Policies guide for policy lifecycle management.

Carry-Over Modifications (Medium): Modifying carry-over allowances for individual leave cycles requires complete policy reconfiguration. Direct edits to carry-over limits are not supported through the UI. See Leave Policies guide for configuration details.

Policy Change Rollback (High): The platform lacks a native rollback mechanism for policy changes. Modifications can unintentionally unassign employees and erase historical data without recovery options. Always verify assignments before saving policy changes.

Data Consistency Rules

  • Automatic groups update dynamically when employee data changes (e.g., department transfer)
  • Manual groups require explicit updates when employee assignments change
  • Structure deletion requires removing all policy mappings first
  • Group deletion is blocked if the group is mapped to active policies

Common Issues

Automatic group shows no employees Selected data point criteria do not match any current employees Review employee profiles to ensure the data point field is populated, or adjust criteria to match existing data

Edge Cases

  • New Employee Onboarding: Employees added to the system after structure creation will not automatically appear in manual groups. Update manual groups explicitly or use automatic grouping based on employee data.
  • Employee Data Changes: When an employee's department, location, or job title changes, automatic groups update immediately. Manual groups require manual reassignment.
  • Policy Overlap: If an employee is assigned to multiple policies (one manual, one structure-based), both policies apply. Ensure policy logic does not conflict.
  • Structure Editing: Modifying a structure's groups (adding, removing, or changing criteria) does not automatically update existing policy mappings. Review and update policy group mappings after structure changes.
  • Bulk Employee Import: When importing employees in bulk, automatic groups update dynamically. Manual groups require post-import assignment.
  • Role Restrictions: Admins with role restrictions see only employees in their assigned groups. This affects reporting, policy assignment visibility, and employee management tasks.

Data Validation Rules

Structure Name: Must be unique across the organization. Duplicate names are not allowed.

Group Name: Must be unique within a structure. Duplicate group names within the same structure are not allowed.

Automatic Criteria: At least one data point must be selected. Empty criteria are not valid.

Manual Employee Selection: At least one employee must be selected to create a manual group.

Performance Considerations

  • Automatic groups with broad criteria (e.g., "All Departments") may include large employee populations. Consider splitting into smaller, more targeted groups for easier management.
  • Structures with many groups (10+) may slow policy configuration screens. Consolidate groups where possible.
  • Manual groups require periodic review to ensure employee assignments remain accurate after organizational changes.

FAQs & Support

Frequently Asked Questions

Can I assign an employee to multiple employee structures?

No. Each employee can only belong to one employee structure at a time. This ensures clear data access boundaries for admins with restricted permissions.

What happens if I delete an employee structure that's assigned to policies?

Deleting a structure removes the group-to-policy mappings. Employees previously in that structure will lose automatic policy assignments unless you reassign them manually or create a new structure.

How do automatic employee groups differ from manual groups?

Automatic groups use system criteria (department, location, job title) to dynamically assign employees. Manual groups require you to select each employee individually. Automatic groups update as employee data changes.

Can I change a structure from automatic to manual grouping after creation?

No. The grouping method is set during structure creation and cannot be changed. You must create a new structure with the desired grouping method.

Why must I create at least two groups per structure?

Structures require a minimum of two groups to enable meaningful segmentation. Single-group structures do not provide the access control or policy differentiation that the feature is designed for.

How do employee structures interact with role-based permissions?

Employee structures control which employee data a custom role can view. Configure this in Role Management by assigning specific groups to each role.

Can I map the same group to multiple policies?

Yes. A single employee group can be mapped to multiple policies (leave, loan, etc.). This allows you to apply a consistent policy set to a specific employee segment.

What happens to existing policy assignments when I enable automatic assignment?

Enabling automatic assignment in a policy (e.g., Leave Policies) replaces manual employee assignments with structure-based mappings. Employees are assigned based on their group membership.
