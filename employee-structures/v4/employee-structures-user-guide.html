What is Employee Structures?

Overview

Employee Structures organize employees into groups using automatic criteria (department, office, nationality) or manual selection. Find it under Settings > Company > Employee Structures.

Create structures with multiple groups, then assign them to restrictable admin roles or use them to auto-assign leave policies and loan policies to specific employee segments.

Key Benefits

  • Eliminate manual role permission updates when org structure changes
  • Reduce leave policy assignment errors by automating group-based rules
  • Control payroll data visibility without individual employee-level configuration
  • Scale admin access controls across departments, offices, or custom segments

Product Foundation Overview

How Employee Structures Fits

Employee Structures are configuration templates that define how employees are organized into groups for access control and policy assignment. They sit in Company Settings and act as a foundational layer for role restrictions and automatic policy assignments.

Mental model: Employee Structure (template) ‚Üí Groups created (automatic or manual) ‚Üí Assigned to restrictable roles or policies ‚Üí Limits admin access scope or automates policy distribution ‚Üí Scales permissions and policy management

Once created, structures control which employees specific admins can view and which employees automatically receive certain policies.

Key Decisions Before Setup

Answer these questions before creating Employee Structures:

  • Will groups change frequently? ‚Äî Choose Automatic grouping if employee attributes (department, office, nationality) drive group membership; choose Manual if you need fixed, hand-picked groups.
  • What data points define your groups? ‚Äî Automatic grouping supports 50+ criteria including Department, Office, Nationality, Job Title, Employment Type, and Custom Fields with AND/OR logic.
  • Which roles need restricted access? ‚Äî Identify restrictable roles (People Manager, Attendance Manager, Expense Manager, Leaves Manager, Payroll table admin, Performance Manager, Transaction processing admin, Adjustment manager) that will use these groups to limit their view.
  • Which policies will use automatic assignment? ‚Äî Structures enable automatic assignment for Leave Policies, Employee Loan Policies, and Leave Salary Policies (not Allowances, Deductions, Variable Pays, or Days in Lieu).

Initial Setup

First-Time Configuration

Before creating Employee Structures, ensure you have planned your organizational grouping strategy and identified which roles will need restricted access to specific employee groups.

Step 1: Access Employee Structures

  1. Navigate to Settings from the sidebar
  2. Click on Company section
  3. Expand the Employee Structures accordion

Step 2: Plan Your Structure Type

Decide between two grouping methods before creating your first structure:

  • Automatic Grouping: Employees are grouped based on criteria rules (Department, Office, Nationality, etc.) ‚Äî best for dynamic organizations where employee attributes change frequently
  • Manual Grouping: Employees are hand-picked for each group ‚Äî best for static organizational structures or special project teams

Minimum Requirement: Each structure must contain at least 2 groups. A single-group structure cannot be created.

Step 3: Identify Restrictable Roles

Before assigning restriction groups to roles, verify which roles in your organization support access restrictions. Navigate to Settings > Organization > Role Management to view roles marked as "Restrictable."

Standard restrictable roles include:

  • People Manager
  • Attendance Manager
  • Expense Manager
  • Leaves Manager
  • Payroll table admin
  • Transaction processing admin

Step 4: Prepare Employee Data (for Automatic Structures)

If using automatic grouping, ensure employee profiles have accurate data in the fields you plan to use for criteria:

  • Department, Office, Job Title
  • Nationality, Employment Type
  • Compensation fields
  • Custom fields (if configured)

Data Quality: Automatic structures rely on clean employee data. Incomplete or inconsistent profile data will result in employees not matching your criteria rules.

Step 5: Document Your Grouping Logic

Before creating structures, document:

  • Which employee groups you need (minimum 2 per structure)
  • The business purpose of each group
  • Which roles will have restricted access to each group
  • For automatic structures: the exact criteria rules for each group

Step 6: Plan Integration with Other Features

Employee Structures enable automatic assignment for several Bayzat features. Identify which features you will configure after creating structures:

  • Leave Policies: Automatically assign leave policies to employee groups
  • Employee Loan Policies: Map loan policies to specific organizational groups
  • Leave Salary Policies: Configure leave salary deductions by group

Note: The automatic assignment toggle for these features only appears after at least one Employee Structure exists.

Key Tasks

Task: Create Employee Structures

Build organizational groupings to segment employees for role-based access control and automatic policy assignment.

Subtask: Create Structure with Automatic Grouping

Use criteria-based rules to automatically assign employees to groups based on profile data.

  1. Navigate to Settings > Company > Employee Structures
  2. Click 'Add New' button
  3. Enter structure Name and Description, select Automatic type
  4. Click 'Next' to proceed to Step 2
  1. Click 'Add Group' and enter group name/description
  2. Configure automatic criteria using available data points (Department, Office, Nationality, Employment Type, Job Title, Compensation, Custom Fields)
  3. Add multiple conditions per block using AND/OR logic
  4. Add multiple blocks to create complex rules
  1. Repeat for additional groups (minimum 2 groups required)
  2. Click 'Next' to proceed to Step 3
  1. Review structure summary showing groups and criteria
  2. Click 'Create' to save

Expected Outcome: Structure appears in list with Automatic type badge and group count. Employees are automatically assigned to groups based on criteria.

Subtask: Create Structure with Manual Selection

Hand-pick specific employees for each group when criteria-based rules don't fit your needs.

  1. Navigate to Settings > Company > Employee Structures
  2. Click 'Add New' button
  3. Enter structure Name and Description, select Manual type
  4. Click 'Next' to proceed to Step 2
  1. Click 'Add Group' and enter group name/description
  2. Use search and filter to find employees
  3. Select employees individually via checkboxes
  4. Click 'Save' to add group
  1. Repeat for additional groups (minimum 2 groups required)
  2. Click 'Next' to proceed to Step 3
  3. Review structure summary
  4. Click 'Create' to save

Expected Outcome: Structure appears in list with Manual type badge. Only selected employees are assigned to groups.

Minimum Requirement: All structures must have at least 2 groups. The system will prevent creation if this requirement is not met.

Task: Manage Existing Structures

View, edit, and delete structures to maintain accurate organizational groupings.

Subtask: View Structure Details

  1. Navigate to Settings > Company > Employee Structures
  2. Review structures list showing Name, Type, Groups, Employees, Created
  3. Click three-dot menu on any structure
  4. Select 'View Groups' to see group breakdown

Expected Outcome: Dialog displays all groups with employee counts for the selected structure.

Subtask: Edit Structure

  1. Click three-dot menu on structure row
  2. Select 'Edit'
  3. Update Name or Description (Type cannot be changed)
  4. Modify groups: add new groups, edit existing groups, or remove groups
  5. For Automatic structures: update criteria rules
  6. For Manual structures: update employee selections
  7. Review changes in Step 3
  8. Click 'Update' to save

Expected Outcome: Structure is updated with new configuration. Employee assignments are recalculated for Automatic structures.

Note: Structure type (Manual or Automatic) cannot be changed after creation. You must create a new structure to use a different type.

Subtask: Delete Structure

  1. Click three-dot menu on structure row
  2. Select 'Delete'
  3. Review warning: structure will be removed from role restrictions and policy assignments
  4. Click 'Delete' to confirm

Expected Outcome: Structure is removed from list. Role restrictions and automatic policy assignments using this structure are disabled.

Impact: Deleting a structure removes it from all role restrictions and automatic policy assignments. Affected roles will no longer have restricted access, and policies will no longer auto-assign based on this structure.

Task: Assign Viewership Permissions to Admin Roles

Control which employees each admin role can view and manage by assigning restriction groups.

Subtask: Configure Role-Based Access Control

Limit admin roles to specific employee groups for data confidentiality and focused management.

  1. Navigate to Settings > Organization > Role management
  2. Click Role assignment tab
  3. Click Edit on an employee to open role assignment dialog
  1. In Step 1, check roles with 'Restrictable' badge (People Manager, Attendance Manager, Expense Manager, Leaves Manager, etc.)
  2. When a restrictable role is checked, 'Restriction groups' dropdown becomes enabled
  1. Click 'Restriction groups' dropdown
  2. Select one or more groups from the list (format: 'Structure Name | Group Name')
  3. Selected groups limit the role's access to employees in those groups only
  4. Click 'Next' to proceed to Step 2
  5. Review role assignments and restrictions
  6. Click 'Save' to apply

Expected Outcome: Admin role is restricted to viewing and managing only employees in the selected groups. Employees outside these groups are hidden from this admin.

Restrictable Roles: Adjustment manager, Attendance Manager, Expense Manager, Leaves Manager, Payroll table admin, People Manager, Performance Manager, Transaction processing admin, and custom roles with restrictable permissions.

Super Admin: This role covers all other roles except Transaction Processor and cannot be restricted to specific employee groups.

Task: Use Structures for Automatic Policy Assignment

Eliminate manual policy assignment by mapping Employee Structure groups to policies.

Subtask: Assign Leave Policies Automatically

Map specific leave policies to employee groups so assignments update automatically when group membership changes.

  1. Navigate to Settings > Leaves > Leave Policies
  2. Click 'Manage Employee Assignment' on a leave policy
  1. Toggle ON 'Enable automatic assignment using grouping structures'
  1. Select an Employee Structure from the dropdown
  1. For each policy row, click 'Assign group' dropdown
  1. Select one or more groups to assign to this policy
  1. Click 'View X employees' to preview assignments
  1. Review employee assignments by expanding group accordions
  2. Close preview and click 'Save' to apply automatic assignment

Expected Outcome: Employees in selected groups are automatically assigned to the leave policy. When group membership changes (due to criteria updates or manual edits), policy assignments update automatically.

Excluded Policy: Days in Lieu (Attendance Policy) cannot be automatically assigned using Employee Structures.

Requirement: The automatic assignment toggle is only available when at least one Employee Structure exists.

Task: View Structure Usage

Track where structures are used across roles and policies to understand impact before making changes.

Subtask: Check Role Assignments

  1. Navigate to Settings > Company > Employee Structures
  2. Click three-dot menu on structure row
  3. Select 'View Employees & Roles'
  4. Review list of employees with roles restricted to this structure's groups

Expected Outcome: Dialog displays all employees whose admin roles are restricted to groups in this structure, helping you understand access control impact.

Automatic Assignment Limitations

Leave Policies

Days in Lieu Exclusion: Attendance-based Days in Lieu policies cannot be automatically assigned using Employee Structures. These must be assigned manually through the Days in Lieu Policy configuration.

Automatic assignment requires:

  • At least one Employee Structure must exist before the toggle appears
  • Structure must contain groups with employees assigned
  • Each leave policy can map to multiple groups from a single structure
  • Changing structure selection clears existing group mappings

Other Policy Types

The following policy types do NOT support automatic assignment via Employee Structures:

  • Allowances
  • Variable Pays
  • Additions
  • Deductions
  • Leave Encashment Policy
  • Daily Wage Calculation
  • Payroll Templates
  • Bank Accounts
  • End of Service Eligibility
  • Social Security Contributions

Data Point Availability for Automatic Criteria

Automatic grouping supports 50+ employee data points across these categories:

  • Profile: Department, Office, Nationality, Gender, Marital Status
  • Employment: Employment Type, Job Title, Reports To, Hire Date
  • Compensation: Salary components, allowances, payment frequency
  • Custom Fields: Any custom fields defined in employee profiles

Data Dependency: Automatic criteria only work if the selected data points are populated in employee profiles. Empty fields will not match any criteria conditions.

Edge Cases

Overlapping Group Membership

By Design: Employees can belong to multiple groups within the same structure. This is intentional for scenarios like "Sales Team" and "Remote Workers" where an employee may qualify for both.

  • Manual structures: Admin explicitly selects employees for each group
  • Automatic structures: Employees matching multiple criteria blocks appear in all matching groups
  • Leave policy assignment: Employee receives the policy if they match ANY assigned group

Empty Groups After Automatic Criteria Changes

When editing automatic criteria, groups may become empty if no employees match the new conditions.

  • System allows saving structures with empty groups
  • Empty groups still appear in restriction group dropdowns
  • Assigning an empty group as a restriction effectively blocks all access for that role

Employee Profile Changes Impact on Automatic Groups

Automatic structures recalculate group membership when employee profile data changes.

  • Department change: Employee automatically moves to the group matching new department criteria
  • Office change: Employee automatically moves to the group matching new office criteria
  • Custom field updates: Employee group membership updates based on new field values
  • Recalculation happens in real-time; no manual refresh needed

Role Restriction Groups and Super Admin

Super Admin Exception: Super Admin role cannot be restricted using Employee Structure groups. Super Admins always have access to all employees regardless of restriction group settings.

Structure Used in Multiple Contexts

A single structure can be used simultaneously for:

  • Role access restrictions (multiple roles can reference the same structure)
  • Leave policy automatic assignment
  • Loan policy automatic assignment
  • Leave salary policy automatic assignment

Changes to the structure (group membership, criteria) immediately affect all contexts where it's used.

Performance Considerations

Large Workforce Structures

  • Structures with 1000+ employees may take longer to load in View Employees dialog
  • Complex automatic criteria with multiple AND/OR blocks may slow down group recalculation
  • Consider splitting very large structures into multiple smaller structures by region or department

Frequent Criteria Changes

Avoid Frequent Edits: Each edit to automatic criteria triggers recalculation for all employees. Plan criteria carefully before saving to minimize recalculations.

Integration Edge Cases

Leave Policy Assignment Conflicts

When an employee belongs to multiple groups mapped to different leave policies:

  • Employee receives ALL mapped policies
  • If the same policy is mapped to multiple groups the employee belongs to, they receive it only once
  • Manual employee-level policy assignments override automatic assignments

Role Assignment with Multiple Restriction Groups

When a role is assigned restriction groups from multiple structures:

  • Role can view/manage employees who appear in ANY of the assigned groups (OR logic)
  • If an employee appears in multiple assigned groups, they are counted once
  • Removing a restriction group immediately revokes access to employees in that group

FAQs & Support

Frequently Asked Questions

Can I assign an employee to multiple Employee Structures?

No. An employee can only belong to one structure at a time. If you need to organize employees in multiple ways, create separate structures with different grouping criteria.

What happens if I delete an Employee Structure that's assigned to leave policies?

Deleting a structure removes it from all policy assignments. Affected policies will revert to manual employee assignment. Always review policy assignments before deleting a structure.

Can I change a structure from Manual to Automatic after creation?

No. The structure type (Manual or Automatic) is set during creation and cannot be changed. To switch types, create a new structure and delete the old one.

How do I know which roles are restrictable?

Restrictable roles display a 'Restriction groups' dropdown in the role assignment dialog. Common restrictable roles include People Manager, Attendance Manager, Leaves Manager, and Expense Manager.

What's the minimum number of groups required in a structure?

Every structure must have at least 2 groups. This ensures meaningful segmentation of your workforce.

Can I use Employee Structures for payroll policies?

Yes. Structures support automatic assignment for Leave Policies, Employee Loan Policies, and Leave Salary Policies. Other payroll settings like Allowances and Deductions do not support grouping-based assignment.

How often do Automatic groups update?

Automatic groups recalculate when employee data changes. If an employee's department, office, or other criteria change, they automatically move to the matching group.

Can I see which employees are in a group before assigning it to a policy?

Yes. Use the 'View Employees & Roles' action on any structure to see the full employee list per group, including their assigned roles.
