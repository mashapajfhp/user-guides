Feature Overview

The Letters feature enables organizations to manage the entire lifecycle of employee letter requests through a centralized platform. From initial submission to final approval and signature, the system automates what was traditionally a manual, paper-based process.

What Letters Does

Letters transforms how organizations handle official employee documentation by providing:

    • Template-Based Letter Generation: Create letters using customizable templates with 80+ dynamic variables that automatically populate employee data, salary information, and company details
    • Multi-Stage Approval Workflow: Route letter requests through designated approvers and authorized signatories, ensuring proper authorization at each stage
    • Digital Signature Management: Upload and embed digital signatures directly into generated PDFs, eliminating the need for physical signing
    • Status-Based Organization: Track all letter requests across three main statuses—Pending (869 requests), Approved (24 requests), and Rejected (78 requests)
    • Real-Time PDF Preview: Review letter content with embedded PDF preview before approval, improving accuracy and reducing errors

    Who Uses Letters

    The Letters feature serves multiple user roles within an organization:

    • Employees: Submit letter requests for salary certificates, employment verification, bank letters, and other official documentation
    • HR Administrators: Create and manage letter templates, configure authorized signatories, and oversee the request pipeline
    • Approvers: Review incoming letter requests, verify details, and approve or reject submissions
    • Authorized Signatories: Provide final approval by digitally signing approved letters, making them ready for download

    Key Capabilities

    Letters provides comprehensive functionality for document request management:

    • Request Submission: Employees select from available templates, specify who the letter is addressed to, and optionally add notes and attachments
    • Approval Processing: Approvers receive notifications, review request details with PDF preview, and can approve or reject with comments
    • Digital Signing: Authorized signatories review approved letters and add their pre-uploaded digital signature to the final document
    • Search and Filter: Find specific requests using search by employee name/ID, filter by department/office/date range, and navigate with pagination
    • Bulk Actions: Select multiple pending requests and perform bulk approval or rejection operations

    Business Value

    Implementing the Letters feature delivers measurable benefits:

    • Time Savings: Reduce letter processing time from days to hours by eliminating manual document creation and physical signature collection
    • Accuracy Improvement: Dynamic variable substitution ensures employee data is always current and accurate in generated letters
    • Audit Trail: Complete visibility into request history, approval chain, and status changes for compliance and record-keeping
    • Employee Satisfaction: Self-service request submission and faster turnaround times improve the employee experience
    • Operational Efficiency: Centralized template management and digital workflows reduce administrative overhead

The Letters feature integrates with the broader Company Docs system, providing centralized document storage and management alongside official letter generation capabilities.

Business Rules & Limitations

Business Rules & Limitations

Understanding the business rules and system limitations of the Letters feature helps you plan your implementation and avoid common pitfalls.

Template Configuration Rules

    • Letter Template Immutability: Once a letter request is created, the selected template cannot be changed. Users must delete the request and create a new one if the wrong template was selected.
    • Template Variable Language: Template variables support multiple languages. The language selector in the template editor determines which variable set is displayed, but all variables are available regardless of language selection.
    • Custom Field Visibility: Custom fields marked as "hidden from employees" are now available in letter templates. This allows administrators to use internal data fields as variables without exposing them to employees in their profiles.

    Approval Workflow Rules

    • Multi-Level Approval Required: All letter requests must pass through two distinct approval stages:
      1. Approver reviews and approves/rejects the request
      2. Authorized Signatory signs the approved letter
    • Approval Sequence: The approval sequence cannot be skipped or reversed. Letters must be approved before they can be signed.
    • Signatory Assignment: Each letter template must have an assigned Authorized Signatory. Letters cannot be generated without a designated signatory.

    Deletion and Recovery Limitations

    Critical Limitation: Deleted letter requests cannot be recovered by users. Once a letter is deleted, it is permanently removed from the system.
    • Delete Availability: The delete function is only available for letters in "Rejected" status. Pending and Approved letters cannot be deleted through the UI.
    • Recovery Process: If a letter is accidentally deleted, recovery requires backend intervention by the Bayzat support team. There is no self-service recovery option.
    • Workaround: To remove a pending or approved letter, first reject it, then delete it from the Rejected tab.

    Document Customization Constraints

    Known Limitation: The Bayzat platform currently lacks a comprehensive letterhead configuration module for HR administrators. This prevents organizations from fully customizing their document branding and header designs.

    • Letterhead Customization: While the template editor provides extensive formatting options, there is no dedicated letterhead management system for controlling document headers, footers, and branding elements across all templates.
    • Workaround: Administrators must manually configure branding elements within each template using the WYSIWYG editor.

    Variable and Field Rules

    • Variable Substitution: Template variables (highlighted in yellow in the editor) are automatically replaced with actual employee data when a letter is generated. Variables that have no corresponding data for an employee will appear blank in the generated letter.
    • Custom Field Availability: All custom fields (both visible and hidden) are available as variables in letter templates. Hidden custom fields were previously unavailable but are now accessible following platform fix TSSD-4089.
    • Signature Embedding: Authorized signatory signatures must be uploaded as image files (recommended size: 120px × 80px). The signature image is automatically embedded in the PDF at the designated location in the template.

    Edit and Update Restrictions

    • Limited Edit Fields: After a letter request is created, only the following fields can be edited:
      • Letter Addressed To
      • Notes for approvers
      • Additional file attachments
    • Read-Only Fields: The following fields cannot be changed after creation:
      • Letter Template
      • Employee (On Behalf Of)
      • Approvers
      • Authorized Signatory

    Status Transition Rules

    • Status Progression: Letter requests follow a linear status progression:
      1. Pending Review → Pending Signature → Signed (if approved)
      2. Pending Review → Rejected (if rejected at any stage)
    • Status Reversal: Once a letter is signed, it cannot be reverted to pending status. Rejected letters cannot be re-opened; a new request must be created.
    • Bulk Actions: Bulk approve and bulk reject actions are available for pending letters, but bulk operations cannot be performed across different status tabs.

    Workflow Automation Limitation

    No Workflow Integration: The Letters feature does not currently integrate with Bayzat's workflow automation system. Letter request events (created, approved, rejected, signed) cannot trigger automated workflows or actions.
    • Manual Notifications: Approvers and signatories must be notified manually or rely on email notifications. Custom workflow triggers for letter events are not available.
    • Workaround: Use related triggers such as "Employee is updated" or "Employee ticket is created" to build workflows that indirectly support letter request processes.

    Data and Reporting Constraints

    • Search Limitations: The search function searches by employee name and ID only. Letter template names, addressed-to fields, and notes are not searchable.
    • Filter Options: Available filters are limited to:
      • Date range (submitted date)
      • Department
      • Office
    • Export Functionality: Individual letters can be downloaded as PDFs, but there is no bulk export or reporting function for letter request data.

    Performance and Scale Considerations

    • Large Template Libraries: Organizations with many letter templates (30+) may experience slower template selection performance. Consider organizing templates by category or archiving unused templates.
    • PDF Generation Time: Complex templates with many variables and embedded images may take longer to generate PDFs. Typical generation time is 2-5 seconds per letter.
    • Concurrent Approvals: Multiple approvers can review letters simultaneously, but the system does not prevent conflicting actions (e.g., one approver approving while another rejects).

Troubleshooting & Edge Cases

Troubleshooting & Edge Cases

This section addresses common issues, error scenarios, and edge cases you may encounter when using the Letters feature.

Common Issues and Solutions

Issue: Letter Template Not Appearing in Dropdown

Symptoms: A letter template does not appear in the template selection dropdown when creating a new letter request.

Possible Causes:

    • Template is marked as "Inactive" in Settings → Letter Templates
    • Template has no assigned Authorized Signatory
    • User lacks permission to view the template

    Solution:

    1. Navigate to Settings → Letter Templates
    2. Verify the template status is "Active" (toggle if needed)
    3. Check that an Authorized Signatory is assigned to the template
    4. Confirm your role has permission to create letter requests
    Issue: Variables Not Populating in Generated Letter

    Symptoms: Template variables appear blank or show placeholder text (e.g., {first-name}) in the generated PDF.

    Possible Causes:

    • Employee profile is missing data for the variable field
    • Variable syntax is incorrect in the template
    • Custom field variable references a deleted or renamed field

    Solution:

    1. Open the employee's profile and verify all required fields are populated
    2. Check the template editor to ensure variable syntax matches exactly (case-sensitive)
    3. For custom fields, verify the field still exists in Settings → Employee Structures → Custom Fields
    4. Test the template with an employee who has complete profile data
    Issue: Cannot Delete a Letter Request

    Symptoms: The Delete button is not visible or is disabled for a letter request.

    Possible Causes:

    • Letter is in Pending or Approved status (delete only available for Rejected letters)
    • User lacks permission to delete letters

    Solution:

    1. Check the letter's current status in the appropriate tab
    2. If the letter is Pending or Approved, reject it first, then navigate to the Rejected tab
    3. Click the Delete button and confirm the permanent deletion
    4. If Delete is still unavailable, contact your administrator to verify your role permissions
    Issue: Authorized Signatory Signature Not Appearing

    Symptoms: The signature does not appear in the generated PDF, or appears distorted.

    Possible Causes:

    • Signature image was not uploaded for the Authorized Signatory
    • Signature image dimensions are incorrect
    • Template does not include the {signature} variable

    Solution:

    1. Navigate to Settings → Letter Templates → Authorized Signatures
    2. Verify the signatory has an uploaded signature with status "Uploaded"
    3. Check that the signature image is 120px × 80px (recommended dimensions)
    4. Open the template editor and confirm the {signature} variable is placed in the correct location
    5. Re-upload the signature if the image appears distorted
    Issue: Letter Stuck in "Pending Signature" Status

    Symptoms: A letter remains in Pending Signature status for an extended period.

    Possible Causes:

    • Authorized Signatory has not received notification
    • Signatory lacks permission to sign letters
    • Signatory is inactive or offboarded

    Solution:

    1. Manually notify the Authorized Signatory to review pending letters
    2. Verify the signatory's role includes letter signing permissions
    3. Check if the signatory's employee status is Active
    4. If the signatory is unavailable, an administrator can reassign the letter to a different signatory (requires editing the template's default signatory)

    Edge Cases and Unusual Scenarios

    Edge Case: Employee Has Multiple Active Roles

    Scenario: An employee has multiple job titles or roles, and the letter needs to reference a specific role.

    Handling:

    • The {job-title} variable pulls the primary job title from the employee's profile
    • If multiple roles exist, ensure the correct role is set as primary before generating the letter
    • For secondary roles, use custom fields to store additional role information and reference those custom field variables in the template
    Edge Case: Letter Addressed to Non-Standard Recipient

    Scenario: A letter needs to be addressed to a government entity, embassy, or other non-standard recipient with specific formatting requirements.

    Handling:

    • Use the "Letter Addressed To" field to enter the full recipient name and title
    • Create a dedicated template for each recipient type with appropriate formatting and language
    • Include recipient-specific salutations and closing statements in the template body
    Edge Case: Employee Data Changes After Letter Approval

    Scenario: An employee's salary, job title, or other details change after a letter has been approved but before it is downloaded.

    Handling:

    • The PDF is generated at the time of approval/signature, not at download time
    • Once signed, the letter reflects the employee data at the time of signature
    • If updated information is needed, reject the existing letter and create a new request
    • Download and archive important letters immediately after signing to preserve the record
    Edge Case: Bulk Approval of Letters with Different Signatories

    Scenario: An approver attempts to bulk approve letters that are assigned to different Authorized Signatories.

    Handling:

    • Bulk approve works regardless of signatory assignment
    • After bulk approval, each letter moves to Pending Signature and appears in the respective signatory's queue
    • Each signatory must individually sign their assigned letters
    Edge Case: Template Contains Arabic or RTL Language Variables

    Scenario: A letter template includes Arabic text or right-to-left (RTL) language content.

    Handling:

    • The template editor supports RTL text direction
    • Use the language selector in the template editor to switch to Arabic variable names
    • Ensure employee profiles contain data in the appropriate language fields
    • Test PDF generation to verify proper RTL rendering
    Edge Case: Letter Request Created for Offboarded Employee

    Scenario: A letter request is submitted for an employee who has been offboarded or is no longer active.

    Handling:

    • The system allows letter requests for offboarded employees (e.g., for employment verification or final settlement)
    • Ensure the employee's profile data is complete and accurate before offboarding
    • Archived employee data remains accessible for letter generation
    • If employee data is incomplete, reactivate the employee temporarily, update the profile, generate the letter, then re-offboard

Glossary

Glossary

Key terms and definitions related to the Letters feature in Bayzat HR.

Letter Request
A formal request to generate an official company letter for an employee. Letter requests go through an approval workflow before the final signed letter is produced.
Letter Template
A pre-configured document format that defines the structure, content, and variables for a specific type of company letter. Templates are created and managed in Settings → Letter Templates.
Template Variable
A dynamic placeholder in a letter template (e.g., {first-name}, {job-title}) that is automatically replaced with actual employee data when the letter is generated. Variables are highlighted in yellow in the template editor.
Authorized Signatory
An employee designated to digitally sign official company letters. Each letter template must have an assigned Authorized Signatory. The signatory's uploaded signature image is embedded in the final PDF.
Approver
The first level of the letter approval workflow. The Approver reviews the letter request and decides whether to approve or reject it before it proceeds to the Authorized Signatory for signing.
Pending Review
The initial status of a letter request when it is awaiting approval from the designated Approver.
Pending Signature
The status of a letter request after it has been approved by the Approver and is awaiting digital signature from the Authorized Signatory.
Signed
The final status of a letter request after the Authorized Signatory has signed it. Signed letters are moved to the Approved tab and are ready for download.
Rejected
The status of a letter request that has been declined by either the Approver or the Authorized Signatory. Rejected letters can be deleted but cannot be re-opened.
Letter Addressed To
The recipient or organization for whom the letter is intended (e.g., "Emirates NBD Bank", "UAE Embassy"). This field is specified when creating the letter request.
On Behalf Of
The employee for whom the letter is being generated. Letter requests can be created on behalf of any employee in the organization.
WYSIWYG Editor
"What You See Is What You Get" - the visual document editor used to design letter templates. It provides formatting tools similar to Microsoft Word or Google Docs.
Variable Language
The language setting in the template editor that determines which set of variable names is displayed (e.g., English, Arabic). Variables are available in multiple languages to support multilingual letter generation.
Custom Field Variable
A template variable that references a custom field created in Settings → Employee Structures → Custom Fields. Custom field variables allow organizations to include company-specific data in letters.
Signature Image
The digital image file (recommended 120px × 80px) uploaded for an Authorized Signatory. This image is automatically embedded in signed letters at the location of the {signature} variable.
PDF Preview
The embedded PDF viewer in the letter detail view that displays the generated letter content before and after signing.
Bulk Actions
Operations that can be performed on multiple letter requests simultaneously, such as bulk approve or bulk reject. Available in the Pending tab when multiple letters are selected.
Template Status
Indicates whether a letter template is Active (available for use) or Inactive (hidden from the template selection dropdown). Template status is managed in Settings → Letter Templates.
Variable Substitution
The automatic process of replacing template variables with actual employee data when a letter PDF is generated. Occurs at the time of approval/signature.
Letterhead
The header section of a company letter that typically includes the company logo, name, address, and contact information. Currently configured manually within each template using the WYSIWYG editor.
Common Variables
A category of frequently used template variables such as first-name, last-name, company-name, date-of-joining, and current-date. Available in the template editor's variable panel.
Payroll Variables
Template variables related to employee compensation, including net-monthly-salary, basic-salary, gross-pay, and allowance-list. Used in salary certificates and employment verification letters.
Document Variables
Template variables for employee identification documents, such as emirates-id-number, passport-number, visa-uid-number, and their expiry dates.
Signatory Job Title
The {signatory-job-title} variable that displays the job title of the Authorized Signatory in the letter. Automatically populated based on the signatory's employee profile.
Letter Request ID
A unique identifier assigned to each letter request. Used for tracking and referencing specific requests when contacting support.
Actioned Status
A status indicator showing that the current user has already taken action (approved or rejected) on a letter request. Prevents duplicate actions by the same user.
Base64 Encoding
The technical method used to embed signature images directly into PDF documents. Users do not interact with this directly; it is handled automatically by the system.
Multi-Level Approval
The two-stage approval workflow for letter requests: (1) Approver reviews and approves, then (2) Authorized Signatory signs. Both stages must be completed for a letter to reach Signed status.

Frequently Asked Questions (FAQs)

Q: Can I change the letter template after creating a request?

A: No. Once a letter request is created, the template cannot be changed. If you selected the wrong template, you must delete the request (after rejecting it if it's not already rejected) and create a new request with the correct template.

Q: How do I recover a deleted letter request?

A: Deleted letter requests cannot be recovered by users. The deletion is permanent. If you accidentally deleted a critical letter, contact Bayzat Support immediately. They may be able to restore it from backups, but recovery is not guaranteed.

Q: Why don't I see the Delete button for a letter?

A: The Delete button only appears for letters in Rejected status. If the letter is Pending or Approved, you must first reject it, then navigate to the Rejected tab to delete it.

Q: Can I create a letter for an offboarded employee?

A: Yes. The system allows letter requests for offboarded employees. This is useful for generating employment verification letters or final settlement documents. Ensure the employee's profile data is complete before offboarding.

Q: What happens if employee data changes after a letter is signed?

A: The signed letter reflects the employee data at the time of signature. The PDF is generated when the letter is signed, not when it is downloaded. If you need a letter with updated information, create a new letter request.

Q: Can I use custom fields in letter templates?

A: Yes. All custom fields (including those marked as "hidden from employees") are available as variables in letter templates. Navigate to the template editor and select custom field variables from the "Employee Custom Fields" category.

Q: How do I create a letter in Arabic or another language?

A: Create a dedicated template for each language. Use the language selector in the template editor to switch to Arabic variable names. Ensure employee profiles contain data in the appropriate language fields. The editor supports right-to-left (RTL) text direction.

Q: Why isn't my signature appearing in the PDF?

A: Check the following:

    • Verify the signature image is uploaded in Settings → Letter Templates → Authorized Signatures
    • Ensure the signature status shows "Uploaded"
    • Confirm the template includes the {signature} variable
    • Check that the signature image dimensions are correct (recommended 120px × 80px)

    Q: Can I bulk approve letters assigned to different signatories?

    A: Yes. Bulk approve works regardless of signatory assignment. After bulk approval, each letter moves to Pending Signature and appears in the respective signatory's queue for individual signing.

    Q: How do I notify approvers and signatories about pending letters?

    A: The system sends email notifications automatically. However, the Letters feature does not integrate with Bayzat Workflows, so custom automation is not available. You can manually notify users or rely on the built-in email notifications.

    Q: What's the difference between "Letter Addressed To" and the recipient's name in the letter body?

    A: "Letter Addressed To" is a metadata field that identifies the intended recipient (e.g., a bank or embassy). The actual salutation and recipient name in the letter body are configured in the template using text and variables.

    Q: Can I export a list of all letter requests?

    A: No. There is no bulk export or reporting function for letter request data. You can download individual letters as PDFs, but you cannot export the list of requests to Excel or CSV.
