What are Approval Flows?

Approval Flows enable HR administrators to define and manage the routing logic for employee requests across the Bayzat platform. Instead of manually reviewing every request, you configure intelligent approval paths that automatically route requests to the right approvers based on criteria such as employee department, grade, location, or request amount. The system supports both simple default flows and advanced conditional flows with complex business rules.

With Approval Flows, you can create sophisticated approval hierarchies that mirror your organization's decision-making structure. Configure parallel approvers where any one person can approve, sequential multi-step approvals, and conditional routing that changes based on request attributes. The feature covers over 40 request types including leave, expenses, loans, attendance regularization, shift changes, and custom ticket types.

Key Benefits

  • Automated Request Routing: Requests automatically flow to the correct approvers without manual intervention, eliminating routing errors and delays
  • Conditional Logic: Create advanced flows with "if-then" criteria that route requests differently based on employee attributes, departments, grades, or custom data points
  • Multi-Step Hierarchies: Build approval chains with multiple sequential steps, ensuring proper governance for high-value or sensitive requests
  • Parallel Approvers: Configure OR logic where any approver in a group can approve, reducing bottlenecks when key stakeholders are unavailable
  • Centralized Governance: Manage all approval flows from a single interface, providing visibility and control over organizational approval policies
  • Role-Based Flexibility: Assign approvers by role (Line Manager, Department Head, HR Admin) or specific individuals, adapting to organizational changes

Product Foundation

Core Concepts

Request Types

Approval Flows support over 40 different request types, each with its own dedicated configuration tab. Common types include Leave, Expenses, Loans, Attendance Regularization, Shift Changes, Work From Home, Business Trips, and custom ticket types. Each request type can have completely different approval logic tailored to its specific governance requirements.

Advanced Flows (If Conditions)

Advanced flows use conditional logic to route requests based on criteria. You define "if" conditions using employee attributes (department, grade, location, custom fields) or request attributes (amount, duration, category). When criteria match, the request follows the advanced flow's approval path. Multiple advanced flows can exist with priority ordering—the system evaluates them top-to-bottom and uses the first matching flow.

Default Flows (Else)

The default flow acts as the "else" condition—it applies when no advanced flow criteria match. Every request type must have a default flow configured. This ensures all requests have an approval path even if they don't meet any advanced flow criteria. Default flows typically use simpler approval structures like routing to the employee's Line Manager.

Approval Steps

Each flow consists of one or more approval steps executed sequentially. A request must be approved at Step 1 before moving to Step 2, and so on. Steps can contain single approvers or multiple approvers. If any approver rejects the request at any step, the entire request is immediately rejected without proceeding to subsequent steps.

Approvers and Roles

Approvers can be assigned by role (Line Manager, Department Head, HR Admin, custom roles) or as specific individuals. Role-based approvers adapt automatically when organizational structure changes—if an employee's Line Manager changes, requests automatically route to the new manager. Individual approvers provide stability for specialized approval responsibilities.

Parallel Approvers (OR Logic)

Within a single approval step, you can configure multiple approvers with OR logic. This means any one of the approvers can approve the request to move it forward. Parallel approvers reduce bottlenecks when key stakeholders are unavailable—the request doesn't wait for a specific person. This is particularly useful for peer-level approvals or when multiple people have equivalent authority.

Criteria and Data Points

Advanced flows use criteria to determine when they apply. Criteria are built from data points—employee attributes like Department, Grade, Location, Employment Type, or custom employee fields. You can create complex conditions using multiple criteria combined with AND/OR logic. For example: "If Department = Sales AND Grade = Senior AND Expense Amount > 5000".

Blocks

Blocks are groupings within approval flows that allow you to organize complex approval structures. The "Add block" functionality enables you to create nested approval logic or group related approval steps together. This is an advanced organizational feature for managing sophisticated approval hierarchies.

Related Features

Approval Flows integrate with and depend on several other Bayzat features:

  • Role Management: Line Manager roles and custom approver roles must be configured in Role Management before they can be used in approval flows. Approvers need appropriate permissions to view and act on requests.
  • Employee Profiles: Approval criteria pull data from employee profiles (department, grade, location, custom fields). Accurate employee data is essential for conditional flows to route correctly.
  • Leave Management: Leave requests submitted by employees are routed through configured approval flows. The approval outcome determines whether leave is granted and reflected in leave balances.
  • Expense Management: Expense claims follow approval flows to ensure financial controls and budget compliance before reimbursement processing.
  • Loan Requests: Employee loan applications route through approval flows, typically requiring finance team approval in addition to line manager approval.
  • Attendance Regularization: Requests to correct attendance records follow approval flows to maintain attendance data integrity and prevent abuse.
  • Custom Request Types: Organizations can create custom ticket types (grievances, IT requests, facility requests) and configure dedicated approval flows for each type.
  • Notifications: The system sends automatic notifications to approvers when requests reach their approval step, and to requesters when requests are approved or rejected.

Feature Usage with Common Workflows

Managing Approval Flows

Approval flows define the routing and approval hierarchy for various request types in Bayzat. You can create advanced flows with conditional logic or configure default flows that apply when no conditions match.

Note: Only Super Admins can configure approval flows. Changes affect all future requests of the selected type.

Creating a New Advanced Approval Flow

Advanced flows allow you to route requests based on specific criteria such as employee attributes, department, or request amount.

  1. Navigate to Settings → Approval Flows
  2. Select the request type tab you want to configure (e.g., Leave, Expense, Air Ticket)
  3. In the Advanced approval flow (If conditions) section, click Add advance flow
  4. In the dialog that appears, configure the following fields:
    • Flow name: Enter a descriptive name for this approval flow
    • Description: Provide details about when this flow applies
    • Criteria section: Click Add criteria to define conditions
      • Select a data point (e.g., Department, Employee Grade, Cost Center)
      • Choose an operator (equals, not equals, contains, etc.)
      • Enter the value to match
      • Add multiple criteria as needed (all must match)
    • Approval flow section: Click Add step to create approval stages
      • Click Add approver within each step
      • Select roles from the dropdown (e.g., Line Manager, Department Head, HR Manager)
      • Configure parallel approvers using OR logic if needed
      • Add multiple steps for sequential approval
  5. Click Save or Add to create the flow
  6. Click Update approval flows at the bottom of the page to apply changes

Viewing Existing Approval Flows

You can view all configured approval flows for each request type to understand current routing logic.

  1. Navigate to Settings → Approval Flows
  2. Use the search bar or scroll to find the request type tab
  3. Review the Advanced approval flow (If conditions) section to see all conditional flows
    • Each flow displays its name, description, criteria, and approval steps
    • Multiple flows are listed in priority order
  4. Review the Default approval flow (Else) section to see the fallback routing
    • This applies when no advanced flow criteria match
    • Shows configured approval steps and approvers

Editing an Existing Approval Flow

Modify approval flows to adjust criteria, add or remove approval steps, or change approver roles.

  1. Navigate to Settings → Approval Flows
  2. Select the request type tab containing the flow you want to edit
  3. Locate the advanced flow in the list
  4. Click the Edit button (pencil icon) next to the flow
  5. Modify any of the following:
    • Flow name or description
    • Criteria conditions (add, remove, or change data points)
    • Approval steps (add or remove steps)
    • Approvers within steps (add, remove, or change roles)
  6. Click Save to preserve your changes
  7. Click Update approval flows at the bottom of the page to apply changes globally

⚠️ Changes Affect Future Requests Only

Modifying an approval flow does not affect requests already in progress. Only new requests submitted after clicking "Update approval flows" will use the updated routing logic.

Deleting an Approval Flow

Remove approval flows that are no longer needed or were created for testing purposes.

  1. Navigate to Settings → Approval Flows
  2. Select the request type tab containing the flow you want to delete
  3. Locate the advanced flow in the list
  4. Click the Delete button (trash icon) next to the flow
  5. Confirm the deletion when prompted
  6. Click Update approval flows at the bottom of the page to apply the removal

Tip: If you delete all advanced flows, the default approval flow will apply to all requests of that type.

Configuring Default Approval Flows

The default flow acts as a fallback when no advanced flow criteria match. This ensures every request has a defined approval path.

  1. Navigate to Settings → Approval Flows
  2. Select the request type tab
  3. Scroll to the Default approval flow (Else) section
  4. Click Add step to create approval stages
  5. Click Add approver within each step and select roles
  6. Configure multiple steps for sequential approval if needed
  7. Click Update approval flows to save changes

Using Parallel Approvers (OR Logic)

Configure multiple approvers within a single step where approval from any one approver is sufficient to proceed.

  1. When creating or editing an approval flow, add a step
  2. Click Add approver multiple times within the same step
  3. Select different roles for each approver
  4. The system will route to all approvers simultaneously
  5. Approval from any one approver in the step allows the request to proceed

Note: If any approver in a step rejects the request, the entire approval process stops immediately, regardless of whether other approvers in that step have approved.

Working with Criteria and Data Points

Criteria determine when an advanced flow applies. You can use a rich set of employee attributes and request properties.

Available data points include:

  • Employee attributes: Department, Grade, Cost Center, Location, Job Title
  • Request properties: Amount, Duration, Date range
  • Custom fields configured for the request type

To add criteria:

  1. In the advanced flow builder, click Add criteria
  2. Select a data point from the dropdown
  3. Choose an operator (equals, not equals, contains, greater than, etc.)
  4. Enter the comparison value
  5. Add additional criteria as needed (all must match for the flow to apply)

Managing Multiple Advanced Flows

When multiple advanced flows exist for a request type, they are evaluated in priority order from top to bottom.

  1. The system evaluates each advanced flow's criteria in sequence
  2. The first flow whose criteria match is applied
  3. If no advanced flow matches, the default flow is used
  4. To change priority, you may need to delete and recreate flows in the desired order

⚠️ Flow Priority Matters

If you have overlapping criteria in multiple flows, only the first matching flow will be used. Order your flows from most specific to most general.

Assigning Line Manager Roles

Before using Line Manager as an approver role, ensure the role is properly assigned in Role Management.

  1. Navigate to Settings → Role Management
  2. Locate the Line Manager role
  3. Assign the role to the appropriate employees who will act as line managers
  4. Return to Approval Flows to configure Line Manager as an approver

Only Super Admin can configure approval flows. Line Managers must have the role assigned in Role Management before they can be used as approvers.

No Workflow Integration Available: Approval flows do not currently integrate with Bayzat Workflows. No triggers or actions are available for approval flow configuration.

Approval flows are internal routing mechanisms that automatically apply when requests are submitted. They operate independently of the Bayzat Workflows automation system.

How Approval Flows Work

When an employee submits a request (leave, expense, air ticket, etc.), the system automatically:

  1. Evaluates all advanced approval flows for that request type in priority order
  2. Applies the first flow whose criteria match the request and employee attributes
  3. If no advanced flow matches, applies the default approval flow
  4. Routes the request to the configured approvers in sequence
  5. Sends notifications to approvers at each step
  6. Advances to the next step when approval is granted
  7. Stops the process immediately if any approver rejects

Business Rules and Limitations

Business Rules

Approval flows operate under the following business rules:

  • Super Admin Only: Only users with Super Admin permissions can create, edit, or delete approval flows
  • Immediate Rejection: If any approver in a step rejects a request, the entire request is immediately rejected and does not proceed to subsequent approval steps
  • Role-Based Approvers: Line Managers and other roles must be properly assigned in Role Management before they can be configured as approvers
  • Feature-Specific Flows: Each request type (Leave, Expense, etc.) has its own independent approval flow configuration
  • Priority Ordering: When multiple advanced flows exist for a feature, they are evaluated in priority order until a matching condition is found
  • Default Fallback: If no advanced flow criteria match, the request routes through the default approval flow

No Known Limitations: Currently, there are no documented limitations for the Approval Flow feature. The system operates as designed with all core functionality available.

Edge Cases

  • Parallel Approvers with OR Logic: When multiple approvers are configured in a single step with OR logic, approval by any one approver advances the request. Ensure this matches your organization's approval policy
  • Multiple Matching Advanced Flows: If an employee's attributes match multiple advanced flow criteria, only the first matching flow (by priority order) is applied. Review priority ordering carefully
  • Approver Role Changes: If an approver's role is removed after a request is submitted but before they approve, the request may become stuck. Manual intervention by Super Admin may be required
  • Employee Attribute Changes: If an employee's attributes change after submitting a request (e.g., department transfer), the request continues through the originally assigned approval flow, not the new one
  • No Default Flow Configured: If no default approval flow is configured and no advanced flows match, requests may fail to route properly. Always configure a default flow as a fallback
  • Complex Block Structures: Using "Add block" to create nested approval logic can become difficult to visualize and troubleshoot. Document complex flows externally for reference

Frequently Asked Questions

Can employees see the approval flow before submitting a request?

No, employees do not see the approval flow configuration when submitting requests. The approval routing is applied automatically based on their profile attributes. Only Super Admins can view and configure approval flows.

What happens if an approver is on leave?

The request will remain pending until the approver returns and takes action. Consider configuring backup approvers or parallel approval steps with OR logic to prevent bottlenecks. Some organizations also configure workflows to escalate or notify alternate approvers after a certain time period.

Can I set up automatic approval for certain requests?

Not directly through approval flows. However, you can configure a single-step approval flow with a specific role, then use Bayzat Workflows to automatically approve requests that meet certain criteria. This requires both approval flow configuration and workflow setup.

How many advanced flows can I create per feature?

There is no documented limit on the number of advanced flows per feature. However, for maintainability, it's recommended to keep the number manageable (typically under 10) and use clear naming conventions to identify each flow's purpose.

Can I copy approval flows from one request type to another?

No, there is no built-in copy function. You must manually configure approval flows for each request type. Consider documenting your approval flow patterns externally to make replication easier.

What's the difference between advanced flows and default flows?

Advanced flows use "If" conditions to route requests based on employee attributes (department, grade, location, etc.). Default flows are "Else" fallbacks that apply when no advanced flow criteria match. Every request type should have a default flow configured.

Can I require multiple approvers to all approve before proceeding?

Yes, configure multiple approvers in a single step without OR logic. The request will only proceed to the next step when all approvers in the current step have approved. This is AND logic (all must approve).

How do I know which approval flow was applied to a specific request?

View the request details in the Requests tab. The approval history shows which approvers received the request, indicating which flow was applied. The flow name is not explicitly displayed, but you can infer it from the approver sequence.
