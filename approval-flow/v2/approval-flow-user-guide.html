What is Approval Flow?

Overview

Approval Flow routes employee requests through designated approvers based on configurable criteria like request type, amount, or department. Administrators define approval chains that automatically direct requests to the right people in the right sequence.

Create flows with multiple approval levels, set conditional routing rules, and assign role-based approvers to ensure requests reach the right decision-makers.

Key Benefits

  • Eliminate manual approval tracking by automatically routing requests to designated approvers
  • Reduce approval delays with clear visibility into request status and pending actions
  • Standardize approval processes across different request types and departments
  • Maintain audit trails showing who approved what and when
  • Scale approval processes as the organization grows without adding administrative overhead

Product Foundation Overview

How Approval Flow Fits

Approval Flow is a transactional routing system that directs employee requests to the appropriate approvers based on organizational hierarchy and business rules. It sits at the intersection of employee self-service and management oversight.

Mental model: Employee submits request → Approval Flow routes to approver(s) → Manager reviews → Request approved/rejected → Action triggered

One approval flow configuration can affect all employees across the organization, making routing rules critical to operational efficiency.

Key Decisions Before Setup

Answer these questions before configuring Approval Flow:

  • Who should approve what? — Define approval authority by request type, amount, or employee level
  • How many approval levels are needed? — Determine if requests require single or multi-stage approval
  • What conditions trigger escalation? — Identify when requests need higher-level approval (e.g., amount thresholds, policy exceptions)
  • Which roles need approval permissions? — Map organizational roles to approval capabilities in the system

Initial Setup & Configuration

Role Assignment Prerequisites

Before configuring approval flows, assign appropriate roles to team members who will participate in approval processes.

Settings → Role Management → Assign Roles

Key steps:

  1. Navigate to Settings and click Role Management
  2. Click Assign Roles
  3. Enter employee name(s) who will participate in approvals
  4. Select appropriate role(s) from the list — multiple roles can be assigned to the same employee
  5. Hover over information icons to review specific permissions for each role

Note: Different roles have predefined permissions that control what approval actions users can perform. Review role permissions carefully before assignment.

Approval Flow Creation

Configure routing rules that determine how requests move through approvers.

Basic setup steps:

  1. Access the approval flow configuration screen
  2. Define flow name and description
  3. Select request types this flow applies to (Leave, Air Ticket, Loan, Expense, etc.)
  4. Configure routing criteria using available data points
  5. Define approver sequence and conditions
  6. Set escalation rules if needed
  7. Activate the flow

⚠️ Role-Based Access: Only users with appropriate role permissions can create or modify approval flows. Ensure Super Admin, People Manager, or Task Manager roles are assigned to users who need to configure flows.

Activation & Testing

Before deploying approval flows organization-wide:

  1. Activate flows in test mode with a small user group
  2. Submit test requests for each configured type
  3. Verify routing follows expected approval paths
  4. Confirm notifications reach approvers correctly
  5. Test edge cases (thresholds, escalations, delegations)
  6. Deploy to full organization once validated

⚠️ Impact Scope: Approval flows affect all employees submitting requests. Test thoroughly before organization-wide activation to avoid blocking critical business processes.

Business Rules & Limitations

Configuration Safeguards

⚠️ Automatic Approval Risk

Letter request systems can be configured for automatic approval without manual signatory review. Verify approval settings before activating any request type to prevent unintended automatic approvals.

Known Configuration Issues

Issue Impact Mitigation
Automatic approval configuration lacks confirmation prompts Requests may bypass required manual review if settings are misconfigured Review all approval flow configurations before deployment; test with sample requests

Best Practices

  • Always test approval flows with sample requests before activating for production use
  • Document approval requirements for each request type to ensure proper routing
  • Review automatic approval settings quarterly to confirm they align with current policies
  • Assign backup approvers to prevent approval bottlenecks during absences

Troubleshooting & Edge Cases

Common Issues

Issue Cause Resolution
Requests approved automatically when manual review expected Approval flow configured with automatic approval enabled Navigate to approval flow settings, disable automatic approval, assign required approvers
Approval notifications not received Approver email settings or notification preferences disabled Verify approver email address in employee profile; check notification settings in user preferences
Request stuck in approval queue Assigned approver unavailable or approval delegation not configured Reassign request to alternate approver or configure approval delegation for absent approvers
Approval flow not triggering for specific request type Request type not linked to approval flow configuration Review request type settings and ensure it is mapped to the correct approval flow

Edge Cases

  • Multi-level approvals with conditional routing: If approval flow includes conditions based on request amount or employee level, test all threshold scenarios to ensure correct routing
  • Approver hierarchy changes mid-approval: If an approver's reporting structure changes while a request is pending, the system may not automatically reroute; manually reassign if needed
  • Bulk request approvals: When approving multiple requests simultaneously, verify each request individually to avoid approving items that require additional review
  • Approval flow conflicts with leave policies: If an approver is on leave during the approval window, configure delegation rules or backup approvers to prevent delays

Validation Checklist

Before deploying approval flows to production, verify:

  • All request types are mapped to the correct approval flow
  • Approver roles and permissions are correctly assigned
  • Automatic approval is disabled for request types requiring manual review
  • Notification settings are enabled for all approvers
  • Backup approvers or delegation rules are configured for key approvers
  • Test requests complete the full approval cycle as expected

FAQs & Support

Frequently Asked Questions

How do I assign approval permissions to team members?

Navigate to Settings → Role Management → Assign Roles, enter employee names, and select appropriate roles. Multiple roles can be assigned to the same employee.

What roles can approve requests?

Super Admin, People Manager, and Task Manager have full approval access. Custom roles can be created by contacting your Customer Success Manager to define specific approval permissions.

Can I see what actions each role can perform?

Yes. Hover over the information icon next to each role to view specific actions and permissions associated with that role.

What if an employee needs approval permissions but doesn't have the Task Manager policy?

Contact your Customer Success Manager to enable the Task Manager policy or create a custom role with the required approval permissions.

Can one person have multiple approval roles?

Yes. You can assign multiple roles to a single employee, allowing them to approve different types of requests based on combined permissions.

How do I know which approval flow applies to a specific request type?

Each request type (leave, expense, loan, etc.) has a configured approval flow visible in the Automations section. Review the flow diagram to see routing rules and approver levels.
