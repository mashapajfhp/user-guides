What is Approval Flow?

Overview

Approval Flow defines who reviews and approves employee requests across modules like Leave, Air Ticket, Expense, Payroll, and more. Configure approval hierarchies with role-based approvers, sequential or parallel routing, and conditional logic based on request attributes.

Add approval steps, assign roles (Line Manager, HR Manager, custom roles), and set priority for advanced flows with criteria-based routing.

Key Benefits

  • Configure approval hierarchies per module (Leave, Payroll, Expense, etc.)
  • Route requests based on criteria like amount, department, or policy type
  • Support parallel (OR) or sequential (AND) approver logic
  • Enable multi-level approvals with custom roles beyond Line Manager
  • Prioritize advanced flows with criteria matching for complex scenarios
  • Eliminate email-based approval tracking with in-platform routing

Product Foundation Overview

How Approval Flow Fits

Approval Flow is a configuration framework that defines routing rules for requests requiring authorization. It sits between employee-initiated actions and their execution, enforcing organizational approval hierarchies.

Mental model: Approval Flow (configuration) → Applied to request type → Routes to designated approvers → Request approved/rejected → Action proceeds or stops

One approval flow configuration can control routing for multiple request types across the entire organization.

Key Decisions Before Setup

Answer these questions before creating approval flows:

  • Which request types need approval? — Leaves, expenses, payroll transactions, or all three
  • Who are your approvers? — Line Managers, department heads, finance roles, or custom roles
  • Sequential or parallel approval? — Must approvers act in order, or can any approver proceed
  • What happens when approvers are unavailable? — Backup approvers, delegation, or ultimate approver bypass

Initial Setup & Configuration

Configuration Walkthrough

Approval flows are configured per module. Navigate to Settings → Approval flows and select the tab for the feature requiring approval (Leave, Air Ticket, Loan, Expense, etc.).

Settings → Approval flows → [Feature Tab]

Each tab displays existing approval steps. Use + Add approval step to configure who approves requests and in what order.

Standard Approval Flow Setup

Standard flows apply a single approval hierarchy company-wide. This is the default configuration for most organizations.

Step 1: Add Approval Steps

Click + Add approval step on the feature tab. Each step represents a level in the approval chain.

Step 2: Select Approver Roles

Choose who approves at this step:

  • Line Manager (1 level, 2 levels, etc.) — Employee's direct manager or higher
  • Custom roles — Sales Director, CEO, Department Manager, Office Manager (configure in Role Management first)
  • Specific employees — Assign by name for fixed approvers

Step 3: Configure Approval Logic

Define how multiple approvers within a step behave:

  • AND condition — All approvers in the step must approve
  • OR condition — Any one approver can approve
  • Sequential steps — Step 2 only receives requests after Step 1 approves

Redundancy tip: Add multiple approvers with OR logic to prevent bottlenecks when someone is unavailable.

Step 4: Save Configuration

Review the approval hierarchy and click Save. The flow applies immediately to all new requests.

Advanced Approval Flow Setup

Advanced flows allow different approval paths based on criteria like policy type, department, office, or request amount.

Settings → Approval flows → [Feature Tab] → + Add advanced flow

Step 1: Name and Describe the Flow

Enter a Flow name (e.g., "High-value Air Ticket Approval") and Description explaining when this flow applies.

Step 2: Add Criteria

Click + Add criteria to define conditions that trigger this flow. Each criterion has:

  • Data Point — Select from 60+ options (Policy Allowance Amount, Currency, Department, Office, Leave Policy, etc.)
  • Operation — Choose comparison logic (is equal to, is greater than, is between, is not equal to, etc.)
  • Value — Enter the threshold or selection (e.g., 3000 for amount, AED for currency)

Step 3: Set Criteria Logic

Choose how multiple criteria combine:

  • Execute when ALL criteria match — Request must meet every condition
  • Execute when ANY criteria match — Request meets at least one condition

Step 4: Add Approval Steps

Configure the approval hierarchy for requests matching these criteria. Same role selection and logic options as standard flows.

Step 5: Save and Prioritize

Advanced flows are evaluated by priority order. Use the ellipsis menu to Move up in priority or Move down in priority.

⚠️ Priority matters: The system uses the first matching flow. If a request matches multiple flows, only the highest-priority flow applies.

Activation and Testing

Approval flows activate immediately upon saving. Test the configuration:

  1. Submit a test request matching your criteria
  2. Verify it routes to the correct approver(s)
  3. Check that rejection/approval behavior matches expectations
  4. Confirm email notifications reach approvers

⚠️ Rejection behavior: If any approver in a step rejects, the entire request is rejected immediately. Rejected requests cannot return to pending status.

Business Rules & Limitations

Configuration Constraints

⚠️ Manual Activation Required: Approval flows cannot be self-enabled by administrators. Backend configuration by Bayzat support is required to activate workflows for your organization.

⚠️ Custom Role Limitations: Custom roles created through the admin panel do not automatically appear in approval flow configurations. Support intervention is required to enable custom roles for approval routing.

⚠️ Notification Timing: The system sends notifications to all approval levels simultaneously, regardless of approval sequence. Approvers receive alerts even when they cannot yet take action.

Operational Rules

Payroll Approval Flow

  • If no approval flow is configured, transactions process automatically
  • Transactions move to approval queue once workflow is set up
  • Approvers must be in the approval hierarchy to act
  • Rejection requires remarks for amendment guidance
  • Multi-level approvals must complete in sequence
  • Both payroll table and EOS transactions follow approval flow
  • Email notifications required for pending approvals

Leave Approval Flow

  • Standard flow applies single approval configuration company-wide
  • Advanced flow supports per-department, per-office, or per-policy variations
  • Sequential approval enforces critical approver first
  • Multiple approvers per step provide redundancy
  • Ultimate approver can skip entire chain for urgent situations
  • Custom roles like Department Manager and Office Manager work similar to Line Manager

Expense Approval Flow

  • For companies with BOTH payroll and expense features enabled, no changes to existing behavior
  • Changes only apply when expenses enabled but payroll disabled
  • "Add to Payroll" replaced by "Mark as settled" only for reimbursements (not corporate cards)
  • Navigation restructure: "Employee work expenses" moves to "Company Expenses" section regardless of payroll status

Edge Cases & Warnings

Approver Unavailability: Multiple approval hierarchy levels can cause delays. Approver unavailable or on leave requires backup approver handling. Multiple approvers per step and ultimate approver role mitigate this risk.

Mid-Workflow Changes: Transaction edited after submission but before approval, or approver role removed mid-workflow, can cause routing issues. Circular approval dependencies must be avoided.

Rejection Handling: Rejection after partial multi-level approval requires full cycle restart with amendments. Timeout for pending approvals should be configured. Approver cannot approve their own transaction.

Concurrent Requests: Multiple transactions pending for same approver or concurrent approval requests can create processing bottlenecks.

Edge Cases

Approver Unavailability Scenarios

  • Approver on vacation: Configure multiple approvers per step for redundancy, or designate ultimate approver for urgent situations
  • Approver role changed mid-approval: Workflow may fail to route correctly. Support intervention required to reassign
  • Extended leave without backup: Ultimate leave approver can bypass entire chain to prevent bottlenecks

Complex Organizational Structures

  • Flat hierarchy: Standard flow may not suit organizations without clear line manager levels. Consider advanced flow with custom roles
  • Matrix reporting: Employee reports to multiple managers. Configure multiple approvers per step or use AND conditions
  • Cross-department approvals: Advanced flow supports department-specific routing. Configure separate flows per department

High-Volume Approval Periods

  • End-of-month payroll rush: Multiple transactions pending for same approver. Consider parallel approvers or higher approval thresholds
  • Annual leave season: Concurrent leave requests can overwhelm approvers. Use multiple approvers per step for load balancing
  • Bulk transaction submissions: Sequential approval of large batches can delay processing. Ultimate approver can expedite critical items

Data Integrity Scenarios

  • Transaction edited after submission: Changes made before approval may not reflect in approval summary. Reject and request resubmission
  • Salary changes during approval: Payroll config changes can be omitted from initial scope. Ensure all changes are captured before submission
  • Recurring deduction conflicts: Problematic recurring deductions prevent approval flow disabling. Resolve deduction issues first

Workarounds for Current Limitations

No OR logic in approval routing: Current system enforces sequential AND logic. To simulate OR logic, create multiple approval flows with different criteria and manually route requests to the appropriate flow.

Cannot use project/client fields for expense routing: Current expense approval lacks advanced criteria. Use manual review or external tracking for project-specific approvals until feature is enhanced.

No automated notification for line managers outside approval chain: When manager raises request on behalf of employee, manually notify relevant stakeholders via email or chat.

FAQs & Support

Frequently Asked Questions

Who can configure approval flows?

Only Super Admin users can configure approval flows. Line Managers and other approvers must first be assigned their roles in Role Management before they can participate in approval processes.

What happens if one approver rejects a request?

If any approver in a step rejects the request, the entire approval process stops immediately and the request is fully rejected. Rejected requests cannot be moved back to pending status.

Can I set up parallel approvals where any one approver is sufficient?

Yes, you can configure approvers to approve in parallel using OR conditions. This allows the request to proceed when any one of the designated approvers acts, rather than requiring all approvers.

How do I add multiple approval levels?

You can add sequential approval steps by configuring additional approval blocks. Each step must be completed before the request moves to the next level in the hierarchy.

What if an approver is unavailable?

Approvers can be delegated if the primary approver is unavailable. You can also configure backup or alternate approvers within the approval flow settings.

Why can't I see the approval flow configuration option?

If approval flow configuration is not visible in your Settings menu, contact your Customer Success Manager to enable this feature for your account.

Do I need to assign roles before setting up approval flows?

Yes, all approvers must have their roles properly assigned in Role Management before they can be added to approval flows. This ensures approval requests route correctly to the designated users.
