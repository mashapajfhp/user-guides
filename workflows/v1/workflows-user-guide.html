Feature Overview

What Bayzat Workflows Does

Bayzat Workflows is a powerful automation engine that connects events across your HR platform to automated actions. When something happens in your organization—an employee joins, a leave request is submitted, a probation period ends—Workflows can automatically trigger notifications, create tasks, update records, or send messages to your team.

Workflows eliminates manual, repetitive tasks by creating intelligent automation sequences. Instead of manually sending welcome emails to new hires or reminding managers about pending approvals, you define the rules once and let Workflows handle the execution.

Who Uses Workflows

    • HR Administrators: Set up organization-wide automation for onboarding, offboarding, and employee lifecycle events
    • Payroll Managers: Automate notifications for salary changes, payroll cycle events, and financial approvals
    • Department Managers: Receive automated alerts about team changes, leave requests, and important milestones
    • System Administrators: Configure integrations with external tools like Slack, email systems, and task management platforms

    Key Capabilities

    • Event-Driven Triggers: 60+ events across Bayzat HR, Payroll, Timeoff, Employee Tickets, and external webhooks
    • Conditional Logic: Apply criteria filters to execute workflows only when specific conditions are met
    • Multi-Action Sequences: Chain multiple actions together in a single workflow
    • Template Library: Pre-built workflow templates for common HR scenarios
    • Custom Variables: Personalize messages and actions with dynamic employee data
    • Execution Tracking: Monitor workflow performance with detailed activity logs showing success, failure, and in-progress executions

Business Value

Workflows transforms your HR operations from reactive to proactive. By automating routine processes, your team can focus on strategic initiatives while ensuring consistent, timely communication across the organization. Workflows reduces human error, improves response times, and creates a more connected employee experience.

Organizations using Workflows report significant time savings in onboarding processes, improved compliance with approval flows, and better visibility into HR operations through automated notifications and task creation.

Product Foundation

Technical Prerequisites

Bayzat Workflows is available to all Bayzat platform users with appropriate administrative permissions. To use Workflows effectively, your organization should have:

    • Active Bayzat Account: Workflows is integrated into the Bayzat platform and requires an active account
    • Administrative Access: Workflow creation and management requires admin-level permissions in Settings
    • Configured HR Data: Employee profiles, departments, offices, and organizational structure should be set up for workflow criteria to function properly
    • Email System: Work email addresses configured for employees to receive workflow notifications

    Module Dependencies

    Workflows integrates deeply with other Bayzat modules and external applications:

    Core Bayzat Modules

    • Bayzat HR: 20 event triggers including employee creation, updates, hire dates, probation end dates, and work week changes
    • Bayzat Payroll: 2 event triggers for salary creation and updates; actions to create pay adjustment requests
    • Bayzat Timeoff: Leave request update triggers
    • Bayzat Employee Ticket: Ticket creation and status update triggers
    • Bayzat Task Management: Create task actions for workflow-driven task assignment

    External Integrations

    • Email: Send automated email notifications to employees, managers, or custom recipients
    • Slack: 4 available actions to send messages to Slack channels (requires webhook configuration)
    • Google Calendar: Create calendar events automatically
    • Mobile Notifications: Send push notifications to the Bayzat mobile app
    • Incoming Integration: Webhook URL triggers to connect external systems
    • Custom n8n Actions: Advanced automation actions including attendance report generation

    Account Limitations

    ⚠️ Workflow Creation Limits

    Demo and trial accounts may have workflow creation limits. The validation testing encountered a 21-workflow maximum on the demo account. Production accounts typically have higher or unlimited workflow capacity depending on your subscription plan.

    Action Required: Contact your account manager to understand workflow limits for your organization.

    System Configuration Requirements

    • Role Management: Line Manager and other approval roles must be configured in Role Management for approval-based workflows
    • Department Structure: Departments and offices must be created in Settings for department-based workflow criteria
    • Approval Flows: Existing approval flow configurations (for expenses, payroll, tickets) work in conjunction with Workflows
    • Slack Workspace: Admin access to your Slack workspace required to generate webhook URLs for Slack integration

    Related Features

    Workflows connects to and enhances these Bayzat features:

    • Approval Flows: Workflows can trigger notifications at different stages of approval processes for expenses, payroll transactions, leave requests, and employee tickets
    • Task Management: Automatically create tasks when workflow conditions are met, assigning them to specific employees or roles
    • Employee Onboarding: Automate welcome emails, document requests, and task assignments for new hires
    • Payroll Cycles: Trigger notifications when payroll cycles close or when salary adjustments occur
    • Leave Management: Automate manager notifications when leave requests are submitted or approved
    • Air Ticket Management: Use the Air Tickets Auto Encashment workflow template to automatically generate encashment requests

    Integration Points

    Workflows serves as the central automation hub connecting:

    • Internal Events: 13+ Bayzat applications generating 60+ distinct event types
    • External Actions: 20+ applications available for workflow actions
    • Data Sources: Employee profiles, payroll data, leave balances, ticket information
    • Communication Channels: Email, Slack, mobile notifications, task assignments

This interconnected architecture allows you to create sophisticated automation sequences that span multiple systems and business processes, all managed from a single workflow interface.
