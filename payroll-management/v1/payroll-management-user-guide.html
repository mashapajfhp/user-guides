What is Payroll Management?

Overview

Payroll Management is Bayzat's centralized system for processing monthly employee salaries, managing expenses, and handling payroll transactions. Access it from the main navigation under Payroll.

The payroll table shows employee compensation data with options to add adjustments, submit transactions, and close monthly pay runs. Use the currency tabs to manage employees paid in different currencies.

Key Benefits

  • Process payroll for multiple currencies in a single system without parallel Excel files
  • Submit partial or full salary payments multiple times per month with flexible transaction control
  • Automate salary calculations with allowances, deductions, and expense reimbursements
  • Generate WPS-compliant SIF files and payslips for closed pay runs
  • Track transaction approval status with audit trails for all payroll changes
  • Carry forward unpaid amounts automatically to the next month's payroll

Product Foundation Overview

How Payroll Management Fits

Payroll Management is a transactional system that processes monthly employee compensation, expenses, and salary payments. It operates as the execution layer where salary configurations are applied to generate actual payment transactions.

Mental model: Salary Configuration (template) ‚Üí Applied to Employees ‚Üí Monthly Payroll Processing ‚Üí Transactions Generated ‚Üí Payments Executed

Each payroll month affects all active employees with configured salaries. Changes to salary components during a month only affect that month's payroll table and future months.

Key Decisions Before Setup

Answer these questions before processing payroll:

  • Which currencies do your employees earn in? ‚Äî UAE companies with overseas employees need multi-currency support to eliminate parallel Excel processes
  • What salary components need tracking? ‚Äî Define allowances, deductions, variable pay categories, and additions before first payroll run
  • How will you process payments? ‚Äî Choose between wire transfer, cash deposit, or check deposit via Lulu Exchange
  • Do you need approval workflows? ‚Äî Determine if transactions require multi-level approval before payment processing

Related Features

  • Salary Configuration ‚Äî Defines base pay, allowances, and deductions that populate the payroll table each month
  • Expense Management ‚Äî Submits reimbursement requests that can be added to payroll or processed offline
  • Attendance Tracking ‚Äî Automatically adjusts payroll based on time off, late arrivals, and violations
  • Payslip Generation ‚Äî Creates employee payslips in base currency after payroll transactions are processed
  • WPS File Generation ‚Äî Produces SIF files for government reporting and bank submission

Multi-Currency Navigation

If your organization manages payroll in multiple currencies, the payroll table displays currency tabs at the top.

  • Each currency has a separate tab (e.g., AED, USD, EUR)

Initial Setup

First-Time Configuration

Before processing your first payroll, complete these foundational configurations to ensure accurate salary calculations and compliant payment processing.

1. Configure Payroll Settings

Navigate to Settings ‚Üí Payroll to establish your payroll structure.

  • Create Allowance Categories: Define custom allowances (housing, transportation, etc.) that will appear in employee salary breakdowns
  • Set Up Addition Categories: Configure positive pay adjustments for bonuses, commissions, and other variable pay components
  • Define Deduction Categories: Establish deduction types for loans, advances, or other salary reductions
  • Configure Currency Support: Enable multi-currency payroll if you have employees earning in non-AED currencies

Note: Categories cannot be created in bulk. Each must be added manually through Settings ‚Üí Payroll.

2. Upload Trade License

Required for payroll processing through Lulu Exchange.

  1. Go to Company Documents
  2. Click Upload file
  3. Select your trade license document
  4. Tick the box to register with Lulu Exchange for Payroll Processing
  5. If using WPS, enter your Freezone Authority Account number

3. Configure Payroll File Templates

Set up templates for bank submissions and government reporting (DEWS, WPS).

  1. Navigate to Settings ‚Üí Payroll ‚Üí Payroll File Templates
  2. Click Add New
  3. Choose template type: Payroll Processing or Reporting
  4. Select a pre-defined template or upload your existing file
  5. Define template details (name, payment method, bank, trade license)
  6. Map data fields to system columns
  7. Save the template

‚ö†Ô∏è Critical: Carefully map data fields during template setup. Incorrect mappings will cause transaction processing failures.

4. Set Employee Base Currency

For each employee earning in non-AED currency:

  1. Access employee salary configuration
  2. Select currency from searchable dropdown (default: AED)
  3. Verify no paid/unpaid items exist before changing currency
  4. Save currency setting

‚ö†Ô∏è Currency Change Restrictions: Cannot update currency if any pay items have been paid, any unpaid items exist (net-pay must be 0), unpaid variable pay exists, unpaid adjustments exist, or unpaid recurring deductions exist for active/future months.

5. Configure Deduction Policies (Optional)

If you need automated attendance violation penalties:

  1. Go to Settings ‚Üí Attendance ‚Üí Deduction Policies
  2. Click Add New
  3. Name the policy and set manager visibility
  4. Choose tracking method (Standard or Advanced)
  5. Define penalty levels (up to 4 levels, 4 occurrences each)
  6. Configure deduction formulas (rate multiplier, salary component, days calculation)
  7. Assign employees to the policy
  8. Save configuration

Why Workflows Are Not Supported

Payroll processing requires human oversight and approval at multiple stages to ensure accuracy, compliance, and proper financial controls. The system is designed around a structured approval flow rather than automated workflows.

Alternative Automation Options

  • Attendance Integration ‚Äî Deduction policies automatically calculate penalties based on attendance violations

Related Automation Features

Edge Cases

Multi-Currency Scenarios

  • Partial currency processing: Each currency tab requires separate transaction submission. Cannot submit mixed-currency batch
  • Currency validation errors: Multiple specific error messages guide resolution. Address each blocker sequentially
  • Migration edge case: Existing employees default to AED. Verify currency assignment before first payroll in new system

Transaction Timing

  • Same-day processing: Cash/check deposits before 12:00 PM enable same-day release, subject to Central Bank delays
  • Check settlement: Banks typically take 1-2 working days. Plan submission timing accordingly
  • Wire transfer verification: Verify fund transfer by end of day before intended payment date
  • Lulu branch deposit: Do not hand over check at Lulu branch (extends turnaround by up to 4 days)

Payroll Closure Scenarios

  • Partial month closure: Can close for all employees, individuals/groups, or specific salary components at different times
  • Unpaid items handling: Fixed pay appears as Arrears. Variable pay, additions, expenses, deductions push to next month as new items
  • Zero net pay employees: Employees with zero unpaid net pay are considered fully paid automatically

Approval Flow Edge Cases

  • Rejection after approval: Transactions can be rejected even after initial approval
  • Rejected transaction handling: Cannot move rejected transactions back to pending. Amounts return to payroll table
  • Approver cannot modify: Approvers review only. Cannot edit submitted transaction details

Deduction Policy Edge Cases

  • Work disruption penalties: Different penalty types apply for work disruption vs standard violations
  • Repeated violations: Violations within policy refresh duration tracked differently than isolated incidents
  • Manager visibility: Line managers may have limited visibility of monetary values based on settings

Data Entry Constraints

  • Allowance categories: Must be created manually. Cannot create in bulk
  • Deduction categories: Must be added manually through Settings. No bulk creation
  • Addition categories: Manual creation only. No batch import available

Export & Reporting Limitations

  • Office column missing: Payroll exports lack office-level granularity for location-specific analysis
  • Paid/unpaid aggregation: Cannot view combined paid and unpaid items in single view. Requires separate tracking
  • Column customization: Cannot hide/show columns. Must work with default view

UI Interaction Quirks

  • Navigation constraints: No click & drag scrolling. Use scrollbars or trackpad

FAQs & Support

Frequently Asked Questions

How do I submit payroll transactions for employees in different currencies?

Navigate to the payroll table and select the currency tab. Choose employees or pay items to include, then click Submit Transaction. Each currency requires a separate transaction submission.

Can I change an employee's currency mid-month?

No. Currency changes are blocked if any paid or unpaid items exist, including approved expenses, variable pay, adjustments, or recurring deductions for active or future months.

What happens to unpaid amounts when I close the payroll month?

Unpaid fixed pay items appear under Arrears in the next month. Unpaid variable pay, additions, work expenses, and deductions are pushed to the following month's payroll table.

Can I undo a submitted payroll transaction?

Yes, but only for custom cycle transactions in the current open month. Click the undo button, optionally enter a rejection reason, and confirm. Rejected transactions cannot be moved back to pending status.

How do I download payroll data for a specific month?

From the Payroll Table, click Download and select the desired month. For closed pay runs, use the Transactions tab to download historical data in Excel format.

Why can't I see all columns in the payroll table?

The payroll table does not currently support customizable column visibility. All default columns are displayed and cannot be hidden or reordered.

Can I filter or search transactions by specific criteria?

Yes. Use filters to search by cardholder/card name, expense category, date range, or transaction details. Filtering capabilities vary by transaction type and status.

How do I track the status of submitted transactions?

Navigate to Finance ‚Üí Transactions. View transactions by status (pending, approved, rejected) and track approver actions, comments, and payment processing stages.

What payment methods are supported for payroll processing?

Bayzat supports wire transfers, cash deposits, and check deposits through Lulu Exchange. Each method has specific timing requirements for same-day salary processing.

Can I process partial salary payments?

Yes. Uncheck specific pay items or employees before submitting a transaction. You can submit multiple transactions within the same month to pay different components at different times.
