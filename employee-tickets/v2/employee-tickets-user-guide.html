What is Employee Tickets?

Overview

Employee Tickets centralizes employee requests through a structured ticketing system accessible via Requests → Employee Tickets. Admins review and approve tickets for attendance regularization, leave encashment, and custom request types.

Create tickets via the purple button, filter by category/type/priority, search by ticket ID or employee name, and approve/reject inline or via detail view.

Key Benefits

  • Eliminates email-based request tracking with centralized ticket management
  • Automates attendance record updates when tickets are approved
  • Routes requests through configurable approval flows by ticket type
  • Tracks ticket status across Pending/Approved/Rejected tabs with activity logs
  • Enables custom ticket types with fields, eligibility rules, and approval routing
  • Integrates with workflows to trigger pay adjustments or notifications

Strategic Context

What Employee Tickets Solves

Replaces manual tracking of employee requests via email or paper forms with a structured digital system.

  • Replaces scattered email threads and spreadsheets
  • Standardizes request submission and approval across all request types
  • Enables audit trails with activity logs and status tracking

Why It Matters

Acts as the central request management layer connecting employees, managers, and HR systems.

  • Routes requests through Approval Flow configurations
  • Drives attendance record updates and payroll adjustments
  • Controls access to custom request types via employee mapping

How It Connects

Transactional feature that processes employee-initiated requests and updates related modules.

  • Impacts individual employee records and payroll when approved

Product Foundation Overview

How Employee Tickets Fits

Employee Tickets is a transactional request system that digitizes internal employee requests across categories like attendance, leave, payroll, and custom processes. It sits between employee-facing request forms and approval routing, capturing structured data that flows through configured approval hierarchies before updating core HR records.

Mental model: Employee submits ticket → Routed through approval flow → Approved/Rejected → Attendance/Payroll/Leave records updated automatically

Each ticket type can have unique fields, eligibility rules, and approval flows, making this a flexible request management layer across the entire HR system.

Key Decisions Before Setup

Answer these questions before configuring Employee Tickets:

  • Which request types need approval flows? — Determine which ticket types (attendance regularization, leave encashment, custom) require line manager vs HR admin approval
  • Who can create tickets on behalf of employees? — Decide if line managers and admins can raise tickets for their team members
  • What custom ticket types are needed? — Identify internal processes (IT requests, office supplies, business travel) that should be digitized through tickets
  • Which employees can access which ticket types? — Map ticket type visibility by department, office, role, or employee group

Related Features

Initial Setup

Before You Begin

Employee Tickets requires minimal initial setup. The feature is ready to use once enabled, with two pre-configured ticket types available immediately: Attendance Regularization and Leave Encashment.

Required Configuration Steps

Step 1: Configure Approval Flows

Set up approval routing for each ticket type to ensure requests reach the right approvers.

Navigate to Settings → Approval Flow → Select ticket type → Configure approval hierarchy using Advanced (If conditions) or Default (Else) flows.

Note: Each ticket type requires its own approval flow configuration. Changes to approval flows apply to new tickets immediately; pending tickets migrate to the new flow, but approved tickets remain unaffected.

Step 2: Verify Employee Mapping

Confirm which employees can access each ticket type.

  • Navigate to Settings → Employee Tickets → Ticket Types
  • Review employee mapping for Attendance Regularization and Leave Encashment
  • Default mapping: All active employees
  • New employees must be manually added to ticket types as needed

Step 3: Configure Admin Roles

Assign admin roles who can view and manage employee tickets.

  • Navigate to Settings → Employee Tickets → Ticket Types
  • Edit each ticket type to assign admin roles with Employee Ticket Admin policy
  • Line Managers automatically see tickets from direct and indirect reportees

Recommended Configuration

Set Up Custom Ticket Categories

Organize ticket types by creating categories for easier management.

  • Navigate to Settings → Employee Tickets → Categories
  • Click Add Category
  • Enter category name and description in English and Arabic
  • Assign ticket types to categories for better organization

Configure Workflows for Automation

Automate actions when tickets are created or approved using the Workflows guide.

  • Navigate to Settings → Workflows
  • Create workflows triggered by "Employee ticket is created" or "Employee ticket status is updated"
  • Configure actions like sending email notifications or creating pay adjustment requests

Enable Notification Preferences

Ensure approvers receive alerts for pending tickets.

  • Notifications are sent automatically via email and in-app alerts for all CRUD operations
  • Approvers receive notifications when tickets are assigned to them
  • Ticket creators receive notifications when tickets are approved or rejected

Validation Checklist

Verify your setup is complete before going live:

  • Approval flows configured: Each ticket type has at least one approver assigned
  • Employee access verified: Target employees can view and submit relevant ticket types
  • Admin roles assigned: HR team members have Employee Ticket Admin policy enabled
  • Test ticket submitted: Create a test ticket to verify the end-to-end approval process

Post-Setup Actions

After initial configuration, consider these optional enhancements:

  • Create custom ticket types for organization-specific requests (business travel, IT assets, office supplies)
  • Set up custom fields using List Library for reusable dropdown options
  • Configure eligibility rules to restrict ticket types by department, office, or country
  • Review and adjust approval flows based on organizational hierarchy changes

Approval Flow

How Employee Tickets Approvals Work

Employees submit tickets through My Requests > My Tickets or admins create tickets on their behalf. Each ticket type follows a configured approval flow—typically starting with the Line Manager, then moving to Super Admin or department-specific approvers. Once all approvers in the sequence accept the ticket, it moves to Approved status and triggers any connected actions (like updating attendance records or creating pay adjustments). Rejected tickets remain visible with rejection comments.

Types Requiring Approval

Over 40 ticket types have configurable approval flows. Common types include:

  • Attendance Regularization — Missing check-in/check-out corrections (default: Line Manager)
  • Leave Encashment — Cash-out unused leave balance (default: Super Admin or Line Manager)
  • Loan — Employee loan requests (default: Super Admin)
  • Expense — Work expense reimbursements (default: Line Manager)
  • Leave Salary Request — Salary advance during leave (default: Super Admin)
  • Shift Change Request — Schedule modification requests (default: Line Manager)
  • Work From Home Request — Remote work approval (default: Line Manager)
  • Business Trip Request — Travel authorization (default: Super Admin)
  • Bank Account Update — Payroll account changes (default: Super Admin)
  • Salary Changes — Compensation adjustments (default: Super Admin)

Each type supports Advanced flows (If conditions based on department, office, amount, etc.) and a Default flow (Else fallback). Configure in Settings > Approval Flow.

Common Approval Scenarios

Typical approval workflows for Employee Tickets:

Scenario 1: Employee submits an attendance regularization ticket for a missing check-out → Line Manager receives notification and reviews ticket details including original check-in time → Manager approves → Attendance record updates automatically with corrected check-out time

Scenario 2: HR admin creates a leave encashment ticket on behalf of an employee → Ticket routes to Super Admin for review → Super Admin checks leave balance calculation and approves → Pay adjustment request generates automatically for inclusion in current payroll cycle

Scenario 3: Employee submits a loan request ticket for 10,000 AED → Line Manager reviews and approves (first level) → Super Admin receives ticket for final approval → Super Admin approves → Loan disbursement processes and recurring deduction schedule creates in payroll

Scenario 4: Employee submits a shift change request → Line Manager rejects with comment explaining scheduling conflict → Employee receives rejection notification with reason → Employee can edit and resubmit the ticket with adjusted dates

Scenario 5: Admin creates a per diem ticket for employee business travel → Ticket uses Advanced flow with condition: if amount > 5,000 AED, routes to CFO; else routes to Line Manager → CFO approves high-value ticket → Workflow triggers two pay adjustment entries automatically

View pending tickets in Requests > Employee Tickets. Use filters to show "Pending approval on me" for tickets awaiting your action. Inline Approve/Reject buttons appear on tickets where you are the current approver.

Edge Cases

  • Clicking row selects checkbox instead of opening ticket: This is by design. Click on the Ticket ID link specifically to open the detail dialog
  • Service period not met for air ticket encashment: Encashment should not be generated until eligibility criteria are satisfied. Verify employee service period in their profile
  • Policy changes mid-cycle: System handles updated intervals gracefully. Pending tickets migrate to new policy rules
  • Employee termination with outstanding tickets: Outstanding encashments need proper handling. Process or reject tickets before employee departure date
  • Multiple air ticket policies: Employees may have different air ticket policies. Verify which policy applies before processing encashment
  • Manual override needed: HR may need ability to adjust automated requests in exceptional cases. Use Edit option in three-dot menu
  • Attendance regularization with shift changes: Shifts must be published as working days before check-ins can be accepted. Check-ins received before shift publication are automatically rejected
  • Custom ticket type with conditional fields: Conditional fields may become invisible based on other selections. Data is retained but hidden

FAQs & Support

Frequently Asked Questions

Can I edit a ticket after it's been approved?

No. Once approved, tickets cannot be reverted to pending. Any changes must be made manually in the employee's record (e.g., attendance or payroll).

Why don't I see the Approve/Reject buttons on some tickets?

Inline approval buttons only appear on tickets where you are assigned as an approver in the approval flow.

What happens when I reject a ticket?

Rejected tickets move to the Rejected tab. Employees receive a notification and can view your rejection reason in the activity log.

Can employees see tickets created on their behalf?

Yes. Employees can view tickets created by admins under My Requests > My Tickets in their employee portal.

How do I configure a custom ticket type?

Navigate to Settings > Employee Tickets > Ticket Types, then click Add Ticket Type. Define fields, eligibility rules, and approval flows. See the Set up Custom Employee Tickets article for step-by-step instructions.

Why is the ticket type field disabled on the create form?

You must select an employee first. The ticket type dropdown populates based on which ticket types that employee is eligible to submit.

Can I change the approval flow for a ticket type that already has pending tickets?

Yes. Pending tickets migrate to the new approval flow. Approved tickets remain unaffected.

What does the Beta badge mean?

Employee Tickets is currently in Beta, meaning new features and improvements are being actively developed. Core functionality is stable and production-ready.

How do I add a custom role to the approval flow?

Custom roles must be enabled for Employee Tickets by Bayzat support. Once enabled, they appear in the approval flow dropdown under Settings > Approval Flow.

Can line managers create tickets on behalf of their team members?

Only if configured. By default, ticket creation requires the Employee Tickets Admin policy. Configure this under Settings > Tickets > Ticket Type > Admin Roles.
