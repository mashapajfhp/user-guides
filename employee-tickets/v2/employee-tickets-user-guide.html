Product Foundation Overview

How Employee Tickets Fits

Employee Tickets is a transactional request system that captures and routes employee requests for attendance corrections, leave encashments, and custom organizational needs through structured approval workflows.

Mental model: Employee submits ticket ‚Üí Routed through approval flow ‚Üí Approver reviews ‚Üí Approved/Rejected ‚Üí System updates (attendance record, payroll adjustment, etc.)

Each ticket type can have its own approval routing, eligibility rules, and custom fields, affecting how requests are processed across departments and offices.

Key Decisions Before Setup

Answer these questions before configuring Employee Tickets:

  • Which request types do you need? ‚Äî Attendance regularization and leave encashment are premade; custom tickets require configuration.
  • Who can submit each ticket type? ‚Äî Map employees by department, office, or individual to control access.
  • Who approves what? ‚Äî Configure approval flows in Settings ‚Üí Approval Flow for each ticket type before employees can submit.
  • What fields are required? ‚Äî Custom tickets need field definitions (text, number, date, list, attachment) before creation.

Initial Setup

Before employees can submit tickets or admins can process requests, configure ticket types and approval flows in Settings.

Configure Ticket Types

Navigate to Settings ‚Üí Employee Tickets ‚Üí Ticket Types to define which request types employees can submit.

For Premade Ticket Types (Attendance Regularization, Leave Encashment)

  • Edit name and description (English and Arabic)
  • Map eligible employees (default: all active employees)
  • Assign admin roles who can view and manage tickets
  • System fields are view-only and cannot be modified

For Custom Ticket Types

  • Create categories to organize ticket types (e.g., Payroll, IT, Attendance)
  • Define ticket name, description, category, and priority
  • Add custom fields: Text, Number, List, Date, Time, Attachment
  • Mark fields as mandatory or optional with character limits
  • Set eligibility rules by Department, Office, Country, Gender

New employees: Must be manually added to ticket types as needed. Default mapping includes only existing active employees at time of creation.

Set Up Approval Flows

Configure who reviews and approves each ticket type in Settings ‚Üí Approval Flow. See the Approval Flow guide for detailed configuration steps.

  • Each ticket type requires its own approval flow configuration
  • Use Advanced flows with If conditions for conditional routing
  • Set Default flow with Else fallback for standard cases
  • Add approval steps with multiple approvers using AND/OR logic
  • Default approver: Line Manager 1 (can be customized)

Approval flow changes: Pending and partially approved tickets migrate to the new flow immediately. Fully approved tickets remain unaffected.

Grant Admin Access

Assign the Employee Tickets Admin policy to roles who need to view and manage tickets.

  • Navigate to Settings ‚Üí Roles & Permissions
  • Select or create a custom role
  • Enable Employee Tickets Admin policy
  • Assign role to relevant users

Note: Line Managers automatically see tickets from their direct and indirect reportees without requiring the admin policy.

Enable Notifications

Ticket creators and approvers receive email and in-app notifications for all ticket CRUD operations by default. Users can manage notification preferences in their account settings.

Verify Beta Access

Key Tasks

Task: Manage Employee Ticket Requests

Review, approve, or reject employee tickets submitted through the system, ensuring requests are processed according to company policies and approval flows.

Subtask: Create Ticket on Behalf of Employee

  • Navigate to Requests ‚Üí Employee Tickets
  • Click Create employee ticket button (top-right)
  • Select employee from dropdown
  • Choose Category, then select Ticket type (enabled after employee selection)
  • Fill required fields
  • Click Submit

Admin only: Creating tickets on behalf of employees requires Employee tickets admin policy.

Subtask: View Ticket Details

  • Click on Ticket ID link (e.g., T-63) to open detail dialog
  • Review ticket info: Category, Status, Priority, Ticket type, Requested by, Created date
  • Check ticket-specific fields (e.g., Leave type, Number of days, Encashable amount for Leave Encashment)
  • View calculation formula if applicable
  • Review activity timeline and comments
  • See assigned approvers and current approval status

Subtask: Approve or Reject Tickets

  • Navigate to Requests ‚Üí Employee Tickets ‚Üí Pending tab
  • Locate tickets pending your approval (Reject/Approve buttons visible inline)
  • Click ticket to review details
  • Click Approve to accept the request, or Reject to decline
  • Add comments if needed by tagging employees using @
  • Ticket moves to Approved or Rejected tab automatically

Important: Once a ticket is fully approved, it cannot be reverted to Pending. Further changes must be made manually in the employee's record (e.g., attendance or payroll).

Subtask: Edit or Delete Tickets

  • Click the three-dot menu icon on any ticket row
  • Select Edit to modify ticket details
  • Select Delete to remove the ticket (if permitted)
  • Select View ticket to open detail dialog

Subtask: Filter and Search Tickets

  • Click Filters button to expand filter panel
  • Apply filters: Category, Ticket type, Priority, Created date range, Departments, Offices, Reports to
  • Check "Pending approval on me" to see only tickets awaiting your approval
  • Use search bar to find tickets by Ticket ID, Employee ID, or Employee name
  • Results update automatically as filters are applied

Subtask: Download Ticket Reports

  • Navigate to Requests ‚Üí Employee Tickets
  • Apply filters if needed to scope the report
  • Click Download button (top-right)
  • Report generates asynchronously and downloads when ready
  • Report includes ticket data from the current tab (Pending/Approved/Rejected)

Approval Flow

How Employee Tickets Requests Route

Employee Tickets requests route through configurable approval flows based on ticket type. Each ticket type can have its own approval hierarchy, with approvers assigned by role (Super Admin, Line Manager) or custom roles. Requests move sequentially through approval steps until all approvers in the flow have reviewed.

Types Requiring Approval

Over 40 ticket types support approval flows. Common types include:

  • Attendance Regularization ‚Äî Routes to Line Manager by default
  • Leave Encashment ‚Äî Routes to Super Admin or Line Manager (configurable)
  • Loan Requests ‚Äî Requires Super Admin approval
  • Expense Reimbursements ‚Äî Routes to Line Manager, escalates to Finance for high amounts
  • Work From Home Requests ‚Äî Routes to Line Manager
  • Business Trip Requests ‚Äî Routes through multi-step approval flow
  • Shift Change Requests ‚Äî Routes to Line Manager
  • Hourly Permission ‚Äî Routes to Line Manager
  • Bank Account Update ‚Äî Routes to Super Admin
  • Employee Grade Change ‚Äî Routes to Super Admin

Total ticket types with approval flows: 40+ (includes custom ticket types created by your organization)

Routing Examples

How different Employee Tickets requests route through approvers:

Example 1: Attendance regularization tickets route to the employee's Line Manager for single-step approval

Example 2: Leave encashment tickets route to Super Admin or Line Manager based on configured approval flow (Advanced flow supports conditional routing with If/Else logic)

Example 3: Loan requests always require Super Admin approval regardless of amount

Example 4: Expense reimbursement tickets route to Line Manager first, then escalate to Finance role if the amount exceeds a defined threshold (configured in Advanced flow)

Edge Cases

Approval Flow Changes

  • Mid-Process Flow Change: Pending tickets automatically migrate to the new approval flow. Approved tickets remain unaffected.
  • Approver Removed: If an approver is removed from the flow while tickets are pending, those tickets route to the next approver in the updated flow.
  • Multiple Approvers: When using AND logic, all approvers in that step must approve before the ticket advances. With OR logic, any one approver can advance it.

Employee Mapping

  • New Employee: New hires do not automatically appear in ticket type eligibility. Add them manually or ensure your filter rules include their attributes.
  • Department Change: If eligibility is based on department and an employee transfers, their access to ticket types updates automatically based on filter rules.
  • Manual Override: Manual include/exclude rules take precedence over filter-based eligibility, even if filters would suggest otherwise.

Custom Ticket Configuration

  • Category Reassignment: Ticket types can be moved between categories. Existing tickets retain their original category assignment.
  • Field Deactivation: Deactivating a field hides it from new tickets but preserves data in existing tickets.
  • List Library Updates: Changes to a shared list affect all ticket types using that list. Edit with caution.

Notifications

  • Non-Approver Visibility: Line managers not in the approval flow cannot receive automated notifications about tickets their reportees submit. They must check manually or be added to the approval flow.
  • Notification Preferences: Users can enable/disable ticket notifications via email or mobile app settings.

Reports and Downloads

  • Asynchronous Reports: Ticket reports are generated asynchronously. Large exports may take time to prepare.
  • Filter Persistence: Applied filters remain active when downloading reports, ensuring exported data matches the view.

Integration Edge Cases

Attendance Regularization

  • Shift Publication Timing: Attendance regularization tickets can only be approved if the shift is published as a working day. Check-ins before shift publication are rejected.
  • Post-Approval Changes: Once approved, attendance changes apply immediately. Further corrections must be made directly in the attendance record.

Leave Encashment

  • Policy Eligibility: Employees must meet service period requirements before encashment requests can be submitted.
  • Payroll Integration: Approved encashment tickets can be added to the current payroll or processed separately. Choose the option that matches your payroll cycle.
  • Reversal: Encashment tickets can be moved back to Pending if errors are found before payroll processing. After payroll is finalized, adjustments require manual correction.

Custom Tickets with Workflows

  • Trigger Timing: Workflow triggers fire when ticket status changes. Ensure approval flow is complete before relying on "approved" triggers.
  • Action Failures: If a workflow action fails (e.g., email send error), check the workflow activity log for details. The ticket status is unaffected.

FAQs & Support

Frequently Asked Questions

Can I edit a ticket after it's been approved?

No. Once a ticket is approved, it cannot be reverted to Pending or edited. Any changes must be made manually in the employee's record (e.g., attendance or payroll adjustments).

Why is the ticket type field disabled on the create form?

The ticket type selector remains disabled until you select an employee. Once you choose an employee, the system filters ticket types based on their eligibility and the categories available to them.

How do I know if I need to approve a ticket?

If you are an assigned approver for a ticket, you will see inline Approve/Reject buttons on the ticket row in the Pending tab. You also receive email and in-app notifications when a ticket requires your approval.

Can I approve tickets on behalf of another approver?

No. Each approver in the approval flow must review and approve the ticket themselves. Tickets move sequentially through the configured approval flow and cannot skip approvers.

What happens if I reject a ticket?

When you reject a ticket, it moves to the Rejected tab and the employee is notified. Rejected tickets can later be moved back to Pending if the employee resubmits or if an admin decides to reconsider.

Why don't I see Approve/Reject buttons on some pending tickets?

Inline approval buttons only appear on tickets where you are the current assigned approver. If the ticket is pending approval from someone else in the flow, you will only see the three-dot menu with View, Edit, and Delete options.

How do I filter tickets assigned to me?

Use the "Pending approval on me" checkbox in the Filters panel. This shows only tickets where you are the current approver in the approval flow.

Can line managers create tickets on behalf of employees?

Yes, if the ticket type configuration allows it. Admins can configure which roles (including Line Managers) have permission to create tickets on behalf of employees. Check Settings > Tickets to verify role permissions for each ticket type.

What does the Beta badge mean?

Employee Tickets is marked as a Beta feature, meaning it is fully functional but may receive updates, refinements, or additional capabilities based on user feedback. Report any issues or suggestions to your Bayzat support contact.

How do I download a report of all tickets?

Click the Download button at the top of the Employee Tickets page. You can apply filters before downloading to generate a report for specific ticket types, date ranges, departments, or statuses. The report downloads asynchronously and you will receive a notification when it's ready.

Can I bulk approve or reject tickets?

Currently, bulk actions via checkbox selection are visible in the UI but the functionality is not fully documented. Use the three-dot menu or inline buttons to approve/reject tickets individually.

Why do custom roles not appear in ticket approval flows?

Custom roles require backend enablement to appear in approval flow configurations. If a newly created custom role is not visible in the approval flow dropdown, contact Bayzat support to enable the role for ticket approvals.
