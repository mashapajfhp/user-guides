What is Employee Tickets?

Overview

Employee Tickets digitizes internal employee requests through a centralized ticketing system. Admins manage tickets under Requests > Employee Tickets, where they can review, approve, or reject submissions across categories like Attendance, Leave, Payroll, and custom types.

Each ticket displays employee name, type, category, priority, creation date, and assigned approvers. Use inline Approve/Reject buttons or the three-dot menu to process tickets.

Key Benefits

  • Eliminate email-based request tracking with structured ticket workflows
  • Route requests through configurable approval flows with visibility into status
  • Automatically update attendance records or payroll adjustments upon approval
  • Create custom ticket types with specific fields, eligibility rules, and approval routing
  • Filter and search tickets by employee, type, category, priority, or date range
  • Track ticket history and comments in a centralized activity log

Initial Setup

Before employees can submit tickets, configure ticket types and approval flows in Settings.

Configure Ticket Types

Employee Tickets ships with two premade types (Attendance Regularization, Leave Encashment) and supports custom types.

Settings ‚Üí Employee Tickets ‚Üí Ticket Types

For premade ticket types:

  • Edit name and description (fields are view-only, cannot add/delete system fields)
  • Map employees who can submit this ticket type
  • Assign admin roles who can manage tickets

For custom ticket types:

  1. Click Add Ticket Type
  2. Enter name, description, category, priority
  3. Add custom fields (Text, Number, List, Date, Time, Attachment)
  4. Mark fields as mandatory/optional
  5. Set eligibility rules by Department, Office, Country, Gender
  6. Save ticket type

Note: New employees must be manually added to ticket types as needed by company policy.

Set Up Approval Flows

Each ticket type requires an approval flow. Configure in the Approval Flow guide.

Settings ‚Üí Approval Flow ‚Üí Select ticket type

Default approval flow: Line Manager 1. Customize with Advanced flows (If conditions) for conditional routing by department, amount, or ticket type.

Important: Changing approval flows affects pending tickets immediately. Approved tickets remain unaffected.

Create Ticket Categories

Organize ticket types into categories (e.g., Payroll, IT, Attendance) for easier navigation.

Settings ‚Üí Employee Tickets ‚Üí Categories
  1. Click Add Category
  2. Enter category name (English and Arabic)
  3. Add description
  4. Save category

Note: Cannot delete categories with associated ticket types. Reassign ticket types to a different category first.

Configure Notifications

Notifications are sent automatically for ticket CRUD operations (create, update, approve, reject) to creators and approvers via email and in-app alerts.

Enable/disable notifications in user preferences (mobile or email).

Optional: Set Up Workflows

Automate actions when tickets are created or status changes. See Workflows guide for configuration.

Example workflows:

  • Send email when ticket is created
  • Create pay adjustment request when ticket is approved

Approval Flow

Settings ‚Üí Approval Flow ‚Üí Employee Tickets

How Approvals Work

Each ticket type has its own configurable approval flow. Tickets route through assigned approvers sequentially or in parallel, with status updates triggering notifications.

Approval signals detected:

  • Approve/Reject buttons on pending tickets
  • Approvers column in ticket list
  • Status transitions (Pending ‚Üí Approved/Rejected)
  • Advanced approval flow with If conditions
  • Default approval flow with Else fallback
  • Step-based approval with Add step/Add block
  • Approver assignment (Super Admin, Line Manager)
  • Activity timeline showing approval actions

  • Default flows: Fallback approval path when no conditions match

Default Flow Example

For Leave tickets, the default flow routes to:

  • Step 1: Super Admin OR Line Manager - 1 (either approver can approve)

Tip: Use Advanced flows to route approvals based on conditions. For example, route high-value expense tickets to Finance Manager, while low-value tickets go directly to Line Manager.

Approval Flow Configuration

Each ticket type's approval flow is configured independently. Key options:

  • Sequential approval: Each approver must approve before moving to the next step
  • Parallel approval: Multiple approvers can approve simultaneously (AND/OR logic)
  • Conditional routing: Route based on ticket attributes (department, amount, priority)
  • Approver roles: Assign by role (Super Admin, Line Manager, custom roles) or specific employees

Configure approval flows in the Approval Flow guide.

Approval Actions

Approvers can:

  • Approve: Move ticket to next approval step or mark as Approved if final step
  • Reject: Move ticket to Rejected status with optional comment
  • Comment: Add internal notes visible to other approvers and admins
  • Tag employees: Notify specific users about the ticket

Important: Once a ticket is fully approved, it cannot be reverted to Pending. Any changes must be made manually in the relevant module (e.g., attendance record, payroll adjustment).

Edge Cases

Approval Flow Changes Mid-Process

If you modify a ticket type's approval flow while tickets are pending:

  • Pending and partially approved tickets automatically migrate to the new flow
  • Fully approved tickets remain unaffected
  • Rejected tickets remain in Rejected tab

Employee Termination with Pending Tickets

When an employee leaves with pending tickets:

  • Outstanding tickets remain in the system
  • Approvers can still process tickets for historical record-keeping
  • Approved tickets after termination do not trigger payroll or attendance updates

Multiple Policies and Ticket Types

Employees may have different ticket types based on policies:

  • Air ticket policies determine encashment eligibility
  • Leave policies control which leave encashment tickets appear
  • Custom ticket types can have department-specific eligibility

Manual Override Scenarios

HR may need to adjust automated ticket requests:

  • Service period not met: Encashment should not generate until eligibility satisfied
  • Policy changes mid-cycle: System handles updated intervals, but manual review recommended
  • Exceptional cases: Use Edit option to adjust ticket details before approval

Shift Publication and Attendance Tickets

Attendance regularization tickets have strict validation:

  • Shifts must be published as working days before check-ins accepted
  • Check-ins before shift publication automatically rejected
  • Check-ins after shift status changes require manual attendance correction

Data Integrity Warnings

Excel Import Constraints: When bulk updating employee records, remove sensitive data (passport numbers, visa details) to prevent upload errors. Download error reports if upload fails and correct issues before re-uploading.

Search History Errors: Deleted employees in search history may cause frontend errors. Clear browser cache or use the global search bar (CMD/CTRL+K) to refresh recently viewed employees.

FAQs & Support

Frequently Asked Questions

Can line managers create tickets on behalf of employees?

Currently, only attendance admins can create tickets. Line managers cannot submit attendance tickets for team members. This is a known limitation for organizations with large, non-desk workforces.

Why don't I see my custom role in the approval flow dropdown?

Newly created custom roles require backend enablement to appear in approval workflows. Contact support to enable your custom role for ticket approval flows.

Can I approve a ticket after it's been rejected?

Yes. Rejected tickets can still be approved later by navigating to the Rejected tab and using the Approve action.

Why is the ticket type field disabled on the create form?

The Category > Ticket type selector remains disabled until you select an employee. Choose an employee first, then the ticket type field becomes active.

Can I see approval buttons for all pending tickets?

No. Inline Approve/Reject buttons only appear on tickets where you are an assigned approver. Other tickets show only the three-dot actions menu.

What happens if I click on a table row instead of the Ticket ID?

Clicking a table row toggles the checkbox selection rather than opening the ticket detail. Click the Ticket ID link specifically to view ticket details.

How do I configure approval flows for 40+ ticket types?

Each ticket type requires individual approval flow configuration. Navigate to Settings > Approval Flow, select the ticket type, and define Advanced flows (If conditions) and Default flow (Else fallback) for each.

Can I revert an approved ticket back to pending?

No. Once a ticket is fully approved, it cannot be moved back to Pending or Rejected tabs. Any further changes must be made manually in the employee's attendance record.
