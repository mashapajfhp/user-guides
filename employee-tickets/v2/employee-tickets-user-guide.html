What is Employee Tickets?

Overview

Employee Tickets standardizes how employees submit and track requests for attendance corrections, leave encashments, and custom organizational needs. Admins configure ticket types, employees submit structured requests, and approvers process them through defined workflows‚Äîeliminating email threads and manual tracking.

Tickets move through approval flows, update source records automatically (attendance, leave balances), and provide audit trails for all requests.

Key Benefits

  • Eliminate email-based request tracking with structured ticket workflows
  • Automate attendance and leave balance updates after approval
  • Enforce consistent approval routing by department, office, or role
  • Audit all request activity with timestamps and approver comments
  • Extend beyond pre-built types with custom ticket configurations

Product Foundation Overview

How Employee Tickets Fits

Employee Tickets is a transactional request system that digitizes internal HR processes like attendance corrections, leave encashments, and custom requests.

Mental model: Ticket Type (template) ‚Üí Submitted by Employee ‚Üí Routed through Approval Flow ‚Üí Processed by Admin ‚Üí Record Updated

Each ticket type can have custom fields, eligibility rules, and approval routing, impacting how requests are processed across the organization.

  • Which employees need access to each ticket type? ‚Äî Determine eligibility by department, office, or role
  • What custom fields are required? ‚Äî List data points needed for each request type (dates, amounts, attachments)

Initial Setup

First-Time Configuration

Before employees can submit tickets, configure ticket types and approval flows.

1. Configure Ticket Types

Navigate to Settings ‚Üí Employee Tickets ‚Üí Ticket Types to view premade ticket types (Attendance Regularization, Leave Encashment) or create custom types.

For premade ticket types:

  • Edit name, description, and employee mapping
  • Configure admin roles who can view and manage tickets
  • System fields are view-only and cannot be modified

For custom ticket types (Release 3):

  • Create categories to organize ticket types
  • Define custom fields (Text, Number, Date, List, Attachment)
  • Set eligibility rules by department, office, country, or gender
  • Configure which roles can create tickets

2. Set Up Approval Flows

Configure approval routing in the Approval Flow guide.

Default approval flow routes to Line Manager 1. Customize flows to:

  • Add multiple approval levels (sequential or parallel)
  • Assign specific roles as approvers
  • Define approval order and logic

Note: Pending tickets automatically migrate to new approval flows if you change the configuration. Approved tickets remain unaffected.

3. Map Employees to Ticket Types

By default, all active employees can access premade ticket types. For custom tickets:

  • Navigate to Settings ‚Üí Employee Tickets ‚Üí Ticket Types
  • Select a ticket type and click Edit
  • Choose "Eligible Employees" tab
  • Select specific employees, departments, or groups
  • Save changes

‚ö†Ô∏è New Employees: Must be manually added to ticket types as needed by your company. They are not automatically included in existing ticket type mappings.

4. Enable Notifications

Email and mobile push notifications are sent automatically for ticket CRUD operations. Configure notification preferences:

  • Creators receive alerts when tickets are approved or rejected
  • Approvers receive alerts for pending tickets requiring action
  • Users can enable/disable alerts via mobile or email settings

Recommended Configuration

For optimal ticket management:

  • Create ticket categories to organize custom ticket types by function (Payroll, IT, Attendance)
  • Use the List Library for reusable dropdown options across multiple ticket types
  • Configure custom roles with Employee Ticket Admin policy for granular access control
  • Test approval flows with a sample ticket before rolling out to all employees

  • Default mapping: All active employees are included by default

Approval Flow Behavior

Mid-process changes: Pending or partially approved tickets migrate to new approval flow when flow is modified. Approved tickets remain unaffected.

  • Sequential processing: Each approver in the flow must review before ticket moves forward
  • Final approval: Once fully approved, tickets cannot be moved back to Pending or Rejected tabs
  • Post-approval changes: Further modifications must be made manually in the relevant module (e.g., attendance record for regularization tickets)

FAQs & Support

Frequently Asked Questions

Can I configure custom approval flows for different ticket types?

Yes. Each ticket type can have its own approval flow configured in Settings ‚Üí Approval Flow. You can set sequential or parallel approvers, assign roles, and define multiple approval levels per ticket type.

What happens to pending tickets if I change the approval flow?

Pending and partially approved tickets migrate to the new approval flow automatically. Fully approved tickets remain unaffected and retain their original approval status.

Can line managers create tickets on behalf of employees?

Yes, if configured. You can assign roles that allow line managers to submit tickets for their reportees. Configure this in the ticket type settings under "Who can create this ticket."

Why don't custom roles appear in ticket approval flows?

Custom roles must be manually enabled for ticket approval workflows. Contact support to enable specific custom roles for use in employee ticket approval flows.

Can employees edit tickets after submission?

Employees can edit pending tickets before approval. Once approved, tickets cannot be edited by employees‚Äîadmins must make changes manually or reject and resubmit the ticket.

How do I restrict which employees can see specific ticket types?

Use employee mapping in ticket type configuration. You can assign access by individual employees, departments, offices, or custom groups. Only mapped employees will see and submit that ticket type.

Can I delete a ticket type that has existing tickets?

No. Ticket types with existing tickets cannot be deleted. You can deactivate the ticket type to prevent new submissions while retaining historical data.

What's the difference between premade and custom ticket types?

Premade ticket types (Attendance Regularization, Leave Encashment) have system-defined fields that are view-only. Custom ticket types allow full control over fields, categories, and eligibility rules.
